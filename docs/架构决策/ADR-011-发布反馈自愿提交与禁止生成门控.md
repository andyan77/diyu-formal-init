# ADR-011：发布反馈自愿提交与禁止生成门控

- 状态：已接受
- 接受日期：2026-07-18
- 对应决策：D-025（2026-07-18 修订）
- 取代范围：[ADR-004](ADR-004-门店内容自主留痕与发布反馈闭环.md) 中的强制反馈门、反馈债务、例外放行及企业跨工作区正文查看建议

## 背景

用户此前原则接受过可配置的“上传上一条发布截图后才能继续”的反馈闭环门。2026-07-18，用户明确以新裁决覆盖旧裁决：系统可以提倡使用者上传此前发布内容的平台截图和反馈，但任何情况下都不能把上传变成继续生成内容的强制条件。

该修订符合系统“赋能而非监管”的定位，也避免首篇内容、平台延迟、批量排期、未发布草稿和多平台账号造成死锁。缺点是反馈样本会存在自选择偏差，不能用缺少截图推断内容表现差或用户未履责。

## 决策

1. 发布截图、平台数据和评论反馈一律由使用者自愿提交。租户、总部、区域或门店管理员均不能配置为强制条件。
2. 未上传、跳过提醒或长期不提交反馈，不得阻断内容生成、修改、重新生成、定稿、导出或下一项任务；系统不建立“反馈债务”，也不需要管理员例外放行。
3. 系统可以在使用者标记内容已发布后进行非阻断提醒，说明上传反馈有助于改进后续生成；提醒必须可以跳过、关闭或稍后处理，不能通过反复弹窗形成事实强制。
4. 用户未提交反馈不能被标记为违规、失职或负向内容信号。系统只记录“当前没有用户上报反馈”，并在分析时明确处理样本缺失和自选择偏差。
5. 用户可以为任意一条已发布内容自愿提交反馈，不必限定为“上一条”。反馈至少关联对应内容版本、平台、实际 `PublishingAccount`、发布时间和数据截取时间。截图中的昵称、头像、账号和评论个人信息默认脱敏；截图及手工数据标记为用户自报，不声称经过平台验证。
6. 自愿反馈可形成 `EvaluationSignal`，用于 [ADR-010](ADR-010-全量生成留痕与机器自评回路.md) 的机器分析；播放、点赞、评论、成交或高采用率都不能自动升级为品牌规则、事实知识或素材授权证据。
7. 企业总部、区域和租户管理员不因反馈回路或系统自评获得跨工作区反馈截图、具体正文和原始提示的查看权。上传者按本人/工作区权限管理其反馈；机器处理继续服从 ADR-010。
8. 将来接入平台接口取得经平台验证的反馈，属于可选连接器和新的证据等级；不能据此取消用户是否连接、是否回传的自主权，除非后续形成新的明确业务与合同决策。
9. ADR-004 中的门店内容自主、系统硬约束、生成版本留痕以及反馈不自动升级为知识继续有效；其“可配置强制门”“反馈义务/债务”“例外放行”和企业跨工作区正文查看建议自本 ADR 生效起废止。

## 影响

- D-025 结论以 2026-07-18 本次裁决为准；原来的强制反馈闭环门不得进入产品或实施基线。
- 产品保留低摩擦反馈入口与温和提醒，但不建设阻断状态机、债务计数、例外审批或批量放行能力，工程复杂度下降。
- 反馈数量可能减少且带自选择偏差，系统质量评估必须结合全量生成行为、采用/编辑信号和其他可得反馈，不能只看主动上传截图的样本。
