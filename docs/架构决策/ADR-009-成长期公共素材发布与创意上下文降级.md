# ADR-009：成长期公共素材发布与创意上下文降级

- 状态：已接受
- 接受日期：2026-07-18
- 对应决策：D-029

## 背景

用户提出在成长期建设区域公共素材和公司公共素材：总部、区域代理和加盟门店的有效账号使用者可以主动贡献本人认可的私人素材；当当前知识与素材不足以组装高质量创作上下文时，系统可以按配置从公共素材补充。用户同时要求第一版只预留扩展能力，不开发公共素材模块。

该方向若把“转入公共区”“总部官方”“正式知识”和“隐藏检索”混成一件事，会穿透私人素材权限、放大低质投稿、跨品牌误用，并用创意素材猜补事实；若第一版直接预建完整空模块，则违反 D-024。

## 决策

1. 第一版继续执行 [ADR-008](ADR-008-个人私有原始素材隔离与首期禁止共享.md)：不提供私人素材共享、上提、转组织、公共素材区或公共检索。公共素材能力只在成长期另行立项、开发、迁移和验收后注册启用。
2. 成长期区分三个素材空间：
   - 区域共享素材池：向该区域合作网络内的授权成员开放；
   - 品牌网络共享素材池：向租户内符合适用品牌条件的授权成员开放；
   - 总部官方素材库：仍由总部维护，具有独立的官方权威等级。
   “素材包”是公共素材版本的清单或保存查询，不为区域复制文件或另建物理数据库。
3. 所有当前有效的自然人账号都可以贡献本人私有素材中的选中版本。贡献时创建权限、版本和生命周期独立的投稿快照与公共版本，不移动私人原件，也不放宽私人素材箱权限；底层文件可以按内容哈希去重。管理员只能查看投稿快照和公共版本，不能借贡献流程打开私人源。
4. **贡献权不等于自动生效权。**区域素材管理员负责激活区域公共版本，总部素材管理员负责激活公司公共版本；只有已激活且被标记为允许自动检索的版本，才可被系统自动调用。`contributed`、`active`、`suspended`、`withdrawn`、`expired` 等状态必须明确留痕。公开可见、允许成员人工选用、允许自动检索和总部官方精选不得互相推定。
5. 素材激活只治理共享资产池，不是区域或总部对加盟店自媒体内容的发布审批，不改变渠道内容自主权。激活也不表示系统已经完成法律权利裁定；贡献者仍对权利说明负责，素材管理员负责租户内部的公共池质量与使用范围管理。
6. 公司公共版本必须声明适用品牌为企业通用或明确品牌集合。公共贡献不自动成为总部官方素材、品牌规则、事实知识或事实证据；热度、复用次数和内容表现也不能改变其权威等级。
7. 所谓“隐藏设置”落实为总部授权管理员可见、默认关闭、带版本和生效时间且全程审计的租户级或品牌级高级开关。普通创作者不负责配置，但生成任务必须提示本次是否使用公共素材、显示来源并允许排除。
8. 每种 `ContentProduct` 通过任务合同声明上下文需求并区分：
   - 品牌/商品资料与明确规则：缺失时不得从公共素材猜补；人工价格、库存、活动未由用户在本次任务提供时，直接不写入方案，只有用户明确要求数据依赖成果时才询问；
   - 创意与生活语境：场景、故事线索、消费者语言、视觉参考、地方生活细节和表达灵感等；只有此类软槽位不足且高级开关开启时，才可触发公共素材补充。
9. 区域/门店任务默认先检索本区域，再按策略扩大到公司范围；总部全国任务直接检索公司范围，除非明确指定区域。召回前必须按租户、品牌、区域、激活/自动检索状态、用途、权利和有效期过滤；排序同时考虑相关性、本地性、新鲜度、多样性、近期复用次数和内容相似度，防止公共池制造同质化。
10. 公共素材、OCR、语音转写、AI 说明和向量都是媒体资产及其派生检索数据，不进入 `KnowledgeStatement`。这些内容按不可信数据处理并与系统指令隔离；生成时把可验证事实和仅供灵感的公共素材分区投喂。
11. 每次自动补充记录任务缺口、策略版本、查询范围、候选与选中公共版本、过滤原因、实际投喂来源及产物派生关系，支持来源解释、排除、撤下影响分析和质量追踪。
12. 第一版不创建公共素材专属表、接口、菜单、权限、索引、开关、后台任务或空提供器。第一版只落实当前已有来源也真实需要的能力：稳定素材编号/版本/来源谱系、内容任务的硬/软缺口表达，以及由品牌知识、组织官方素材和本人选中素材实际使用的版本化上下文来源合同；成长期再注册公共素材模块。

## 影响

- D-029 关闭，公共素材成为已确定的成长期路线，不增加第一版功能范围和交付承诺。
- ADR-008 的第一版私人素材隔离完全不变；将来增加公共素材也不能通过修改私人区访问控制实现。
- D-028 已由 [ADR-010](ADR-010-全量生成留痕与机器自评回路.md) 关闭；D-011 后续已由 [ADR-026](ADR-026-内容前情复用与正式知识升级分离.md) 关闭。BBS/讨论区仍须独立治理，不能借公共素材模块越权或捆绑开发。
- 成长期详细设计仍需确定公共版本撤回后的既有使用处置、人工可见/选用细则和质量验收阈值，但这些不反向阻塞第一版。
