# ADR-006：赋能而非监管与责任不转移

- 状态：已接受
- 接受日期：2026-07-17
- 对应决策：D-027

## 背景

笛语为品牌企业、区域代理和加盟终端提供知识、内容生成、陈列建议、规则检查和审计能力。若把这些辅助能力写成系统替客户承担事实确认、素材授权、专业判断、最终复核或发布监管责任，会使产品角色、组织责任和验收合同混乱；反过来，若把“赋能工具”写成系统提供方完全免责，也会错误排除其对自身系统和约定能力的责任。

## 决策

1. 笛语是赋能系统，不是品牌企业、区域代理或加盟店的经营监管者，也不接管其日常审批岗位。
2. 笛语不代替各经营主体完成事实确认、素材授权、专业判断、最终复核、制作、执行和发布；各经营主体分别对自身输入、决定和行为负责。
3. 具体操作人与所属组织之间的内部责任由组织制度和协议确定，系统不作劳动关系、内部处分或法律责任裁定，只保存必要的身份、版本和行为审计链。
4. 权限隔离、明确规则提醒和“不得擅自编造/改写输入”等辅助防错能力，不构成“法律已经合规”“平台一定放行”或“事实由系统担保”的承诺，也不转移发布/执行主体的最终复核责任。第一版不对人工输入的库存、价格、活动等经营信息作真实性判断或冲突裁决。
5. SaaS 提供方仍对自身提供的系统能力、安全、租户与权限隔离、审计、受托数据处理、接口约定行为、明确承诺功能的正确执行，以及适用法律规定的其他不可排除义务负责。
6. 一个结果可以存在多段责任：事实或素材提供方对输入负责，选择、修改、制作、执行和发布者对相应行为负责，系统提供方对其系统能力负责；不得机械地把全部责任推给最后点击发布的人，也不得由系统替合同和法律作最终责任裁定。

## 正式口径

> **笛语负责赋能，不替客户监管；不接管各经营主体的事实确认、素材授权、专业判断、最终复核、制作、执行与发布责任。品牌企业、区域代理和加盟店等经营主体分别对自身输入、决定和行为负责；具体操作人与所属组织的内部责任由组织制度和协议确定，系统只作审计追溯。系统提供方对自身系统能力、安全、权限、审计、受托数据处理、明确承诺功能的正确执行以及适用法律规定的其他不可排除义务负责。**

## 2026-07-18 内容输入补充说明

用户进一步明确：使用者对其输入或配置的人设、经历、生活事件以及最终发布负责；笛语不调查创始人、店长、加盟商或其他内容角色的人生经历是否在外部世界真实发生，也不把“人设是否会崩塌”做成系统真实性审批。

这一补充不等于把用户声明认证为客观事实，也不等于取得第三方人物、媒体或跨工作区使用的法定授权。系统将其记录为 `user_provided` 内容前提，在指定品牌、账号、角色和内容产品范围内使用；同时区分用户前提与系统新增的专业解释、情景演绎和其他派生断言。系统不得无输入依据改变人物核心身份，也不得把自身派生循环升级成商品、人物履历或定量经营硬事实。具体增益与回写合同由 D-031 继续裁决。

系统按品牌方提供的商品资料和版本化规则辅助防错，并忠实使用用户本次人工经营输入；不承诺这些外部资料本身绝对真实、规则完整无遗漏、内容必然合法或平台一定放行，也不判断动态经营真值；本 ADR 第 4—6 条继续有效。

## 影响

- 内容生产、陈列搭配及未来应用统一沿用该责任原则。
- 产品界面不得显示“系统已保证合法”“平台一定通过”“发布后责任由系统承担”或“使用系统即可免审”。
- 硬阻断可以作为产品安全边界存在，但不能被描述为系统已经接管客户监管。
- 审计记录用于还原责任链，不等于系统对最终法律责任作出判决。
