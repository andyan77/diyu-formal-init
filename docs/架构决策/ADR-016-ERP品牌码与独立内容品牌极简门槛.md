# ADR-016：ERP 品牌码与独立内容品牌极简门槛

- 状态：已接受
- 接受日期：2026-07-21
- 对应决策：D-021
- 上游决策：[D-014 品牌内容战略基线与动态创作方向编译](../讨论纪要/2026-07-21-046-P3D黄金选择与品牌创作方向编译候选.md)

## 背景

一个服装企业可能经营多个商品品牌码，但组织、人员、渠道和账号并未按品牌拆分。ERP 品牌码只能证明商品归属被区分，不能证明每个品牌已经具有独立定位、目标人群关系和内容人格。若系统仅凭品牌码制造不同声纹、故事或价值观，就会把模型创作冒充企业事实。

## 决策

第一版不建设 0—3 级品牌成熟度状态机，只使用一道业务门：

1. **只有 ERP 品牌码时**：该品牌码仅用于商品归属、检索和过滤；内容沿用当前企业已经确认的共同表达。系统不得凭空生成该品牌独立的定位、人格、故事、价值观、目标受众关系或声纹。
2. **升级为独立内容品牌时**：必须由企业按照 D-014 明确确认该品牌独立、当前有效的内容战略基线。确认生效后，该品牌才取得独立内容身份，系统可以据此编译品牌内容和后续账号矩阵。
3. **组织是否拆分不构成门槛**：同一套人员、组织和渠道可以服务多个独立内容品牌；集团式独立组织也不会仅因组织完整而自动证明品牌内容基线已经成立。组织与品牌继续保持正交。
4. **内容身份不等于权限或账号**：升级为独立内容品牌不自动创建发布账号、内容角色、组织权限或审批关系；这些仍按企业 IP 矩阵和既有账号合同独立配置。
5. **以当前确认版本为准**：没有当前有效的独立品牌基线时，系统回到企业共同表达，不从历史稿、模型推断、高表现内容或商品差异自动升级。

## 明确不做

- 不建设四级状态机、成熟度评分器、自动升级算法或品牌人格推断 Agent；
- 不要求企业为每个 ERP 品牌码建立独立组织架构；
- 不把商品品类、价格带、Logo、账号名称或模型生成内容单独当作独立品牌人格证据；
- 不在本决策冻结 `Brand` 的字段、表或物理状态，具体表达留给 D-016。

## 影响

D-021 正式关闭。第一版独立内容品牌的唯一激活依据是企业确认的 D-014 品牌内容战略基线，而不是 ERP 品牌码或组织拆分。D-016 不再需要设计品牌成熟度状态机；待 D-005 关闭后即可综合内容与经营陈列需求冻结最小语义模型。
