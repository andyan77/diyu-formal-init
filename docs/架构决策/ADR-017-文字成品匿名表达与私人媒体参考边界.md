# ADR-017：文字成品匿名表达与私人媒体参考边界

- 状态：已接受
- 接受日期：2026-07-21
- 对应决策：D-030
- 上游边界：[ADR-008：个人私有原始素材隔离与首期禁止共享](ADR-008-个人私有原始素材隔离与首期禁止共享.md)、[ADR-012：授权期个人隔离与退出后所属主体接管](ADR-012-授权期个人隔离与退出后所属主体接管.md)、[ADR-015：首期平台媒体成品与跨平台重编译](ADR-015-首期平台媒体成品与跨平台重编译.md)

## 背景

笛语第一版只生产文字型内容成品。D-032 所称视频和图文成品，是正文、台词、解说、分镜、动作、画面安排、声音建议和制作说明等可执行文字方案，不是系统生成、渲染或编辑图片与视频文件。

账号私人素材区可以保存本人用于创作的图片和视频，但“媒体可作参考”不等于系统必须建立媒体清理、匿名化或生成能力。此前按文字、截图、照片、语音、视频分别设计清理副本的方案错误扩大了第一版范围，现由本 ADR 取代。

## 决策

1. **第一版只交付文字型内容成品**：系统不生成、渲染、剪辑、修改或匿名化图片和视频文件。
2. **私人媒体只服务该账号创作**：用户主动保存到账号私人素材区的图片和视频可以依照既有私人素材规则，在该账号后续内容任务中作为参考；临时使用但未主动保存的素材只服务当前任务。
3. **作用域不因使用而扩大**：私人媒体不因被系统调用而进入品牌知识库、组织官方素材区或公共素材区，授权期内不允许其他账号查看；账号停用后的处置继续适用 ADR-012。
4. **生成文字默认去标识化表达**：成品涉及具体顾客时，默认不复述姓名、联系方式、账号号、订单号等个人标识，改用自然的非识别性称谓。这是一条文字表达规则，不创建顾客实体、顾客画像或顾客数据业务。
5. **交互只保留友情提示**：系统可以在上传或首次使用私人媒体时提示用户先排除不希望用于创作的信息，但不建立风险评分、逐项确认或审批门。
6. **输入理解与内容生产分开**：未来实现是否能直接理解图片或视频属于多模态输入能力，不等于多模态内容生产，也不由 D-030 承诺。

## 明确不做

第一版不建设：

- OCR 定位并回写遮挡图片；
- 人脸识别、人物授权判断或自动模糊；
- 声纹识别、自动变声；
- 视频逐帧追踪、匿名视频输出；
- 图片/视频生成、渲染和自动剪辑；
- 顾客库、CRM、顾客画像、权利图谱或逐素材审批；
- 为 D-030 单独建立状态机、评分器或多套工作流。

## 向后兼容与后续扩展

第二阶段或更后阶段可以按真实需求增加图片/视频理解、媒体编辑、媒体生成或专门清理能力，但必须作为独立、版本化、可选的扩展合同接入：

- 不改变第一版文字成品合同和既有私人素材权限；
- 不把旧素材静默纳入新的读取、加工或共享范围；
- 未开启扩展能力的租户和旧任务继续按本 ADR 正常工作；
- 真正立项时再确定输入类型、供应商、处理范围和验收，不在第一版预建空流程、空表或空菜单。

## 影响

D-030 正式关闭。第一版不再需要“敏感信息清理支持矩阵”，也不为 D-016 增加顾客、媒体清理或审批类核心对象。系统只保留私人素材既有作用域和生成文字中的非识别性表达规则；未来扩展不得倒写成第一版已经具备多媒体理解、编辑或匿名化能力。
