# ADR-001：知识库混合路线

- 状态：已接受
- 接受日期：2026-07-17
- 对应决策：D-012

## 背景

项目需要同时处理可验证商品与经营事实、品牌私有知识、长文与媒体证据、复杂服装关系、内容生成和多租户权限。完整本体、知识图谱、SQL、向量检索和 GraphRAG 不处于同一层级，不能作为单项互斥技术选择。

## 决策

1. 从可验收业务任务开始，同步建设约 15—25 个核心类别的轻量语义模型；不先建设覆盖整个服装行业的完整形式化本体。
2. 第一阶段以关系型 SQL 保存权威事实、权限、版本、有效期、审批、状态和审计。
3. 原始文档和媒体文件完整保留；切块、全文索引、摘要和向量均作为可重建的派生检索能力。
4. 知识图谱先由 SQL 中的实体、关系和条件化陈述表达；只有真实任务证明关系遍历成为瓶颈时，才生成独立图查询投影或引入图数据库。
5. GraphRAG 不进入第一阶段权威知识链路；只有全库主题、模式、冲突等跨大量文档综合分析成为稳定高价值任务后，才按租户隔离进行实验。
6. 任何新增技术都必须通过同一黄金问题集比较事实正确、证据可追溯、硬约束违规、租户泄漏、更新时效、成本和维护复杂度。

## 影响

- 语义模型与业务任务共同演进，避免图谱先行和过度建模。
- 商品、库存、价格、权限和授权不会被向量相似度或模型抽取结果替代。
- 第一阶段技术栈保持简单，同时保留向图谱和 GraphRAG 演进的路径。
- 具体数据库产品不由本 ADR 锁定；第一版语义模型现为 25 类候选（包含陈列模块私有的 `DisplayArtifact`），具体类名、关系和必填属性仍由首期业务合同反推后冻结。
