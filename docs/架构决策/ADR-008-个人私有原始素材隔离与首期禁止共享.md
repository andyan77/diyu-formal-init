# ADR-008：个人私有原始素材隔离与首期禁止共享

- 状态：已被 ADR-012 部分修订
- 接受日期：2026-07-18
- 对应决策：D-003（私有素材访问与复用部分）

> **2026-07-18 正式修订**：用户把“私人”限定为所属经营主体授权期间，并正式接受退出后所属主体接管方案。本 ADR 第 1、2、4—9 条在授权有效期间继续适用；第 3 条以及第 4、5 条的退出后部分由 [ADR-012](ADR-012-授权期个人隔离与退出后所属主体接管.md) 取代。品牌总部不得借停用越级取得区域代理或独立加盟店素材。

## 背景

用户要求每个自然人账号的私有素材只由本人查看、编辑和删除；品牌总部、区域代理、同店其他账号及普通管理员均无权访问。曾讨论通过“发布独立副本”把私人素材转入组织工作区，但该流程会增加发布、授权、撤回、版本同步和退出处置复杂度，首期没有足够业务价值。

## 决策

1. 每个自然人登录账号可以拥有 `creator_private` 私有原始素材箱；可多人运营的 `PublishingAccount`、组织岗位和系统服务身份不能拥有个人私有素材箱。
2. 私有原件仅账号使用者本人拥有业务界面的读取、编辑和删除权限；总部、区域代理、同店其他账号和普通管理员均无人工读取权限。受控系统服务只在用户主动发起生成任务时处理所选原件。
3. **[退出部分已被 ADR-012 取代，保留作历史]** 原结论为：总部暂停或终止账号、门店或区域的租户使用资格，不取得其私有原件的访问、占有或复用权。现行规则是授权期间仍不得访问，退出窗口届满后由上传时记录的实际所属经营主体接管，详见 ADR-012。
4. 第一版不提供私有原件转组织素材、跨账号共享、跨层级授权复用、共享副本或素材讨论区；私人素材只能由本人用于自己的生成任务。
5. 总部、区域和门店仍可分别维护由其授权管理员直接录入的组织官方素材工作区，但它与个人私有原件完全分开，二者之间首期没有转换通道。
6. 私有素材箱不等于私人工作成果区。总部员工在岗位任务中形成的 `ContentBrief`、`ContentItem`、人工修改版本和审计记录仍进入总部组织业务工作区，由品牌企业管理；代理与门店生成的业务内容分别进入其经营主体工作区。
7. 使用者可以编辑或删除私有原件并立即停止未来检索和生成。删除不级联抹除已经发生的生成、导出、发布和最小审计链；只保留必要的素材编号、哈希、引用关系、操作者、时间和删除状态。
8. 未来若出现明确的素材交流需求，可以另行立项 BBS/讨论区；当前只进入路线图，不预建表、接口、菜单、权限或空后台任务。
9. 系统机器自评回路不由本 ADR 决定，已由 D-028/[ADR-010](ADR-010-全量生成留痕与机器自评回路.md) 确认；机器分析不得反向打开、复制或长期隐藏保留个人私有原件，也不产生企业人工查看权。

## 后续路线补充（不改变首期结论）

2026-07-18，用户确认在成长期另建区域公共素材和公司公共素材，并允许使用者主动贡献本人认可的私人素材；区域/公司公共版本分别由区域/总部素材管理员激活后，才可被系统自动调用。D-029 已关闭并形成 [ADR-009](ADR-009-成长期公共素材发布与创意上下文降级.md)。在成长期模块正式立项、迁移和验收以前，本 ADR 的第一版禁止共享结论完全不变。

公共素材模块与 BBS 是两种不同能力。私人原件不能被公共权限直接穿透；使用者贡献时发布一个权限和生命周期独立的公共版本，并继续遵守版本化模块合同。第一版不为该成长期模块预建公共作用域、表、接口、菜单、索引、开关或后台任务。

## 影响

- 首期不会出现私人素材共享、上提、授权副本和撤回同步流程，显著降低权限与生命周期复杂度。
- 区域代理首期不能进入门店成员的私人素材箱帮助整理，也不能把优秀私人原件直接上报总部；其赋能通过知识、任务方法和系统生成内容的抽象反馈完成。
- 将来增加公共素材模块或 BBS/讨论区都属于新模块的前向扩展，必须使用版本化合同，不能复用或悄悄放宽现有私人素材权限；二者也不得相互捆绑。
