# ADR-022：真人未成年人原件首期禁入与亲子内容保留

- 状态：已接受
- 接受日期：2026-07-22
- 对应决策：D-020
- 决策母稿：[D-020 真人未成年人原件首期准入边界确认稿](../讨论纪要/2026-07-22-085-D020真人儿童素材首期边界候选.md)
- 上游边界：[ADR-006](ADR-006-赋能而非监管与责任不转移.md)、[ADR-017](ADR-017-文字成品匿名表达与私人媒体参考边界.md)、[ADR-021](ADR-021-组织官方素材工作区与持续供给.md)

## 背景

笛语第一版只交付文字型内容成品，但总部、区域和门店账号可以在私人素材箱或组织素材区保存图片、视频。若其中包含可识别真人未成年人，笛语即使只把原件作为创作参考，也已经参与保存、调用或可能的外部模型处理，不能用“系统只生成文字”“品牌已授权”或“用户自行负责”消除这一处理边界。

D-020 只决定真人未成年人原件能否进入第一版，不限制亲子题材、童装商品或亲子品牌内容生产。

## 决策

### 1. 亲子内容保留，真人未成年人原件首期禁入

第一版继续支持亲子穿搭、家庭生活、成长场景、家长顾虑、儿童商品解释和亲子门店关系内容。

总部、区域和门店账号均不得把可识别真人未成年人的照片、视频或声音上传、保存或复用到私人素材箱和组织素材区。禁入同时覆盖当前对话临时附件：不得以“不长期保存”为由把原件直接发送给外部模型。总部已经在笛语之外取得拍摄、营销或发布授权，也不作为第一版例外。

由原件产生的缩略图、转写、摘要、嵌入或其他派生载荷，只要仍可还原、识别或合理关联到具体未成年人，也适用同一规则。已经去除个人关联的普通生活观察仍可作为非识别性文字种子。

### 2. 用简单业务边界代替年龄识别

第一版不要求门店判断真人的精确年龄，也不建设人脸识别、年龄识别或监护关系判断。素材中的真人明显或可能是未成年人，并且能够被识别或合理关联到具体个人时，账号使用者不得上传笛语。

这是一条第一版产品准入规则，不改变法律对儿童或未成年人的年龄定义，也不表示系统已经自动发现所有误传文件。

### 3. 亲子内容使用可替代输入

亲子任务继续使用：

- 童装商品资料，以及平铺、衣架、假人模特等不含真人儿童的商品画面；
- 不含可识别未成年人的门店空间、货架、包装和服务设施素材；
- 家长、店员或品牌账号的成人叙述；
- 不指向具体儿童的生活观察；
- 用户明确要求的假设、演绎或虚构亲子场景。

系统仍可交付完整的亲子题材正文、台词、解说、分镜、动作、声音和拍摄说明，但不得把虚构内容冒充真实儿童案例、真实试穿结果或真实顾客反馈。品牌在笛语之外依法组织真人拍摄和发布时，第一版不要求把原件或成片回传笛语。

### 4. 误传只停止使用和删除，不建设儿童素材流程

素材入口应以普通用户能够理解的方式提示不得上传可识别真人未成年人照片、视频或声音。仅有提示不构成完整禁入行为：当用户已经明确说明、文件已有明确标记，或当前任务上下文已经知道附件含有可识别真人未成年人时，系统必须在持久化保存或发送外部模型之前拒绝。

对于系统无法自动识别的违规误传，第一版不承诺年龄识别；一旦发现，应停止未来调用并删除原件及系统控制范围内仍可还原或识别的派生载荷，不得在账号退出后进入组织接管区，也不得转为组织、公共或训练素材。不得以审计、备份或训练为名保留隐藏可还原副本；必要事件记录只保留账号、时间和删除结果等不可还原信息。

第一版不为执行该边界建设监护人授权表单、合同核验、媒体审批、自动打码、儿童顾客库、家庭关系图谱或通用权利图谱。

### 5. 后续只能以独立版本化模块增加

只有真实试点证明真人未成年人原件对亲子业态具有不可替代价值，才另行立项。未来模块需要独立确认处理目的和必要性、监护人告知与同意、用途和期限、最小访问及记录、撤回与删除、外部模型或供应商边界、个人信息保护影响评估以及适用的合规审计。

未来模块可以复用既有素材底座，但不得把“允许儿童素材”作为普通素材开关，也不得反向改变第一版私人素材和组织素材的既有含义。

## 与既有决策的关系

- ADR-017 的“私人媒体保持原件、不做媒体匿名化”继续适用于普通媒体；正因为第一版不清理媒体，真人未成年人原件不能进入。
- ADR-021 的私人/组织双入口和各层级素材视图继续成立，但任何入口都不构成真人未成年人原件的例外。
- ADR-006 的“赋能而非监管”继续成立；品牌和使用者对自身制作、事实确认与发布负责，笛语仍对自己实际提供的存储、访问和模型处理能力负责。

## 明确不做

第一版不建设：

- 真人未成年人素材上传、保存、检索、复用或外部模型理解能力；
- 通过当前对话临时附件、批量导入或外链调用绕过禁入边界；
- 总部授权素材包、员工自己孩子或私人素材箱的特殊例外；
- 年龄识别、儿童人脸扫描、监护关系识别或合同真实性判断；
- 监护人授权、逐文件审批、撤回工单、自动打码或视频匿名化；
- 儿童顾客库、家庭关系图谱或通用权利图谱。

## 影响

D-020 正式关闭。第一版保留亲子题材的完整文字生产能力，但不具备真实儿童上身分析、动作验证、本地儿童原件复用或连续真人家庭 IP 共创能力，项目不得把这些未实现能力写入首期承诺。

该决策是首期业务准入边界。它要求对已知或用户明确声明的真人未成年人原件在保存和外部模型调用前拒绝，但不承诺系统能够自动识别所有未知误传，也不证明生产入口已经完成技术实现；生产上线前仍需结合实际部署、素材来源和供应商安排进行专业法律审查。未来增加真人未成年人原件能力必须形成新的明确决策和版本化扩展，不能静默放宽 ADR-022。
