# ADR-014：内容生成运行语义与评测分层

- 状态：已接受
- 接受日期：2026-07-19
- 对应决策：D-031（内容生成共用运行语义）
- 详细母稿：[笛语第一版五类内容产品合同](../内容产品/笛语第一版五类内容产品合同讨论稿.md)

## 背景

D-004/ADR-013 已确认 P1—P5 五类内容产品，但尚不足以唯一决定用户怎样与系统协作、LLM 可以增益到什么程度、哪些内容可以跨任务继续存在、系统怎样交付成品，以及质量检查应在线还是离线执行。

用户要求笛语保持自然、友好和主动的知识协作能力，同时必须能够收敛到内容生产与经营陈列两类核心成果；私人记忆采用轻量方案；系统只向用户交付成品；第一版不接 ERP、不建设交易真值裁决；整个方案必须遵守结果优先、风险相称和最小充分原则，禁止流程过度工程化。

## 决策

### 1. 自然协作与任务形成

第一版交互 UI 在逻辑顶层明确区分“内容生产（对外）”与“陈列搭配（对内）”。用户进入其中一个应用后，每轮输入先作为该应用内的自然对话处理，不自动成为正式任务；顶层选择确定默认应用合同，但不强迫用户填写表单或预先选择 P1—P5。系统可以回应、引导、挖掘、提出选题与计划建议；只有用户已提出可执行要求、接受方案或选择继续已有任务时，才形成 `task_seed`。信息足够时直接执行，只有缺少不可替代输入或用户明确要求的成果与当前应用冲突时作一次最小追问/切换提示。新账号由系统通过轻量问答和已有品牌/账号上下文提出冷启动策划，不建设自治排程系统。

### 2. 日常内容、素材与私人协作偏好分开

日常互动和生成内容默认不进入长期记忆；内容素材只有用户主动保存后才能跨任务复用。系统只自动维护一份轻量、本人专属的协作偏好，以减少追问和提高意图理解。用户可以查看、纠正、删除、关闭偏好，也可以开启一次不读取且不写入偏好的临时会话；这些操作不删除系列计划、已生成内容或 ADR-010 业务留痕。

### 3. 真实种子可充分增益，但关键前提不得改写

用户按真实经历提供生活或工作种子时，系统默认可增加幽默、比喻、情绪、节奏、转折、画面和拍摄设计，不逐项追问普通创作细节；不得改变人物身份、关系、事件结果及用户明确给出的时间、地点、数量等承重前提。完整虚构或戏剧化改写必须由用户明确要求，或明确作为假设、情景演绎或虚构呈现。

### 4. 只保留关键来源的最小内部记录

用户成品不显示主张分类、来源标签、评分或运行轨迹。第一版不建设独立主张对象、逐句标注、权利图谱或专用主张数据库；只复用任务种子、必要的品牌/商品资料引用、素材引用和生成事件，防止创作内容或临时输入反向污染知识库。不得保存或展示模型隐藏推理。

### 5. 人工经营输入不触发交易真值裁决

第一版不接 ERP/POS。用户本次提供的价格、库存、活动等经营信息只作为当前方案和建议的输入；笛语不产生交易、不核验现实真值、不做来源排名、新鲜度评分或冲突裁决。非必要信息缺失时直接省略并继续；只有用户明确要求的数据依赖成果缺少最低输入时，才作一次最小追问。系统不得擅自改写用户输入。

### 6. 只向用户交付完整成品

系统最终呈现与任务和媒体相匹配的成品，可按需包含标题、正文、完整台词或解说、字幕、镜头/图片顺序、拍摄与表演提示、造型、场景、节奏、封面互动、背景音乐方向、环境声和转场音效等。提示词、检索、来源分类、评分、重试、淘汰候选和运行轨迹不进入成品。普通质量问题后台改写；只有缺少不可替代输入或命中权限、隔离、明确硬禁项时，才给出一句最短行动提示。

### 7. 独立单篇、连续系列与用户编排优先

内容连续性采用“独立单篇 / 连续系列”；连续系列仅再区分主题系列与叙事系列，并使用一份轻量系列卡片。系统行为本身不自动改写系列前情，但用户明确制定、接受、跳集、插集、改序或重设时立即生效。已有发布记录和用户明确给定的前情都可以支持“上一期”等承接表达，系统不得自行编造。发布历史是上下文，不是创作通行证。

### 8. 一个独立成品只执行一个主要产品合同

系统根据用户意图和产品任务书自动路由。一个独立成品只执行一个 P1—P5 主要合同；构图、镜头、造型、商品解释、本地语境和平台适配属于共享能力，由系统内部按需组合。只有用户明确要求另一项可独立采用的成果时，才形成第二个成品并分别路由。共享能力条件不足时优先省略、替代或降级，不阻断仍能成立的主要成品。

### 9. 日常轻量自检，完整对照离线执行

日常生成只做快速必要自检，覆盖权限与隔离、明确硬禁项、主任务完成、关键前提与既有输入忠实、品牌相关、应用/产品路由及明显模板化和不可制作问题；不为每次请求额外生成多篇反事实对照稿。

换品牌、换账号、换商品、换地区或换创作权限等完整单变量对照，只用于黄金任务、会实质影响生成行为的版本回归，以及 ADR-010 范围内确有诊断目的的必要抽样。抽样比例、阈值、相似度算法、模型评委组合和永久权重必须由真实评测校准，本 ADR 不予冻结。

## 明确不做

- 不把笛语扩展为无边界通用聊天产品，也不把策划建议自动变成任务、采用或发布；
- 不建设私人本体、记忆图谱、专职 Memory Agent、复杂工作流或在线全量多路评测系统；
- 不向用户或企业管理员展示后台运行报告、具体内容抽样轨迹或模型隐藏推理；
- 不在本 ADR 冻结数据库表、对象是否一等化、代码枚举、评分阈值、模型/SDK 供应商、平台和媒体规格。

## 后续边界

- D-016 决定对象、表/JSON/投影和持久化身份；
- D-030 后续已由 [ADR-017](ADR-017-文字成品匿名表达与私人媒体参考边界.md)极简关闭：第一版只做文字成品中的非识别性表达，不建设媒体清理或匿名媒体生成；
- D-032 决定平台、媒体成品和跨平台验收合同；
- P1—P5 专属条款仍须逐类聚焦验收并建立黄金夹具，完成前不得把语义母稿称为完整开发验收合同。

## 影响

- D-031 九项全部关闭，后续不再重构其共用运行语义；
- 日常生产成本与延迟不会因完整反事实测试而成倍增加；
- 离线评测仍能验证品牌、账号、商品、本地语境和创作权限变化是否产生应有差异；
- 下一项工作转为 P1 专属合同的聚焦验收，而不是继续增加共用字段或提前设计物理数据库。
