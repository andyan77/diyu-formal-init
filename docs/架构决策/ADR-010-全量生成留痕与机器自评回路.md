# ADR-010：全量生成留痕与机器自评回路

- 状态：已接受
- 接受日期：2026-07-18
- 对应决策：D-028

## 背景

用户明确：系统生成的内容无论是否被使用者采用，都必须留痕并接受有价值的机器分析；这既用于识别系统缺陷，也为系统在稳定发展阶段持续迭代提供反馈回路。但该回路不赋予品牌企业、总部或其他企业人员查看具体生成内容的权利。完整分析和迭代引擎放在系统稳定发展阶段，第一版只建立能够积累真实历史数据的基础。

必须纠正两个容易产生的误解：第一，“全量留痕”与“第一版只留空接口”不能同时成立；若第一版不真实记录事件，成长阶段就没有历史样本。第二，“系统自我进化”不能自动解释为跨品牌、跨租户训练共享模型，也不能让生产系统自行修改规则或提示词。

## 决策

1. 本 ADR 所称“内容”限定为内容生产模块的 `ContentItem` 及其版本。每次生成尝试都创建独立的 `GenerationRun` 和输出版本；无论输出被采用、修改、明确放弃还是从未处理/发布，都进入机器质量评估范围。失败、超时、规则阻断和无输出运行也记录为运行样本。陈列模块的 `DisplayArtifact` 仍须留痕，但其正确性、执行反馈和机器评估合同不能照搬内容指标，由陈列模块单独定义。
2. 第一版必须真实记录，而不是只声明未来接口。最小记录包括：稳定运行编号，租户/品牌/工作区，任务与内容产品，操作者身份引用，模型、提示模板、知识与规则版本，输入素材/事实的受控引用或哈希，输出版本引用及哈希，规则检查与错误，生成耗时，以及后续的选中、编辑、放弃、导出、发布和自愿反馈事件。以上均为系统内部记录，不作为用户成品、审阅视图或企业管理员运行报告；分析记录不复制私人原件，也不保存模型隐藏推理。
3. “全部用于分析”不等于“全部都是正向样本”。全部运行可用于覆盖率、失败类型、规则命中、稳定性和质量漂移评估；采用、修改、明确拒绝、发布和有效反馈分别是不同信号。单纯“未采用/用户离开”只能标为结果未知，不能当作负向信号；只有使用者明确拒绝、删除或给出原因时，才可形成相应强度的负向信号。任何未采用内容都不能被当作优质内容经验。
4. 该回路是**机器处理权**，不是企业人工查看权：
   - 品牌总部、区域代理和租户管理员不能因系统自评或数据飞轮打开其他工作区的生成正文、原始提示、反馈截图或私人素材；
   - 企业仍可按正常业务权限查看其自己总部组织工作区中的工作成果，这来自组织业务所有权，不来自机器自评回路；
   - 账号使用者仍可按其本人和工作区权限使用自己的内容；
   - 企业管理员只保留账号治理、安全和运行所必需的最小操作元数据，不获得跨工作区正文查看权；
   - 系统运营人员也没有日常人工查看权，确需故障处置时只能走最小范围、限时、审批并留痕的受控应急访问。
5. 第一版只建设真实使用的事件采集、版本链、权限隔离、保留策略和版本化 `GenerationEvent` 合同；不建设完整数据飞轮平台，不做模型微调，不做自动策略升级，也不预建空分析菜单或自修改代理。成长阶段再以独立分析消费者接入现有事件合同。
6. 成长阶段的自评回路可以形成质量指标、失败聚类、内容结构与风格标签、采用/修改/放弃信号和候选改进方案。任何提示、路由、规则或模型更新必须先形成候选版本，在隔离环境通过回归评测和人工发布审批后才能进入生产；禁止生产系统自主改写自身规则。
7. 生成内容、反馈和派生信号不能自动升级为事实型 `KnowledgeStatement` 或正式品牌规则。D-011 后续已由 [ADR-026](ADR-026-内容前情复用与正式知识升级分离.md) 关闭；内容前情、品牌战略、方法、规则和事实分别使用其既有对象与确认边界。
8. 本 ADR 不授权跨品牌、跨租户复用具体内容、派生特征或训练模型。D-010 后续已由 [ADR-025](ADR-025-首期禁止跨品牌跨租户内容学习与复用.md) 正式关闭：第一版连同匿名汇总和派生复用一并禁止；模型供应商也不得在未经单独明确授权时把输入输出用于训练或改进共享模型。
9. “留痕”不等于永久保存全部正文和敏感数据。完整生成版本、机器分析样本和最小审计摘要分别使用明确的保留期、删除/匿名化规则与加密隔离；正文到期后仍可按审计政策保留不可还原的编号、哈希、版本谱系和聚合评价。具体保留期限进入部署与数据治理配置，不得用无限期保存作为默认值。
10. 企业无权因本回路查看具体内容，也不表示企业永远不能获得任何产品质量报告。将来若提供企业侧汇总质量报告，必须另行定义不可还原、不可指向具体账号/正文的指标合同，不能由本 ADR 自动推出。

## 影响

- D-028 关闭，其准确名称从“总部数据飞轮权”纠正为“系统机器自评回路”；企业人工查看不属于该回路。
- 第一版增加的是生成过程本来就需要的真实事件与版本留痕，不增加成长阶段分析引擎、模型训练或自修改能力。
- 全量样本能支持未来评估，但不会把废稿自动当成正向经验，也不会绕过私人素材和工作区权限。
- D-010、D-011 后续已分别由 ADR-025、ADR-026 关闭；具体数据保留期限继续独立治理。
