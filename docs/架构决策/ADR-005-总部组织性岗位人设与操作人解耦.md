# ADR-005：总部组织性岗位人设与操作人解耦

- 状态：已接受
- 接受日期：2026-07-17
- 对应决策：D-026

> **范围说明（2026-07-21 更新）**：本 ADR 的“系统不做内部协议和逐任务权属核验”继续有效。外部/真人素材是否需要独立 `RightsRecord` 属于 D-016 最小语义模型问题；D-030 已由 ADR-017 极简关闭，只规定私人媒体参考和生成文字非识别性表达，不承担媒体清理或授权判断，二者不得再捆绑。

## 背景

品牌企业希望总部员工基于公司规划的岗位人设生成内容，岗位、人设、企业发布账号运营和工作成果能够在员工更替时持续归公司管理。同时，系统必须保留谁实际操作的审计记录，不能使用多人共享的岗位登录。

## 决策

1. 总部 `ContentRole` 默认是公司规划、不能识别具体员工的组织性内容人设，由品牌企业控制并可授权不同员工先后使用。
2. 实际操作人使用一人一号的自然人登录身份；企业授予或撤销其总部工作区成员资格、系统权限和岗位/人设/发布账号运营授权。员工离岗时暂停或终止其总部工作区成员资格、撤销授权并使相关会话失效，再显式交接组织资产；自然人登录身份不转交，历史记录不改名、不覆盖。
3. 总部工作区、组织岗位、虚构内容人设、企业 `PublishingAccount` 运营授权、任务和业务内容由公司统一查看、管理和交接；不设置员工个人私密工作成果区。
4. 公司与员工之间的劳动管理、义务责任协议及职务成果权属超出本系统业务范围。系统不保存或核验内部协议，不把内部员工协议、职务成果权属或默认虚构岗位人设的员工授权设置为逐任务门卡；通用事实、外部素材、真人媒体、品牌授权和平台规则仍正常执行。
5. 总部内容记录 `business_controller = 品牌企业`、实际操作人和版本链。该字段表达系统内业务控制关系，不等同于系统对著作权归属作出法律裁定。
6. 平台只最小化保存登录和操作审计，并按权限、保留期与安全措施保护；这些属于身份、安全和审计基础设施，不形成员工隐私审批或绩效管理模块。
7. 若未来内容实际使用可识别员工的姓名、肖像、声音、真实经历、数字人或声音克隆，该对象不再属于默认虚构岗位人设，而作为真人媒体资产进入通用授权规则。

## 影响

- 公司岗位人设和官方账号运营不会随员工离职中断。
- 审计能够区分“谁实际操作”与“内容代表哪个组织人设”。
- 首期不开发员工协议管理、劳动关系管理、逐任务员工授权或职务成果版权判定功能。
- `ContentRole` 与系统权限角色保持分离；`PublishingAccount` 成为独立业务对象。
