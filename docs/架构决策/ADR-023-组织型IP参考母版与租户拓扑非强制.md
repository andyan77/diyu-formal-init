# ADR-023：组织型 IP 参考母版与租户拓扑非强制

- 状态：已接受
- 接受日期：2026-07-22
- 对应决策：D-002
- 决策母稿：[组织型 IP 基线复审与五类内容产品推进建议](../讨论纪要/2026-07-18-026-组织型IP基线复审与五类内容产品推进建议.md)
- 上游边界：[ADR-014](ADR-014-内容生成运行语义与评测分层.md)、[ADR-019](ADR-019-第一版25类最小语义模型与兼容扩展边界.md)、[ADR-020](ADR-020-多品牌共享隔离与经营关系.md)、[ADR-021](ADR-021-组织官方素材工作区与持续供给.md)

## 背景

《服装品牌组织型 IP 内容战略与知识库建设基线》完整描述了品牌母 IP、总部—区域—门店协同、多角色权威、内容比例、阶段排期和知识目录。若把整份附件直接当成 SaaS 产品规格，系统就会要求所有新租户拥有相同渠道层级、账号数量和运营流程，进而为单店、集合店、买手店或其他真实业态制造不存在的总部、代理和加盟结构。

D-019 选择“单品牌企业＋一个区域代理＋3—5 家加盟店”作为首个正式试点，只是试点范围收敛，不是笛语的通用租户准入条件。

## 决策

### 1. 附件是参考母版和能力蓝图

附件不作为所有租户必须逐项执行的产品规范。系统保留其中可跨业态成立的方法：

- 依据企业真实组织、渠道和经营关系规划内容账号及角色；
- 明确操作人、发布账号、内容角色、代表组织和角色权威边界；
- 让共同品牌或企业核心保持一致，同时允许不同真实组织和账号进行差异表达；
- 用企业确认版本取代模型临时想象的人设和组织身份。

附件中的固定角色数量、内容比例、总部先行周期、阶段排期、逐级审核、栏目数量和完整知识目录只作咨询及规划参考，不成为所有租户的准入条件、运行硬门、本体约束或验收清单。

### 2. 三级渠道是条件性拓扑，不是租户资格

`Tenant` 可以只有一个 `OrganizationalParty`，也可以具有两级、总部—区域—门店三级或其他真实合作拓扑。不存在的总部、区域代理、加盟店或其他层级不得为了套用附件而创建。

三级渠道只是一组在实际业务存在时成立的组织和 `OperatingRelationship`，不是固定组织树、数据库层级或功能解锁条件。账号和角色规划仍然需要，但规模随真实拓扑缩放；单店租户可以只有当前确实需要的店主、买手、店长或其他真实角色和发布账号。

### 3. 单店集合店和买手店可以独立成租户

满足平台商业与部署准入条件的集合店、设计师买手店或其他独立零售商，可以由一个真实经营组织作为独立 `Tenant` 入驻，不要求拥有自有服装品牌、上级总部、区域代理或加盟网络。

“达到一定体量”属于商业签约和部署配置条件，本决策不凭空冻结营业额、门店面积、SKU 数、账号数或门店数量，也不把这些指标写入本体。

单店独立租户不是新业态类，不新增 `StoreTenant`、集合店专用数据库、业态状态机或特殊账号体系。它继续使用 `Tenant`、`OrganizationalParty`、`PublishingAccount`、`ContentRole`、`Product`、`MaterialAsset` 和既有任务/成果对象。

### 4. 集合店以自身身份表达并显式引用上游品牌

没有自有内容品牌的集合店或买手店，以自己的企业、店铺或策展账号作为主要表达身份，并可以在具体任务中显式引用零到多个上游品牌和真实商品；不得因此混合上游品牌人格、冒充品牌设计者或官方账号。

销售、经销或选品关系只证明相应经营资格，不产生品牌治理权、官方发声权、内部知识权或素材读取权。上游品牌知识和规则只有来源清楚、版本当前且明确适用时才能调用；同名品牌在另一个租户中的数据不会自动跨租户合并。

集合店拥有自有品牌，或者某个上游商品品牌需要成为独立内容身份时，仍按 D-014/ADR-016 完成相应战略基线确认后再启用，不因商品品牌码自动生成人格。

### 5. D-014 按真实表达身份低门槛启动

新租户仍需用 D-014 的低门槛方式形成内容战略和首批账号方案，但目标对象按真实业务确定：

- 自有品牌企业确认品牌内容战略基线；
- 集团或多品牌企业分别确认企业共同表达及确需独立发声的品牌基线；
- 没有自有品牌的集合店或买手店确认自己的企业/店铺表达基线，不替每个上游品牌建立人格；
- 最小账号/角色方案依据真实组织形成，一个组织和一个首批账号也可以成立，不强制称为三级矩阵。

行业品牌原型和角色模板只负责预填和起草，企业确认版本才可用于正式运行。

### 6. 演示品牌与真实试点不改变通用准入

“折线之间”继续作为具有总部—区域—门店结构的完整合成演示实例，用于展示该类品牌如何运行；它不是租户默认复制模板，也不证明所有租户都需要三级渠道。

D-019 的首个正式试点范围保持不变。允许单店集合店入驻属于产品模型的适用资格和前向兼容，不等于集合店已经纳入首期试点，也不等于真实单店、多品牌数据接入或生产隔离已经验证。

## 与既有决策的关系

- ADR-019 的 `Tenant`、`OrganizationalParty`、`Brand` 三轴足以表达单组织租户，不新增核心对象；租户不是品牌总部的同义词。
- ADR-020 已冻结集合店以自身身份表达并引用上游品牌的边界，本决策只明确其可以成为独立租户，不扩大上游品牌权利。
- ADR-021 的总部、区域和门店素材视图是相应组织实际存在时的条件性视图；单店租户只形成本组织素材视图和自然人私人素材箱，不创建空总部或区域视图。
- D-019 继续限定首个真实试点，不限制系统以后服务其他真实租户形态。

## 明确不做

本决策不新增：

- 强制三级组织树或新租户三级渠道校验；
- 单店、集合店、买手店专用核心类别或数据库；
- 业态分类状态机、固定规模阈值或复杂租户资格规则引擎；
- 按附件一次性创建全国账号、完整知识目录或空组织视图；
- 集合店对上游品牌人格、内部知识或素材的自动继承。

## 影响

D-002 正式关闭。组织型 IP 母版提供方法，不规定租户必须具备的组织形状；系统始终依据租户真实存在的组织、品牌/企业表达身份、账号和经营关系形成内容方案。

该决策不需要新增黄金任务、字段、评分器或本体分支。单店集合店的语义已经由 ADR-019/020 支持，但真实单店租户入驻、生产隔离、付费意愿和运营效果仍未验证，不能由架构兼容性代替真实试点证据。
