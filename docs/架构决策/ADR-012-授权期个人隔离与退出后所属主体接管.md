# ADR-012：授权期个人隔离与退出后所属主体接管

- 状态：已接受
- 接受日期：2026-07-18
- 对应决策：D-003
- 修订范围：[ADR-008](ADR-008-个人私有原始素材隔离与首期禁止共享.md) 的账号退出部分

## 背景

用户最初要求个人素材箱始终只有账号使用者本人可以访问，随后明确以最新表述覆盖旧结论：“私人”只定义在所属经营主体授权账号使用期间；账号停用或人员离开后，若使用者未在退出期限内清理，剩余素材应由公司处置。

系统同时包含品牌总部、区域代理和独立加盟店。这里的“公司”不能一律解释为采购系统的品牌总部，必须对应账号实际所属经营主体，否则总部可以借停用账号越级取得渠道主体素材。

## 决策

1. 个人隔离素材的“私人”只表示账号授权有效期间的个人独占可见和操作状态，不表示系统对永久法律所有权作出认定。
2. 授权有效期间，只有账号使用者本人可以查看、编辑、删除和在本人任务中使用原件；总部、区域、同店其他账号及普通管理员不得进入。
3. 账号停用后立即停止正常业务能力，并进入固定退出窗口。窗口内，原使用者只能通过隔离入口查看、导出或删除原件，不能新增素材、生成内容、分享或投稿；管理员只查看数量、容量和截止时间等处置状态，不读取原件。
4. 退出窗口届满后，未清理原件自动转入素材上传时记录并冻结的实际所属经营主体接管区：总部员工对应品牌企业，区域人员对应区域代理经营主体，加盟店员工对应独立加盟店经营主体；不得根据执行停用的人或停用时临时改写的组织关系确定接收方。
5. 所属主体的授权素材管理员可以查看、下载、归档、删除，或在具体业务任务中人工选用接管素材；第一版不增加逐文件交接确认、多级审批或所谓权属迁移流程。
6. 接管区不是组织官方素材区。接管素材默认不参加自动检索，不自动进入组织官方素材、区域/公司公共素材、正式知识、训练数据或私人原件机器自评；人工选用不需要先把它升级为官方素材，但该动作必须留痕。
7. 品牌总部仅因采购系统、管理租户成员资格或执行停用，不得取得区域代理或独立加盟店接管区的素材。区域代理同样不得越级取得独立加盟店素材。
8. 状态转换只改变平台内的可见与操作主体，不创建或修改与该访问控制无关的业务事实。是否需要独立素材使用范围对象，由 D-016 最小语义模型单独决定。
9. 无法解析有效的上传时所属经营主体时，禁止回落给品牌总部；系统删除原件并只保留不可还原的最小审计摘要。
10. 同一自然人在退出窗口内恢复同一经营主体成员资格时，可以取消退出截止时间并恢复原个人隔离状态；素材已进入接管区后，即使账号恢复也不自动回转。
11. 系统记录账号、上传时所属经营主体、停用人及时间、退出截止时间、策略版本、素材编号/哈希、数量/容量以及本人导出/删除、系统接管/清除和后续人工选用事件；不以审计名义保留另一份隐藏原件。
12. `exit_window_days` 是试点开通前必须填写的版本化“租户数据生命周期/账号退出策略”配置，不是 D-003 的语义分歧。第一版租户内统一配置，不做用户级、账号级或逐文件覆盖；停用发生时冻结策略版本和截止时间，后续修改只影响新的停用事件，不得追溯缩短已经开始的窗口。未配置不得投产，具体天数在试点上线配置清单中确定。

## 对 ADR-008 的影响

- ADR-008 关于授权有效期间仅本人可见、不得主动共享/上提及首期不建设共享流程的结论继续有效。
- ADR-008 第 3 条原“停用后企业不取得控制”的历史结论由本 ADR 取代。
- ADR-008 第 4、5 条的“不互转”限定在授权有效期间；退出期满进入所属主体接管区是账号生命周期处置，不是账号使用者主动共享，也不等于进入官方素材区。

## 影响

- D-003 的“谁接管、可以做什么”已经关闭，不再与顾客敏感信息清理或素材权属模型捆绑讨论。
- 退出窗口的具体天数进入试点部署配置档案；数值变化不改变本 ADR 的主体映射和权限语义。
- 所属主体可以处置和人工使用退出素材，同时避免未筛选原件自动污染组织官方库、公共池、知识库和自动检索上下文。
