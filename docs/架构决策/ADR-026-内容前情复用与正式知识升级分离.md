# ADR-026：内容前情复用与正式知识升级分离

- 状态：已接受
- 接受日期：2026-07-22
- 对应决策：D-011
- 上游边界：[ADR-010](ADR-010-全量生成留痕与机器自评回路.md)、[ADR-014](ADR-014-内容生成运行语义与评测分层.md)、[ADR-019](ADR-019-第一版25类最小语义模型与兼容扩展边界.md)、[ADR-025](ADR-025-首期禁止跨品牌跨租户内容学习与复用.md)

## 背景

笛语既要让账号和连续内容拥有真正的前情，又不能把每次聊天、每篇草稿或每条高表现内容自动写回长期记忆，更不能让系统生成内容反向证明商品、品牌、人物或经营事实。

D-031/ADR-014 已经冻结：日常互动和生成默认不进入长期记忆；用户主动保存或明确编排系列时才可跨任务复用；轻量私人协作偏好独立存在。本决策不重开记忆架构，只关闭“内容前情怎样继续使用”和“什么才能成为正式知识”之间的最后边界。

## 决策

### 1. 日常内容默认停留在当前任务

日常输入、系统增益和生成成品默认只在当前任务及其版本链中有效。生成、浏览、中间稿选择、导出、采用、高播放、评论、截图或机器评分，都不会单独把整篇内容、人物设定、商品结论或表达方式自动写入长期内容记忆或正式知识。

ADR-010 要求的生成与行为留痕继续存在，但留痕是历史和评估信号，不等于后续创作可以自动调用。

### 2. 用户明确要求复用时直接成立

以下任一行为都构成用户对内容前情的明确授权：

- 主动保存为以后可用的内容素材或前情；
- 明确纳入某个账号设定、主题系列或叙事系列；
- 自然语言要求“接着上一条”“延续之前那篇”或指定某一历史内容继续创作；
- 明确要求跳集、插集、改序、重设或沿用某一已存在的系列前提。

系统不要求用户为了同一意图再执行一次“保存”或填写表单。能在当前账号和合法作用域内唯一找到目标内容时，直接把该内容的必要前情作为本次任务优先参考；存在多个合理候选且选择会实质改变结果时，只作一次最小确认；历史内容已不可用时，只请用户提供标题、片段或必要摘要。该明确指令至少授权当前任务或用户指定系列使用，不自动扩大为账号全部任务的长期记忆。

### 3. “延续爆款”是优先参考，不是盲目复制

用户直接说明希望延续之前的一条爆款内容时，系统默认采纳这一创作意图，不要求截图、平台数据或外部核验。“爆款”在此只表示用户提供的表现判断，不自动成为平台事实、因果结论或正式知识。

系统应优先识别并延续原内容中仍然适用的受众问题、关系张力、观点、叙事未完线索、内容结构或视觉记忆点，再结合当前品牌、账号、平台、时间和制作条件重新编译。当前有效的品牌/账号身份、商品资料、价格或活动输入、平台规则及用户本次明确要求优先于旧稿；旧稿中的过期或冲突信息必须删除、替换或收窄。不得机械复制原稿、把偶然表现总结成确定方法，或在没有依据时宣称某个结构就是爆款原因。

### 4. 已发布历史只提供有范围的公开承接

已发布记录可以在原发布账号、平台及系列范围内支持“上一期”“上次我们聊过”等经过记录的真实承接。即使系统没有发布记录，用户明确要求按“上一期”或某一既定前情创作时，系统也可以把它作为用户本次给定的内容前提直接执行，但不得把该前提冒充为平台已验证的发布事实。两种情况都不会自动把整篇内容升级为账号现实履历、人物事实、商品事实或品牌正式知识。

用户可以要求跳过、插入、重排或重新指定某一集；系统不得用既有集号反向限制用户。虚构或演绎系列的前情可以继续留在相应系列中，但不能升级为真实人物经历、顾客记录或门店历史。

### 5. 内容前情与正式知识使用不同的确认动作

内容前情只服务后续表达与连续性；正式知识会成为多个任务的稳定依据，因此必须由相应责任来源明确提交或确认：

| 要保存的内容 | 正式归宿 | 最小成立条件 |
|---|---|---|
| 品牌长期选择、拒绝和关系姿态 | `BrandStrategyBaseline` | 对应品牌或企业范围的授权维护人确认版本 |
| 可重复使用的内容生产或制作方法 | `GenerationMethod` | 对应范围的授权维护人确认方法版本 |
| 必须遵守的表达、业务或评测规则 | `PolicyRule` | 对应责任组织确认规则及适用范围 |
| 商品、品牌、经营、平台等事实陈述 | `KnowledgeStatement` + 必要 `EvidenceRecord` | 对应责任来源另行提交或确认，并保留最小来源定位 |
| 账号或系列前情 | 账号配置或 `ContentSeries` | 用户主动保存、明确纳入或本次明确要求复用 |
| 采用、发布、表现、评论和机器判断 | `ActivityEvent` | 如实记录事件，不自动升级 |

品牌自己的正式商品资料或授权维护人对本组织事实的确认可以成为责任来源；本决策不要求外部第三方举证，也不建设逐条合同审批或事实裁判系统。系统只记录最小来源和适用范围，不替企业承担真实性责任。

### 6. 生成成品不能给自己作证

系统可以从内容、反馈和表现中提出候选观察，但生成成品、用户采用、高播放、评论和机器评分都不能独立证明其中的商品属性、人物履历、经营结果、品牌动机或通用专业结论。

候选表达若被授权维护人确认，可以按其真实含义进入品牌基线、方法或规则；事实型内容必须由责任来源另行提交或确认，不能通过“这篇内容已经发过”完成循环升级。

### 7. 私人协作偏好保持独立

系统自动维护的轻量私人协作偏好只回答“怎样更好地帮助当前用户”，例如偏好直接成稿、少追问或更口语化。它不保存具体生活事件、人物关系、内容前情、品牌规则或商品事实；用户仍可查看、纠正、删除、关闭或开启不读写偏好的临时会话。

## 最小实施与验收要求

D-011 关闭不新增核心对象。第一版复用 ADR-019 已有的 `BrandStrategyBaseline`、`GenerationMethod`、`PolicyRule`、`KnowledgeStatement`、`EvidenceRecord`、`ContentSeries`、内容版本和 `ActivityEvent`，并满足以下最小检查：

1. 一句普通日常输入未被用户保存或纳入系列时，不会在无关后续任务中自动出现；
2. 用户说“延续之前那条爆款”且历史目标唯一可识别时，系统直接形成续作，不要求先保存或上传数据截图；
3. 若存在多个可能目标，只追问一次“具体是哪一条”，不重新要求用户描述全部需求；
4. 高表现或机器高分内容在无人明确保存、确认知识版本时，仍只保留为事件与候选观察；
5. 已发布内容只在相应账号、平台和系列中支持真实公开承接；
6. 品牌方法、规则和事实分别进入正确对象，成品正文不能成为自身唯一来源；
7. 账号私人前情、品牌知识及其他品牌/租户内容不会互相串用。

## 明确不做

第一版不建设：

- 把全部聊天记录或全部生成正文塞入长期记忆；
- 私人记忆图谱、专职 Memory Agent、复杂审批状态机或知识晋升积分；
- 按播放量、采用率或模型评分自动修改品牌战略、方法、规则或事实知识；
- 为“延续爆款”强制要求平台截图、数据证明或重复保存动作；
- 用历史爆款机械套写新内容或把相关性冒充因果；
- 跨账号、跨品牌或跨租户自动共享内容前情。

## 影响

D-011 正式关闭。内容连续性继续保持自然、轻量和用户编排优先；正式知识升级使用既有对象和最小责任来源，不新增知识审批平台。

至此，进入最小生产物理架构和首批领域候选资产激活所需的 D-008、D-010、D-011 均已关闭。下一阶段应直接消费用户提供的 62 个文件，先为第一条软件纵向闭环选出最小知识切片并形成物理映射，不重造领域库，也不全量自动激活。
