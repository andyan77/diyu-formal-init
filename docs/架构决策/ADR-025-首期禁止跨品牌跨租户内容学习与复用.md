# ADR-025：首期禁止跨品牌跨租户内容学习与复用

- 状态：已接受
- 接受日期：2026-07-22
- 对应决策：D-010
- 上游边界：[ADR-010](ADR-010-全量生成留痕与机器自评回路.md)、[ADR-020](ADR-020-多品牌共享隔离与经营关系.md)、[ADR-024](ADR-024-首期租户强逻辑隔离与专属部署触发边界.md)
- 后续决策：D-011 已由 [ADR-026](ADR-026-内容前情复用与正式知识升级分离.md) 关闭

## 背景

笛语会记录生成版本和必要行为事件，并允许系统在已经授权的租户、品牌和账号范围内进行机器自评。但“系统能够分析”不等于“可以把甲品牌或甲企业的数据拿去帮助乙品牌或乙企业”。即使删除名称，内容、经营反馈、受众反应和派生特征仍可能包含品牌方法、商业秘密或可被反推的独特信息。

第一版没有真实证据证明跨品牌、跨租户汇总或训练带来的收益足以抵消隔离、授权、审计和再识别风险，因此不为假设中的数据飞轮提前建设共享学习能力。

## 决策

### 1. 第一版禁止三类跨范围使用

第一版不跨品牌、跨租户进行：

1. **内容复用**：复制、改写、检索或推荐其他品牌/租户的用户输入、生成内容、人工修改、素材、连续记忆、评论、截图或经营反馈；
2. **派生复用**：共享由上述数据形成的嵌入、标签、聚类、偏好、采用模式、表现规律、提示策略或其他可用于内容生产的派生特征；
3. **模型学习**：使用上述数据进行跨范围微调、训练、路由优化、提示优化、候选排序或共享内容策略升级。

“匿名”“去掉品牌名”“只看统计结果”不会自动改变该结论。只要结果来自特定品牌或租户的内容与经营数据，并被用于帮助其他品牌或租户，就属于本决策禁止的跨范围使用。

### 2. 跨品牌包括同一租户内的不同品牌

同一企业租户经营多个品牌时，品牌 A 的内容、反馈、连续记忆和派生规律不会自动用于品牌 B。企业共同知识只有在授权维护人明确建立并符合 ADR-020 的企业共同范围时才能供多个品牌使用：它可以是在覆盖品牌中原义成立的内容，也可以是明确以企业或集团为主体的共同事实，但不得承载单一品牌独有的声纹、商品故事、活动或连续记忆。它是经过确认的业务资产，不是系统从品牌数据中自动学习得出的共享结果。

用户明确发起的多品牌任务可以按 ADR-020 同时引用所选品牌及商品，这属于当前任务的显式上下文编译，不等于后台跨品牌学习。

### 3. 原作用域内机器自评继续成立

ADR-010 已确认的全量生成留痕和机器自评不被取消。系统只能处理已经获准记录的 `GenerationRun`、内容版本、`ActivityEvent` 及其受控引用，并继续继承原有租户、品牌、组织和账号范围；不得把本决策解释为新增读取私人原件、私人协作偏好或其他工作区正文的授权。系统可以在该合法范围内分析失败类型、规则命中、采用、修改、拒绝和自愿反馈，用于质量评估与候选改进。

机器自评不增加企业人员或系统运营人员的日常人工查看权，也不能把私人原件、其他工作区正文或反馈截图打开给总部。任何策略更新仍须形成候选版本并通过回归与人工发布，不允许生产系统自行改写规则。

### 4. 领域通用知识和必要技术遥测不属于跨租户内容学习

以下能力继续允许，但不得被包装成跨租户内容飞轮：

- 使用独立取得、来源明确并已经按领域流程确认的领域通用知识；
- 使用用户提供的首批领域候选资产，经项目复核激活后向适用租户提供；
- 处理不包含正文、素材、品牌语义、账号偏好或经营含义的最小技术遥测，用于计费、容量、安全、错误率和服务可靠性；
- 执行法律、安全和故障处置所必需的最小审计。

技术遥测不能反向用于内容风格、选题、账号人格、商品解释或经营策略优化；一旦包含这些业务语义，就重新落入本 ADR 的禁止范围。

### 5. 外部模型供应商不得成为绕行通道

向外部大模型发送数据不等于授权供应商训练或跨客户复用。第一版选型和合同必须确保笛语提交的输入、输出及派生数据不会在未经单独明确授权的情况下被模型供应商用于训练或改进共享模型；不能用供应商默认条款绕过本项目的品牌和租户边界。

### 6. 未来开放必须重新立项

未来只有真实收益已经成立，并获得涉及品牌或租户的明确授权时，才可以为具体用途重新立项。届时必须重新说明使用哪些数据、服务谁、能否撤回、如何防止还原、怎样审计以及如何证明收益；不得依靠本 ADR 预留空表、共享特征库、全局内容索引或默认勾选。

## 最小实施与验收要求

D-010 关闭的是使用边界，不证明系统已经实现。后续软件至少必须证明：

1. 机器分析任务继承原生成记录的租户、品牌、组织和账号作用域，且不读取私人原件或私人协作偏好；
2. 不存在跨品牌或跨租户的内容索引、向量近邻、特征库、训练集或候选策略池；
3. 在品牌 A 和诱饵品牌/租户 B 数据高度相似时，A 的内容结果与候选改进仍不调用 B 数据；
4. 删除或修改 B 的内容与派生记录，不改变 A 的承重输出；
5. 技术遥测与内容/经营数据分开，不能从技术报表还原正文、品牌方法或账号偏好；
6. 外部模型调用的合同与配置符合“不用于未授权共享训练”的要求。

## 明确不做

第一版不建设：

- 全租户内容语料池、跨品牌向量索引或共享特征库；
- “行业最佳文案”“高表现门店规律”等由租户数据自动汇总的公共能力；
- 基于多租户内容与反馈的模型微调、提示自动优化或全局排序器；
- 为未来跨租户学习预建授权中心、匿名化流水线、贡献积分或收益分配；
- 把普通技术遥测扩张为内容与经营数据采集。

## 影响

D-010 正式关闭。第一版只建设作用域内机器自评和必要事件留痕，不建设跨品牌、跨租户内容飞轮。共享领域知识继续来自独立领域资产和明确复核，不从租户生成内容中自动反向提炼。

本 ADR 不阻止未来另行论证，但未来方案必须作为新的版本化决策提出，不能把“匿名”或“平台优化”当成既有授权。
