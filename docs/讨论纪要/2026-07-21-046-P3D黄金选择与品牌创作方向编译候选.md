## 一、正式名称与通俗解释

**正式名称：D-014 运行化补充——品牌内容战略基线与动态创作方向编译。**

通俗说，就是：

> **系统不先从“幽默、治愈、励志、故事、口播”里选一种风格套上去，而要先理解这个品牌长期服务谁、在具体生活矛盾中保护什么和拒绝利用什么，再结合当前发布账号从哪里观察、目标受众此刻为什么值得看、本次任务要交付什么，在当前上下文下择优形成一条创作方向。**

这不是新增 D-033。D-014 已经原则确认“通用创作策略、品牌表达档案、动态趋势与任务运行态分层治理”，本轮只把它变成可被 P1—P5 和黄金任务消费的最小运行合同，避免重复决策和流程过度工程化。

## 二、必须纠正的一个误区

“不同品牌定位和受众对应不同内容风格”只说对了一半。它们不会唯一决定一篇内容必须是幽默、文艺、热血或故事化，而是共同决定：

- 哪些观察和判断属于这个品牌；
- 哪些表达虽然可能有流量，但品牌不采用；
- 这个品牌怎样与受众相处；
- 哪些创作路线在当前账号和任务中成立，哪些不成立。

最终表现还必须同时受到发布账号、任务种子、P1—P5 主产品合同、当前受众情境、连续前情、平台和制作条件影响。因此，第一版不得建设“设计师品牌 = 文艺、运动品牌 = 热血、加盟品牌 = 接地气”之类的静态映射，也不得建设“品牌类型 × 产品 × 平台 × 情绪 × 拍法”的巨大矩阵。那只是把通用 AI 模板按租户重新分组，最终仍会同质化。

## 三、系统只需要四层解耦信息

### 3.1 品牌稳定层：品牌内容战略基线

一份由企业确认、可版本化的品牌基线，第一版只回答：

1. **服务谁、面对什么具体矛盾**：不是只写年龄和消费力，而是目标人群反复面对的生活、身体、审美和选择张力；
2. **稳定选择与拒绝**：品牌通常保护什么、可以牺牲什么、不利用什么焦虑、不会把什么问题简单化；
3. **观察世界的方式**：品牌更容易看见身体与动作、角色切换、结构与秩序、日常毛边、商品取舍或其他具体关系；
4. **与受众的关系姿态**：平等同行、专业帮助、审美启发或其他被企业确认的待人方式；
5. **表达幅度而非固定语气**：基准气质、允许变化的范围及明确不能漂移到的表达；
6. **少量正反例及理由**：保存“为什么属于/不属于我们”，不要求模型照抄句式、结构和镜头；
7. **版本与生效时间**：新配置影响后续任务，不倒改历史成品。

“设计审美/生活方式、全国大众零售、运动与功能、加盟网络/本地门店、批发与潮流货、亲子生活、集合策展/买手”等行业原型只能作为品牌入驻的预填和提问工具，不能直接成为品牌最终人格。原型允许组合或选择“都不像”，不能被建成互斥品牌分类树。江南布衣、安踏、斐乐等真实品牌名称只可在讨论或访谈中帮助理解，不作为正式模板标签，更不能暗示系统模仿其表达。企业确认后的自身基线始终优先。

#### 3.1.1 租户的第一个内容业务里程碑

租户完成开通并具备管理员账号后，第一个**内容业务里程碑**不是填写空白战略问卷，也不是先批量创建 IP 账号，而是用最低负担形成并确认一份最低可用的品牌或企业/店铺内容战略基线：

1. **行业原型预填**：系统结合领域通用知识和企业已经提供的品牌资料，先给出一个或两个最接近的可组合原型，并预填一版可能的受众矛盾、稳定取舍、关系姿态和表达边界；原型只是起点，不是结论；
2. **少量具体选择与对照**：企业只需围绕真实内容例子作“像/不像、用/不用、A/B、两者都可、两者都不对”等轻判断。对照必须暴露品牌怎样处理身材焦虑、商品缺点、未解决生活、用户自主权和热门议题等具体取舍，不能只问“幽默还是高级”；
3. **简短对话补齐差异**：系统只追问会实质改变内容方向、又无法从资料和选择中取得的少量问题；企业可以回答“不知道”，系统继续给候选表达供其纠正，不把入驻做成品牌咨询长审计；
4. **系统生成人话版完整草案**：系统把预填、选择和简短访谈收敛为一份普通服装老板或市场负责人能够读懂的基线草案，并可附少量同种子极短方向预览帮助判断；企业看到的是可修改成品，不是字段、标签、评分或模型参数；
5. **企业修改并确认版本**：企业只需判断“这版准不准”，改掉不准确的几句话并确认生效版本。这里的确认是业务含义确认，不是签字、法律审批或新增复杂审核流。

未经确认的草案、系统推断和试生成可以用于继续对话、资料整理和预览，但不能静默成为正式品牌或企业/店铺人格，也不能据此批量激活正式 IP 账号。确认基线后，系统再依据真实组织提出最小账号/角色方案、首批账号角色画像及账号冷启动内容计划：

```text
品牌或企业/店铺内容战略基线
→ 按真实组织形成的最小账号/角色方案
→ 首批账号的观察资格、服务对象与表达职责
→ 账号冷启动选题和生产计划
```

该顺序取代此前“品牌入驻先规划 IP 矩阵”的旧顺序；行业角色模板仍用于第二步账号/角色规划。真实存在多组织、多层级或多账号协同时可以形成矩阵，单店租户不需要虚构总部或区域。一个企业存在多个商品品牌码时，不因此自动为每个品牌码生成独立人格：只有企业确认该品牌独立的 D-014 内容战略基线后才取得独立内容身份，其余先继承企业共同表达；没有自有品牌的集合店以自身企业/店铺身份表达并显式引用上游品牌。该极简门槛已由 [ADR-016](../架构决策/ADR-016-ERP品牌码与独立内容品牌极简门槛.md) 和 [ADR-023](../架构决策/ADR-023-组织型IP参考母版与租户拓扑非强制.md) 关闭。

本轮只冻结上述交互结果和先后关系，不冻结究竟分几屏、使用向导还是对话、A/B 题数、按钮名称、语音或文字输入以及草案的物理存储方式。

“折线之间”已经确认进一步补全为一份展示上述入驻过程、确认结果和运行成品的完整合成演示品牌；但它是一个原创完成样板，不是行业原型，也不能被新品牌一键复制。具体定位、必要信息补全规则与边界见[讨论纪要 047 确认稿](2026-07-21-047-折线之间完整演示品牌定位与边界候选.md)。

### 3.2 账号稳定层：IP 矩阵与账号表达画像

账号层只回答：

- 它位于总部、区域还是终端；
- 代表哪个组织、服务哪类受众；
- 它合法的观察位置、专业资格和生活进入方式是什么；
- 它通常优先谈什么、可以怎样说、不能冒充谁。

同一品牌的总部设计/商品专家、区域经营者和终端店长可以共享品牌基线，但不能因此说成同一种人。品牌决定共同内核，账号决定“谁从哪里说”。

### 3.3 当前任务层：本篇为什么值得做

当前任务组合：

- 用户种子或系统发现的内容机会；
- 一个主要 P1—P5 产品合同；
- 本次具体目标受众及其所处情境；
- 用户明确的题材、系列、商业目的和事实前提；
- 当前账号近期已经确认或发布的紧凑摘要，用于避免重复同一种隐喻、反转和收尾。

目标受众不是 CRM 顾客画像，也不要求记录个人隐私。它是非个人化的内容受众模型。更重要的是，品牌与受众关系有时会决定一个种子是否适合成为该品牌的正式内容；不适合时应继续友好讨论或换路由，不能硬做品牌化包装。

### 3.4 动态表现层：平台、趋势与制作条件

平台、趋势、时长、口播/演示/故事/图文及一人或团队制作只决定“这一次怎样呈现”。趋势必须带平台、版本、生效和失效时间；它可以改变开场、节奏、镜头和互动方式，不能反向改写品牌长期价值、账号身份或事实边界。具体媒体合同继续归 D-032。

## 四、一次运行时怎样择优形成方向

运行时只进行一次合成：

```text
一个主要 P1—P5 产品合同
+ 当前版本品牌内容战略基线
+ 发布账号及内容角色
+ 本次目标受众情境
+ 任务种子、事实边界与可选连续前情
+ 平台、趋势和制作条件
→ 本篇择优创作方向
→ 一个完整成品
```

“本篇创作方向”只需是一份紧凑的内部运行摘要：

```yaml
creative_direction:
  mainline: 本篇最值得表达的一项具体矛盾、观察或变化
  brand_account_basis: 为什么这个品牌体系中的这个账号会这样看和待人
  audience_payoff: 目标受众看完新增什么理解、感受、判断或关系认识
  expression_choice: 当前条件下最合适的表达形态、节奏与情绪幅度
```

它不展示给普通用户，不是模型隐藏推理，不要求成为新本体类或四个固定数据库字段；D-016 再决定它是临时 JSON、版本摘要还是其他实现。系统最终仍只向用户交付完整成品。

日常 `focused` 任务默认在当前上下文下择优形成一个方向并做快速必要自检；这不是不可更改的客观唯一答案，用户可以随时要求换角度、换情绪或换制作形式。只有用户明确探索、黄金任务或确有多个承重方向时，才生成 2—3 个完整路线比较。不建设“品牌风格 Agent”，也不让每次生成都跑多路反事实。

## 五、为什么当前两版胜出

在“折线之间”当前冻结基线下：

- 服务 30—45 岁、在工作、家庭与个人生活间切换的城市女性；
- 不把人压回身份和生活的标准答案；
- 允许矛盾、未知和没有完成的情绪存在；
- 终端店长从整理门店和参与普通生活的位置发声。

因此：

1. **包 1 选择版本三《今天不把它扶正》**：店长“归位”的合法观察动作，与品牌不急于把差异纠正成错误的待人方式同源；它比通用感官留白更有账号承重，又比“脑内开会”更不容易让形式装置压过真实。
2. **包 2 选择版本一《门口那份早餐，先别替它解释》**：它让早餐、怨气和没有道歉同时存在，最符合品牌不替受众修成标准答案的关系姿态；若强行使用盘点隐喻，虽然更机灵，却可能让关系痛感退化为段子。

这不是说两个版本在所有品牌中绝对最好。若换成强调“把感受、事实和下一步分清”的品牌，同一咖啡种子可能更适合版本二式的清晰结构；若换成强调行动和边界表达的品牌，家庭种子也可能更适合版本三式的立场路线。若品牌目标人群和账号生活位置根本不承接家庭议题，则包 2 不应被强制生产为该品牌的 P3。

## 六、第一版怎样避免长期同质化

系统采用“稳定内核、动态表达”：

- 稳定的是品牌选择与拒绝、观察方式、受众关系、账号位置和事实边界；
- 动态的是本篇矛盾、情绪温度、幽默与否、是否给结论、口播/演示/故事/视觉形式、节奏和声音；
- 近期内容只用于防止反复使用同一种“语义动作”，例如每篇都由小事升华、每篇都自我和解、每篇都以“先放着”收尾；
- 用户采用、高播放或系统评分只能形成待观察信号，不能自动改写品牌战略或升级为正式表达规则。品牌基线的变化必须由企业明确确认。

## 七、最小验收，不新增日常负担

只在黄金任务、行为实质变化的版本回归和必要诊断中做以下检查：

1. 同种子、同账号、同受众，只换品牌稳定基线，核心矛盾、判断、关系姿态或收束必须实质变化；只换品牌名、形容词或语气即失败；
2. 同品牌、同种子，只换账号，合法观察位置和人格贡献必须变化；
3. 同品牌、同账号，只换目标受众情境，观看回报、进入角度或是否值得生成必须变化；
4. 只换平台或制作方式时，表现可以变化，但核心意义、事实和品牌/账号身份不得漂移；
5. 两个品牌版本都必须是好内容，不能故意把对照品牌写差来证明差异。

日常生成不执行这些完整对照，只做当前上下文是否真正参与、是否可被任意品牌/账号替换的快速自检。

## 八、与现有决策的归属

- **D-014**：确认品牌内容战略基线与动态创作方向编译的业务语义；
- **D-016**：以后决定现有 `BrandCore`、`AudienceProfile`、`ExpressionProfile`、`ContentRole`、`PublishingAccount`、`PolicyRule` 等逻辑含义如何落库或投影；当前不新增核心类别；
- **D-021（已关闭）**：只有 ERP 品牌码时沿用企业表达；企业确认该品牌独立的 D-014 内容战略基线后才取得独立内容身份，见 ADR-016；
- **D-032**：决定平台趋势和媒体表现怎样适配，不负责定义品牌稳定内核；
- **P3-D**：用本轮两个黄金包证明该机制必要，不因此新增第三个黄金包或重开 P3-A—C。

第一版明确不建设静态风格矩阵、空白长问卷、每品牌独立 Agent/模型微调、品牌风格知识图谱、永久“品牌浓度分”、自动学习品牌人格、复杂配置审批流或用户可见的运行报告。

## 九、正式决策文本（已确认）

> **笛语采用“品牌内容战略基线 + 动态创作方向编译”。品牌租户以版本化方式确认其目标人群关系、稳定选择与拒绝、观察方式、关系姿态、表达幅度、账号矩阵及少量正反例；系统在每次任务中结合当前账号、受众情境、任务种子、主要内容产品合同、连续前情、平台和制作条件，在当前上下文下择优形成一条创作方向并交付一个完整成品。风格、结构、情绪和形式是动态结果，不建立静态风格矩阵；高表现和系统推断不能自动改写品牌战略。**

本条已经确认，并已由 P3-D 的包 1 换品牌、包 2 换账号检查完成最小验证。P3 已关闭，不继续扩充 D-014、架构或样例；唯一下一动作转为 P4-A。
