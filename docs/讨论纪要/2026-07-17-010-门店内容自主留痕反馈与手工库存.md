# 2026-07-17 门店内容自主、留痕反馈与手工库存

> **后续裁决（2026-07-18）**：本纪要第六、七节记录的是当时方案，已被用户最新裁决取代。截图只能自愿上传，未上传不得阻断任何生成或后续任务；不存在反馈债务、强制门或例外放行。当前基线见 [ADR-011](../架构决策/ADR-011-发布反馈自愿提交与禁止生成门控.md)。其余内容自主、留痕和手工库存结论不因此失效。

## 一、本轮可以直接确认的业务规则

1. **加盟终端内容自主**：加盟店生成、编辑和对外发布内容，无需区域代理或品牌总部人工审批。
2. **总部/区域是赋能者，不是必经审批人**：两级通过知识库、官方商品事实、品牌规则、活动素材、内容策略和示例向门店交付能力。
3. **两条业务可见规则成立**：
   - 总部已发布的品牌规则、商品资料和活动素材，被授权门店可看、区域可看、总部管理；
   - 门店原始照片、语音、观察记录和未完成草稿，只有本店可看，区域和总部默认不可看。
4. **所有层级生成行为留痕**：总部、区域和门店工作区均记录实际操作人的登录身份；这里的人员登录身份与自媒体发布账号是两个对象。不因门店自主发布而取消事后可追溯性。
5. **无 ERP 也可使用库存搭配/陈列建议**：本店被授权账号可手工输入本店库存，但必须显式标记为门店自报快照。

## 二、必须纠偏：系统框架不能只等于平台红线

平台规则是必要约束，但不是全部约束。例如：

- 门店写错面料成分、功能或可售库存，即使未触发平台审核，仍是不可接受的虚假事实；
- 使用未授权顾客人脸、声音或他人图片，即使平台允许上传，也不代表系统可以放行；
- 门店自行承诺未被品牌授权的价格、活动或售后政策，平台不一定拦截，但会造成品牌和经营风险。

因此系统约束分为：

| 约束类型 | 处理方式 |
|---|---|
| 法律、隐私、知识产权、人物/媒体授权 | 硬阻断或要求补齐证据 |
| 商品、价格、库存、门店事实和品牌授权范围 | 按 A 权威新鲜、B 授权自报、C 过期/无来源/冲突分级；只有 A 可作对外确定性硬事实，B 是本任务不可篡改但需复核的自报输入，C 只能参考/降级 |
| 各自媒体平台规则 | 带平台、版本、生效时间维护；系统检查不代表平台保证放行 |
| 品牌风格、表达偏好和经营建议 | 默认提示、解释和留痕；除非涉及上述硬约束，不变成总部人工审批 |

正式表述应为：

> 加盟店在法律/隐私、知识产权与授权、可验证事实、品牌授权范围和各平台规则等系统硬约束内，自主生成、编辑和发布内容，无需区域或总部人工审批。

## 三、“完整留痕”的可实施定义

每次生成或重生成均建立审计事件，至少记录：

- 谁在什么工作区、什么时间发起；
- 任务类型、目标平台与所用发布账号；
- 生成结果快照、版本关系与用户修改；
- 使用的知识、证据、商品、品牌规则和平台规则版本；
- 硬约束检查、警告、用户确认和例外处理；
- 是否被选中定稿、导出、标记发布、回传反馈或废弃。

总部**审计管理员**默认只查看上述审计元数据和不可指向个人的聚合结果，用于事后追溯，不代表发布前审批；不因此获得私有原件、原始提示、生成正文、反馈截图或未脱敏个人信息的人工查看权。

“完整留痕”不等于永久保存每个敏感原件，也不保存模型隐藏推理；原文件仍按授权、撤回、保留期和删除规则处理。

## 四、无 ERP 的门店手工库存

手工库存快照至少记录：

- 门店；
- 来源类型 = 门店手工填报；
- 截至时间、填报人和新鲜度/有效期；
- 覆盖范围：全店、某品类或指定商品；
- 盘点质量：准确计数或大致估计；
- SKU/条码、数量及数量口径；
- 可售数量和是否有可用于陈列的实物/陈列样；
- 门店确认“这是该时点的自报情况”。

旧快照不被改写；重新盘点或确认无变化均生成新版本。输出上限按输入质量决定：

| 输入情况 | 系统最高可给的结果 |
|---|---|
| 有效期内、准确盘点、SKU/空间/任务输入完整 | “基于门店自报库存的执行候选”；执行前仍需门店确认库存未变 |
| 只覆盖部分商品 | 只对该范围给建议，不声称完成全店优化 |
| 估算、缺数量或空间信息不足 | 概念/模拟建议，不给精确数量和位置 |
| 过期、SKU 无法解析或数量冲突 | 停止库存型推荐，只提供通用搭配知识 |

手工库存属于动态经营数据，不进入长期知识库或向量索引。未来若接入 ERP/POS，继续使用同一快照合同，只更换来源、映射和校验等级。D-023 后续已按“外部系统可选接入、供应商无关快照与降级合同”关闭，见 [ADR-007](../架构决策/ADR-007-外部经营系统可选接入与库存事实降级.md)。

本轮取消的是**内容发布前的区域/总部审批**，不是取消陈列执行确认。基于手工库存产生的陈列结果最高称为“执行候选”，至少由本店在实际执行前确认库存、空间和安全条件未变。

## 五、搭配/陈列不能用一层四选一分类

“销售搭配、橱窗模特、中岛、挂杆”应拆成三个维度：

| 维度 | 例子 | 含义 |
|---|---|---|
| 任务目的 | 销售连带/顾客穿搭、视觉陈列 | 为顾客/导购推荐商品组合，还是为门店展示空间安排商品 |
| 陈列区域 | 橱窗、入口、卖场中部、墙面 | 方案发生在门店哪里 |
| 陈列载体 | 模特、中岛台/架、挂杆、层板 | 商品放在什么实体设施上 |

因此：

- 销售搭配 = 销售连带/穿搭任务，不必绑定陈列空间；
- 橱窗模特搭配 = 视觉陈列 + 橱窗区域 + 模特载体；
- 中岛搭配 = 视觉陈列 + 卖场中部 + 中岛台/架；
- 挂杆搭配 = 视觉陈列 + 指定区域 + 挂杆载体。

第一版可支持这些维度和常用值，但首期验收仍只选一个具体组合完成闭环，避免一次做完所有店内陈列类型。

## 六、用户要求的“上一条发布截图”闭环

用户的目标是强制内容生产者把真实发布结果、平台数据和评论带回系统，而不是无限生成却不复盘。该目标合理，但原始“每生成一次都检查上一条截图”不可直接实施：

- 一篇内容可能重生成和改写多次，不应形成多篇发布债务；
- 用户可能批量创作后分日发布，全局锁会破坏排期；
- 同一内容可能发布到多个平台/账号，“上一条”不能全局混算；
- 首篇、未发布、已废弃、零评论、评论关闭、平台延迟或内容下架都可能造成死锁；
- 评论截图可含昵称、头像、账号和个人观点，不能不经脱敏地进入共享数据。

## 七、D-025 当时结论（现已被 2026-07-18 裁决取代）

功能正式命名为：**发布反馈闭环门**。

1. 租户可关闭或开启；关闭后不限制新任务，但不删除既有记录。
2. 只对被配置为**实际运营自媒体发布账号**的工作区生效。总部/区域只生产知识、模板或活动素材而不直接运营发布账号时，不被门卡锁定；如总部自己运营官方账号，则同样适用。
3. “上一条”按**工作区 + 平台 + 发布账号**分别计算，不跨账号或跨平台串账。
4. 只有内容被“选中定稿”、“导出”并“标记已发布”后，才产生反馈义务。同一任务内的改写、重生成和多版本不受限。
5. 开启时，上一条已发布内容必须上传反馈证据或取得有理由且留痕的例外放行，才能开启下一条发布任务。默认允许同时未闭环发布任务数为 1，批量排期时可配置提高。
6. 首篇、未发布/已废弃、零评论/评论关闭、平台数据延迟、已下架等必须支持显式状态或管理员例外，放行人、理由和时间全部留痕。
7. 反馈证据至少关联平台、发布账号、对应内容、发布时间和数据截取时间；截图中的用户昵称、头像和账号等默认脱敏。
8. 截图默认标记为**用户自报证据**，可进行重复图和明显篡改提示，但不声称已由平台验证真实；后续如接入平台接口，再增加“平台已验证”等级。
9. 播放、点赞、评论或成交数据回传后只进入候选反馈区，不会自动升级为品牌真理或通用知识。

推荐状态流：

`Generated/Edited 生成/编辑 → Selected 选中定稿 → Exported 已导出 → Published 已发布待回传 → FeedbackReported 已回传（自报） / Exempted 例外放行 → Closed 闭环`
