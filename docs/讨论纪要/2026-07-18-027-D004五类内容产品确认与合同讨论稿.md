# D-004 五类内容产品确认与合同讨论稿

## 一、本轮用户确认

用户正式接受以下五类内容产品作为 D-004 基线，并同意据此输出逐类内容产品合同讨论稿：

| 编号 | 内容产品 | 核心价值 |
|---|---|---|
| P1 | 真实穿衣问题与购买决策 | 把用户的身体、场景和顾虑转化为可执行的选择判断，降低试错成本与购买风险 |
| P2 | 商品真相与专业取舍 | 把商品从卖点宣传还原为真实取舍，讲清它解决什么、牺牲什么、适合谁、何时不该选 |
| P3 | 品牌/账号观点与生活叙事 | 把日常生活种子转化为稳定的角色观点与连续叙事，形成可识别的人格和长期关系 |
| P4 | 一线问题回应与本地生活 | 把评论、门店观察和本地日常转化为持续回应，建立近场信任、熟悉感与到店理由 |
| P5 | 造型灵感与视觉叙事 | 把商品组合成可识别的审美世界，制造穿着想象、视觉欲望与品牌记忆 |

最短业务区分固定为：

> **P1 帮助选择，P2 解释商品，P3 建立人格，P4 经营关系，P5 创造欲望。**

正式决策见 [ADR-013](../架构决策/ADR-013-五类内容产品正式基线.md)。

## 二、必要拆分：D-004 不等于平台与媒体形式已经确认

此前 D-004 同时夹带了“内容产品、媒体形式和平台”三个问题。用户本轮只明确确认五类内容产品，因此不能把此前专家提出的“抖音 + 小红书”或“短视频 + 图文”冒充共同结论。

处理结果：

- D-004 只关闭五类稳定内容产品范围；
- 首期平台、媒体成品、时长/图片组和跨平台验收拆为 D-032；
- 抖音短视频脚本/分镜与小红书图文/视觉规格仍是当前建议，不是已确认范围；
- D-030 的素材敏感信息清理支持矩阵改为依赖 D-032，而不是依赖已关闭的 D-004。

## 三、逐类合同讨论稿的共同骨架

每类合同均回答：

1. 服务谁、解决什么稳定问题；
2. 接受哪些一句话种子，哪些任务必须改路由；
3. 系统必须取得哪些品牌、账号、受众、商品、事实和素材上下文；
4. LLM 可以增加哪些结构、专业解释、叙事和视觉演绎；
5. 哪些人物、商品、价格、库存、活动和效果事实绝不能猜；
6. 怎样形成不可替换的 `brand_relevance_path`；
7. 商品承担 `none/scene/evidence/solution` 中哪种作用，以及内容独立承担何种商业意图；
8. 输出哪些内部简报、脚本、图文或视觉规格；
9. 采用与发布后怎样维持账号连续性，哪些内容不得回写为事实；
10. 怎样验收，以及哪些反例直接判定失败。

每个成品必须只有一个主产品，允许最多两个辅助产品。主产品由“最终按哪种稳定价值验收”决定，而不是由是否出现商品、图片或情绪决定。

## 四、五类之间的硬边界

- P1 从人的穿着任务与现实约束出发，最终必须给出有条件的选择、反选和替代。
- P2 从真实商品或设计选择出发，必须绑定实际商品/系列及当前事实，不能用通用面料科普冒充品牌商品真相。
- P3 回答“这个品牌/账号怎样理解生活”，商品可以为空，但品牌关联和账号连续人格不可为空。
- P4 回答“这个门店/区域怎样回应身边的人和当下”，必须有真实近场锚点，不建立顾客实体、顾客画像或 CRM。
- P5 把真实品牌商品组织成可识别的视觉世界，创建简报前至少绑定一个真实品牌商品；没有真实商品时不得创建 P5 候选或成品，应改路由 P3 或请求用户选择商品。

P5 的 `ContentStylingSpec` 只服务外部内容表达。任务一旦要求门店空间版本、SKU 陈列数量、库存约束、橱窗/中岛/挂杆执行、审批、巡检或整改，就必须停止 P5 路由并进入 `display-merchandising`，输出 `DisplayArtifact`。

## 五、第一组黄金任务分配

为避免 P5 正式进入基线后仍缺乏独立验收，12 个黄金任务分配为：

- P1 三题：身体/场合选择、两个商品取舍、P1 与 P5 联动；
- P2 两题：设计/材料真实取舍、价格价值与“不该买”边界；
- P3 三题：生活种子品牌化、普通经历形成观点、采用/发布连续性；
- P4 两题：评论/一线问题回应、带时效的本地生活；
- P5 两题：真实商品组合的可复现视觉故事、总部—区域—门店矩阵差异压力测试。

平台变体必须等 D-032 确认后再写入黄金任务验收，不能先写死当前平台规格。
