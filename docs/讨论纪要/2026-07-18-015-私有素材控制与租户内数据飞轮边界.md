# 2026-07-18 私有素材控制与租户内数据飞轮边界

> **后续裁决（2026-07-18）**：用户已把“总部默认数据飞轮权”纠正为“系统机器自评回路”。全部内容生产版本都可由系统机器留痕和分析，但该回路不给企业人员跨工作区查看具体正文、原始提示、反馈截图或私人素材的权利。D-028 已关闭，当前基线见 [ADR-010](../架构决策/ADR-010-全量生成留痕与机器自评回路.md)。

> **退出规则已正式修订（2026-07-18）**：用户后来把“私人”限定为所属经营主体授权期间，并撤回本纪要关于“店员离职后店主仍不取得平台控制”的结论。授权期内隔离仍有效；退出窗口届满后由上传时记录的实际所属经营主体接管，正式规则见 [ADR-012](../架构决策/ADR-012-授权期个人隔离与退出后所属主体接管.md)。

## 一、用户提出的核心观点

1. 品牌总部和区域代理默认都不能访问门店私有素材区；除素材账号使用者外，任何人都不能查看，使用者可以编辑、删除。
2. 总部管理员可以禁用某个账号或某一区域账号，但该管理权限不意味着总部取得这些账号的私有素材。
3. 总部默认对系统生产的全部自媒体内容拥有数据蒸馏和数据飞轮权。

## 二、直接接受的业务边界

- 总部和区域代理不因上下级关系、系统采购关系或管理员身份自动获得门店私有原始照片、视频、语音、观察记录和未完成草稿的查看权。
- 总部暂停或终止渠道成员的租户使用资格，只改变其后续系统准入，不改变私有素材的控制关系，也不把素材转移给总部。
- “总部能否通过机器学习租户内生成内容”与“总部人员能否查看原始素材”是两项不同权限，可以分别控制。

## 三、必须纠正：“账号私有原件”不能等同于“全部业务资产”

用户要求的个人私有区可以成立，但“账号”必须限定为一人一号的自然人登录身份，不能包括可由多人运营的自媒体 `PublishingAccount`、组织岗位账号或系统服务身份；个人私有原件、组织业务素材和系统生成内容也不能混成同一个资产层。

第一版只保留两条彼此隔离的素材入口，不提供二者之间的转换：

| 入口 | 控制者 | 人工可见范围 | 用途 |
|---|---|---|---|
| 个人私有原始素材箱 `creator_private` | 自然人账号使用者 | 仅本人；总部、区域、同店其他成员及普通管理员均不可读 | 个人采集、整理和发起生成所用原件 |
| 组织官方素材工作区 `workspace` | 品牌企业、区域代理或加盟店经营主体 | 该经营主体授权的成员 | 由组织授权管理员直接维护的官方业务素材；不从个人私有区转入 |

云端 Agent 在用户主动发起任务时必须由受控服务身份读取所选原件，否则无法完成生成；因此“谁都不能查看”准确指其他业务账号和普通管理员没有人工读取权限，不能虚假承诺连系统处理服务都无法接触数据。

私有素材箱不等于私人工作成果区。总部员工可以拥有仅本人可见的原始素材箱，但其在总部岗位任务中形成的 `ContentBrief`、`ContentItem`、人工修改版本和审计记录仍进入总部组织业务工作区，由品牌企业管理，D-026 不变。代理和门店的系统生成业务内容分别进入其经营主体工作区。

用户当时明确取消“把选中版本发布到组织工作区”的动作。授权有效期间，第一版个人隔离素材只能由本人用于自己的生成任务，不支持转组织、跨账号共享、跨层级复用或讨论。本段原有“店员离职后店主也不取得平台控制”已被后续用户表述撤回；账号退出后的处置以 ADR-012 为准。

未来若确有素材交流需求，可单独立项 BBS/讨论区；当前只记入路线图，不预建任何表、接口、菜单、权限或空后台任务。

## 四、编辑和删除必须与完整留痕兼容

自然人使用者可以编辑其私有原件的说明和标签、创建新版本，也可以删除私有原文件并立即停止未来检索和生成使用。组织官方素材是另一入口，由组织管理员单独维护，不与个人原件联动。

私有原件一旦被生成任务、定稿内容或发布记录引用，删除原件后系统仍保留最小不可变的素材编号、哈希、引用关系、操作者、时间和撤回/删除状态；生成内容、导出、发布和审计记录不级联删除。不能允许任何账号借删除原件抹掉已经发生的责任链。

## 五、“数据飞轮权”必须拆成五种不同能力

| 能力 | 当前建议 |
|---|---|
| 总部人员查看门店私有原件 | 默认禁止 |
| 总部取得、复制或改编某一份私有素材/具体内容 | 第一版禁止；不提供授权共享或转组织流程 |
| 租户内部机器处理系统生成的 `ContentItem`、人工改动和 `EvaluationSignal` | 同一租户、同一品牌合作网络内默认允许，但不因此向总部人员展示私有原件、原始提示、生成正文或反馈截图 |
| 将蒸馏结果用于策略或事实知识 | 去原文结果只能先进入候选区；候选策略经品牌业务确认可成为 `PolicyRule`，但生成内容绝不能直接成为事实型 `KnowledgeStatement`，事实必须另有独立来源和证据 |
| 跨品牌、跨租户训练或复用 | 默认禁止；仍由 D-010 单独决定 |

以下是形成 ADR-010 前的分析过程；其中“总部数据飞轮”的主体表述已被后续裁决替代。第一版建议进一步限定：

1. 系统可对全部生成任务做运行质量分析，例如失败类型、采用率、修改幅度和规则命中；单纯未采用只能记为结果未知，只有选中、明确拒绝、人工修改、定稿、发布和自愿反馈等明确事件才能形成相应强度的行为信号，不能把未采用草稿自动当作优质或失败经验。
2. 第一版不做模型参数微调。飞轮只形成去原文的内容类型、结构/风格标签、采用/修改/拒绝信号、表现区间、聚合指标和候选策略；不得输出可识别顾客、单个账号、门店私密故事，或可重建原始素材/具体原文的结果。
3. “允许机器蒸馏”不等于“允许总部员工打开原文”，也不等于“允许总部复制某条门店内容”。人工查看权、机器处理权和具体复用权分开授权与审计。
4. 该租户内默认处理政策应在代理/加盟店账号被邀请使用系统时明确展示，不要求每次生成重复确认；系统只执行已配置政策并留痕，不替各方裁定最终法律权利。

## 六、本轮只请求确认的裁决

用户已裁决 D-003 的私有素材访问和复用部分：

> **每个自然人登录账号可以拥有仅本人可见的私有原始素材箱；总部、区域、同店其他成员和普通管理员均无人工查看权，总部暂停账号或区域使用资格也不取得素材。私有区不等于私人工作成果区：总部岗位任务和系统生成业务内容仍进入总部组织工作区。第一版私人素材只能由本人用于自己的生成任务，不提供转组织、共享、上提或跨层级复用。使用者可以编辑、删除私有原件并停止未来使用，但不能级联删除已经发生的生成、导出、发布和最小审计链。**

授权期个人隔离部分形成 [ADR-008](../架构决策/ADR-008-个人私有原始素材隔离与首期禁止共享.md)；账号停用后的退出窗口与所属主体接管已由 D-003/[ADR-012](../架构决策/ADR-012-授权期个人隔离与退出后所属主体接管.md) 正式关闭。D-028 后续已关闭并形成 [ADR-010](../架构决策/ADR-010-全量生成留痕与机器自评回路.md)。
