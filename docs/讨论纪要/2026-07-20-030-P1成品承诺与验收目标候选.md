- 对照清单：[P1—P5 可验收合同唯一 TODO 清单](../内容产品/P1-P5可验收合同唯一TODO清单.md)

## 一、用户本轮纠偏

1. P1 不应要求用户先把穿衣或购买问题表达清楚。系统应利用领域通用知识、品牌专属知识、商品知识和 LLM 理解，从一句模糊输入中发现可能成立的典型问题、回应或建议。
2. A 知识解释、B 条件化决策、C 可执行行动、D 传播竞争和 E 双层结果的价值都可以吸收；不应因为各自存在缺点就删掉相应能力。

## 二、必要纠偏：知识是发现与求解依据，不是人的事实

上述方向可以实现，但不能把“品牌商品知识”写成 P1 唯一锚点。否则系统会从“品牌有什么货”反推“受众应该有什么问题”，重新退化为隐蔽卖货。

P1 更准确的稳定锚点是：

> **目标人群的穿衣选择机会。**

- 领域通用知识用于发现和解释代表性的选择矛盾；
- 品牌目标人群、立场、审美与服务知识用于决定从什么角度帮助选择；
- 商品知识是判断证据或候选解法，不是每篇 P1 的强制起点；
- 模糊输入允许系统创造“本篇要回答的内容问题”，但不允许创造用户本人的身体、职业、预算、经历或现实结果。

例如用户只说“这条高腰阔腿裤今天到了”，系统可以发现“小个子怎样判断裤长”“胯部紧张时看哪三个信号”“通勤正式感与舒适怎样取舍”等代表性方向，但不能断言用户本人个子矮、胯宽或明天要见客户。

## 三、正式名称：P1 弱种子主动编译

通俗说，就是用户不必先把题目想清楚；系统先判断是否已有内容生产意图，再把一句话、商品、图片、场景、评论或账号前情解释成可成立的穿衣选择内容。

### 3.1 极简运行规则候选（转入 P1-B）

1. **已有内容生产意图，且一个方向明显成立**：直接交付一个聚焦的 P1 成品，不先盘问。
2. **已有内容生产意图，但存在 2—3 个会实质改变成品的方向**：先给不超过三个简短选题方向并推荐一个；它们是策划建议，不是三份完整成品。用户选择或修正后，再生成一个成品。
3. **用户明确要求探索、策划或多路线**：才提供 2—3 个真正不同的方向或成品候选。
4. **缺少的信息可用条件分支消化**：直接完成；只有缺少不可替代信息、导致判断不能成立时，才问一个最小问题。
5. **普通闲聊尚无内容生产意图**：自然回应或轻量提出内容机会，不自动创建正式 P1 成品、任务或长期记忆。
6. **输入的主要价值不在帮助选择**：正确路由 P2/P3/P4/P5，不把任何模糊输入强行改造成 P1。例如“今天的蓝山咖啡居然是甜的”默认更接近 P3。

极简过程只有：

```text
模糊种子
→ 识别可观察锚点与内容生产意图
→ 结合目标人群、账号、领域知识和品牌/商品知识发现选择机会
→ 直接择优完成，或在真正分叉时给最多三个简短方向
→ 一个 P1 品牌成品
```

种子解释只服务当前任务，不需要独立意图图谱、多 Agent 推理链或用户可见的后台报告。

## 四、A—E 的统一身份

A—E 不再是五个互斥方案，也不能成为五个 UI 模式、五套 Agent 流程、五组字段或每篇机械必备的五个栏目。它们应收敛为“**一个 P1 合同中的四种能力与一个成品承诺**”：

| 原方案 | 在统一 P1 中的身份 | 最小边界 |
|---|---|---|
| A 知识解释 | 解释“为什么这样判断”的共享能力 | 服务选择判断；单独只有科普不能过 P1 |
| B 条件化决策 | P1 不可丢失的价值主干 | 必须形成选择、反选、两者都不选、条件分支或选择方法 |
| C 可执行行动 | 按任务和输入调用的行动增益 | 可给试穿观察点、穿法或步骤；缺少非必要输入不阻断主成品，也不自动升级为完整 P5 |
| D 传播竞争 | 品牌成品的表达质量 | 要有内容张力、品牌辨识和可使用性；不承诺流量、互动或成交 |
| E 双层结果 | 唯一成品合同的总体概括 | 同一成品同时让受众获得选择价值、让创作者获得可直接使用的品牌内容；不是第五套能力或第二份交付 |

因此不能简单“给每个缺点再加一层规则”。那会把五个方案堆成重型规则引擎，并使每篇内容结构雷同。第一版只保留四条不变量：

1. **B 不得缺失**：没有实质选择判断，就不是合格 P1。
2. **A/C 按需增益**：不为凑栏目强塞长篇知识或完整造型方案。
3. **D 不得压过 B**：好看、好笑、有钩子不能替代帮助选择，也不形成流量保证。
4. **不得猜补事实**：系统可以提出代表性内容问题，不能把推断冒充用户、商品或经营事实。

## 五、同一弱种子怎样工作

种子：**“这条高腰阔腿裤今天到了，帮我做内容。”**

若当前品牌受众和账号栏目已明显聚焦职业女性通勤，系统可以直接选择“见客户时，阔腿裤的松弛感怎样不变成拖沓”并完成一篇 P1。

若当前上下文不足以在下列方向中合理择优，系统只给简短方向：

1. 小个子怎样判断裤长和腰线；
2. 胯部容易紧张时看哪几个版型信号；
3. 通勤正式感与活动舒适怎样取舍。

这些方向来自典型目标人群问题与真实商品结构的交集，不表示用户本人具有相应身体特征。用户选定后，系统只交付一个完整成品；除非用户明确要求多路线。

## 六、统一成品承诺候选

> **P1 不要求用户预先提出明确的穿衣或购买问题。在用户已有内容生产意图时，系统可以从明确问题或一句话、商品、图片、场景、评论、账号前情等模糊种子中，结合目标人群、账号、领域知识及品牌/商品知识，主动识别可成立的穿衣选择机会。主方向明显时直接交付一个品牌内容成品；存在多个会实质改变成品的方向时，先给不超过三个简短方向并推荐一个，用户选择或修正后再完成成品。最终成品应使目标受众识别条件、理解取舍并形成可执行选择，同时成为创作者可直接使用的品牌内容。系统可以创造内容问题，不能创造用户事实；不机械堆满全部能力，也不承诺穿着、购买、传播或交易结果。**

## 七、建议验收目标

一个 P1 成品至少应满足：

1. 模糊种子足以形成条件化判断时，系统主动完成或提出必要方向，不先索取身高、体重、预算、库存等非必要信息；
2. 推断只形成代表性内容问题，不把身体、职业、预算、经历或场景推断写成用户事实；
3. 最终有实质选择判断，不以泛知识、安慰、商品介绍、品牌口号或传播钩子替代；
4. 说明决定性依据与适用/反选条件；行动建议只在确有价值且输入支持时加入；
5. 商品只作为证据或解法，允许“不适合、暂时不需要买、先用已有衣物解决”；未提供价格、库存和活动时直接省略；
6. 品牌知识实际改变判断角度、优先顺序或表达立场，不靠品牌名、Logo 或结尾 CTA 过门；
7. 关键场景、顾虑或商品事实改变时，结论应合理变化；
8. 默认只呈现一个完整内容成品，不向用户展示种子推断、能力编排和后台检查过程。

## 八、P1-A 正式确认

用户于 2026-07-20 正式确认：

> **P1 不要求用户先提出明确问题；内容生产意图成立后，系统可以从弱种子主动发现代表性选择问题。商品可以成为输入起点，但目标人群的选择机会是价值锚点；A—D 由一个 P1 合同自适应组合，E 是总体成品承诺。系统可以创造内容问题，不能创造用户事实。**

这关闭了 P1 的成品价值与总体承诺，但没有顺带冻结“何时直接生成、何时给方向、何时追问”及 P1/P2/普通交流的精确路由；这些只进入 P1-B。

## 九、交互工作台构想（记录，不进入 P1 合同）

用户提出：未来交互 UI 应按用户要做的事情组织入口，可以参考通用知识协作产品的功能区，候选包括随心输入、多集连续叙事、私人素材区、企业知识/素材管理等；系统底层知识库对普通用户不可见，品牌企业管理员可在授权范围内维护品牌专属内容。

当前只确认以下边界，不设计页面、菜单或技术实现：

1. 这是跨内容生产、系列、素材和企业管理的**交互工作台信息架构**，不属于 P1 产品合同，也不打断 P1—P5 当前推进顺序；
2. “随心输入”是自然协作入口，“多集连续叙事”是系列工作方式，不是新的 P6 内容产品；
3. 私人素材区继续遵守个人隔离边界，不能因出现在同一个 UI 中而向企业管理员开放；
4. “企业开发素材区”暂不定名：若指企业授权人员维护、供组织使用的品牌/商品/创意资产，应沿用 D-018 的“组织官方素材区”；若指总部、区域和门店共同投稿的共享池，则属于 D-029 成长期公共素材，第一版不出现空菜单或空模块；
5. 企业账号管理员可以分配“品牌知识维护人”职责，但不因管理员身份自动获得知识编辑权；被授权维护人只能在本租户、本品牌范围内提交、编辑、启用、停用和维护版本，不直接访问数据库表、系统检索过程或领域通用知识库；普通创作者不看技术后台，但仍可在业务中使用相应知识并提出纠错；
6. 首版导航、角色可见性和页面构成，应等 P1—P5、D-018 素材边界、D-005 陈列范围与 D-032 媒体成品基本明确后统一收口，当前不新增 UI 模块或权限实现。
