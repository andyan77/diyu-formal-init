# IP 同心圆、竞争性增益与账号连续记忆

日期：2026-07-18

状态：**用户已明确内容系统的价值愿景；架构师已给出 D-031 的具体设计建议，待用户裁决后进入实施基线。**

> **V0.3 后续修订（D-031 九项仍待裁决）**：本纪要最初把 `conversion` 混入商品角色，并把“导出明确版本”视为采用，这两点已被否定。用户现已确认商品角色与商业目标分开、`exported` 非采用等稳定原则；V0.3 又将认知变化与可观察商业行动分开，并拆分账号配置、账号连续记忆和系列状态。当前完整口径见[五类内容产品合同 V0.3](../内容产品/笛语第一版五类内容产品合同讨论稿.md)和[讨论纪要 029](2026-07-18-029-D031语义歧义审查与合同V03决策母稿.md)。

## 用户本轮明确的核心观点

1. 用户只输入一句生活琐事时，系统不应停留在润色或复述，而要完成有真实增益的内容创作。
2. 增益程度需要有标准；用户期待成品在自媒体内容市场中具有很强的相对竞争力，并能跟随平台内容趋势。
3. 用户采用后的系统增益内容应由产品和知识工程架构设计决定能否进入可复用内容记忆，不能把这个问题继续留给普通使用者自行理解。
4. 内容连续性和账号长期一致性是核心能力，不能把每次生成当作互不相关的单篇文案。
5. 无论总部品牌 IP 还是加盟门店本地化 IP，都应以服装业务为商业圆心，向目标人群的生活、关系、情绪、价值观、成功失败和人生体会扩展。
6. 企业员工、区域代理、加盟店经营者及其家庭和真实生活共同构成有烟火气的品牌 IP 矩阵。系统价值不是批量生产宣讲稿，而是帮助品牌形成有专业、有生活、有情感、有趣味且可以持续积累的人间叙事。
7. 用户随后明确纠正：父母、爱人、子女、情绪、失败和幽默等，属于系统账号人设能够生产和延续的内容，不是数据库中的具体实体类型对象。

## 必须纠正的四个概念

### 1. “极度竞争力”不能成为系统保证

内容表现还受账号基础、发布时间、拍摄与剪辑、商品和价格、投放、同期竞品及平台分发影响。系统可以承诺：

> 用可重复评测生成并筛选**相对更强的内容候选**，再用真实发布反馈持续校准。

系统不能承诺必然爆款、平台必然给流量或发布前已经证明市场竞争力。正式产品用语采用“高竞争力候选”“相对竞争优势”“待发布验证的内容假设”。

### 2. 商品是商业锚点，但不应成为每条内容的强制结尾

如果把 SKU 设为所有内容唯一圆心，每段夫妻、亲子、工作或人生故事最后都会被迫硬接一件衣服，这会重新变成用户反对的广告片。

更准确的内容核心是一个三角：

- 品牌/账号对目标人群的稳定承诺与立场；
- 品牌/账号与目标人群要建立的长期关系；
- 商品、服务和组织行动对上述承诺的真实证明。

商品仍是笛语区别于泛情感内容工具的商业锚点，但单条观点或生活叙事允许不出现商品。要求的是一个系列和一段时间的内容组合能够回到目标人群、品牌立场、穿着生活、门店关系或专业能力，不要求每一篇强行带货。

### 3. 组织人物网络不是同心圆的某一层

总部、区域、门店、员工、经营者和家人回答的是“谁在说、谁提供生活种子”；同心圆回答的是“这段内容离品牌核心有多远”。两者必须成为正交维度，不能混成一棵组织树或一套人物本体。

父母、爱人、子女等在业务语义上只是账号叙事中的关系角色或人物称谓，相关生活事件可作为用户声明内容种子；情绪、失败、成功、冷眼和幽默等是内容主题、人物状态、叙事语气或表达手法。它们均不成为共享数据库实体类型。即使某位家人长期出现在系列里，第一版也只在该发布账号的连续记忆中保存作用域受限的称谓、已采用前情和未完线索，不建立人物主数据。只有将来确实独立运营发布账号、具备稳定受众与内容任务时，才可能形成独立 `AccountPersona/ContentRole`。

### 4. 平台趋势是有时效的覆盖层，不是品牌人格的外圈

平台热点、搜索词、流行结构和互动方式会快速变化，应记录平台、观察时间、适用受众/账号、关联问题、有效期和过期状态。它们决定当期如何包装和分发内容，不能反向改写品牌的稳定立场和账号人格。

## “IP 同心圆”的正式建议模型

同心圆应作为内容战略和任务路由视图，不直接作为数据库层级或全系统本体骨架。

| 语义圆层 | 主要内容 | 商品绑定 |
|---|---|---|
| 核心层 | 品牌承诺、账号立场、目标人群关系、商品/服务真相 | 可作为证明，但不要求每条出现 |
| 专业邻接层 | 穿着、版型、审美、身体、场合、购买、搭配和服务判断 | 常用作证据或解决方案 |
| 目标人群生活层 | 工作、家庭、亲子、婚姻、年龄变化、城市与门店日常 | 可不出现，或自然成为场景 |
| 公共文化层 | 社会情绪、文化讨论、时代议题和有资格参与的热点 | 原则上不硬接商品 |

“欢喜、悲伤、失败、成功、冷眼、无厘头”不是新的圆层或本体类别，而是账号人设可以持续生产的内容主题、人物状态、叙事语气和表达手法；父母、爱人和子女也不是实体类型，只是在具体内容和账号连续记忆中按需出现的关系角色。

每个内容任务只需保存一条轻量关联路径：

`生活/经营种子 → 目标人群问题 → 发声账号及其立场 → 品牌关系 → 商品/服务角色（可为空）`

商品在内容中的作用与商业行动目标正交表达：`product_role = none | scene | evidence | solution`；`commercial_intent` 另行表示考虑、咨询、到店、查看详情或购买。如果删掉最后的商品段落，前文的故事和观点完全不受影响，商品也没有证明或解决任何问题，通常就是硬插广告。

组织角色作为另一条矩阵轴：

- 总部：提供品牌立场、设计/商品真相和组织现场，不假装拥有具体门店日常；
- 区域：继承品牌核心，负责气候、商圈、城市文化和跨店经验的区域转译，不重定义品牌；
- 门店：继承品牌核心，以店主/店员人格、本地关系、服务现场和当天问题形成生活内容，不冒充总部发布全国性结论。

因此，笛语的 IP 结构不是“一张越来越大的圆”，而是：

> **稳定的内容语义圆层 × 总部/区域/门店发声角色 × 带失效时间的平台趋势滤镜。**

## 一句话输入允许丰富到什么程度

不能用“最多补多少字、多少细节”定义丰富程度。正确原则是：

> **最大有效增益，最小无依据核心改写。**

`ContentProduct.enrichment_policy` 第一版只需区分四类行为：

1. **语义与结构增益，可自动完成**：识别人物关系、情绪张力、受众问题、内容立场，生成开头、节奏、转折、比喻、镜头与互动落点。
2. **专业与生活解释，可条件化完成**：调用商品和服装领域知识，给出带适用条件、取舍和替代方案的判断；不得把审美经验说成人人适用的确定效果。
3. **叙事演绎，可受控完成**：补充读者不会误认为承重事实的通用场景、叙事视角和表现形式。系统新编的具体对白、动作、心理和现场细节若会被理解为真实发生，应标为情景演绎或让用户轻量确认。
4. **核心事实改写，禁止自动完成**：不得无输入依据新增或改变人物核心身份、重大人生经历、商品参数、价格、活动、精确交易结果、功能效果和定量经营事实。

高竞争创作模式可针对同一种子生成 2—3 条**结构真正不同**的路线，例如专业判断、生活叙事、幽默观察，而不是三份同义改写；再通过统一评测选择和修改。普通快速任务不必强制生成三份，避免成本和操作负担。

## 高竞争力候选的首期评测标准

发布前先过四道硬门：

1. 系统未擅自新增承重硬事实或改变核心人设；
2. 生活题材与目标人群、账号表达资格、品牌/穿着/消费/门店/本地生活至少存在一条自然关联，不靠硬插商品；
3. 不是总部稿只换城市、人名或同义词的矩阵复制，也不与本账号近期内容高度重复；
4. 使用的平台规则和趋势信号仍在有效期内。

通过硬门后，按 0—4 分评估八个维度：

1. 受众问题或情绪张力是否具体；
2. 是否相对一句输入产生了真正的新判断、新理解、新情绪或娱乐价值；
3. 商品、角色、事件、本地场景、品牌立场和系列前情的组合是否具有专属性；
4. 专业解释是否诚实说明适用条件、取舍和反选；
5. 叙事是否有具体场景、变化、冲突或惊喜，而不是 AI 金句堆积；
6. 是否以目标平台原生的结构、节奏、搜索和互动方式表达；
7. 是否承接账号此前内容、回应未完问题，并为下一次关系积累留下空间；
8. 是否能用当前人物、商品、门店和素材真实拍摄、图文化或执行。

首期可以临时把“总分不低于 24/32、任一维度不得为 0 且通过全部硬门”作为进入人工候选的起始线，但它不是行业真理。必须用 10—12 个黄金任务、同账号近期内容和真实发布数据校准，不能靠架构师凭空固定永久权重。

## 用户采用后的连续记忆设计

第一版必须把五件事分开：

| 机制 | 回答的问题 | 能否自动成为事实 |
|---|---|---|
| `GenerationEvent` 全量留痕 | 系统曾生成、修改、采用、放弃、导出或反馈了什么 | 否 |
| `AccountContinuityMemory` 账号连续记忆 | 这个发布账号已经采用/表达过什么、留下什么问题和承诺 | 否 |
| `SeriesState` 系列状态 | 系列进行到哪里，哪些问题未完，下一集候选是什么 | 否 |
| `ExpressionProfile` 表达档案 | 账号长期喜欢怎样说、避免怎样说 | 只形成表达规则，不是事实 |
| `KnowledgeStatement/PolicyRule` 正式知识与规则 | 什么可以被当作当前有效事实或正式品牌方法 | 只能走独立确认和来源流程 |

### 什么叫“采用”

- 用户显式点击采用/定稿或“记住这一版”时，记录采用/定稿事件并形成连续记忆候选；仅生成、浏览、自动保存或导出不算采用。
- 采用不等于发布。只有用户标记为已发布或提交发布反馈后，下一篇才可以对受众说“上一期我们讲过”；内部采用但未发布的内容只能用于避免重复和规划。
- 用户后来采用新版时，新增一个有来源的记忆修订，不抹掉历史事件。

### 采用后自动写入的安全内容

- 本条采用版本的短摘要、来源内容与版本编号；
- 该账号已经采用过的观点和主题，语义标为 `prior_statement`，不标为客观事实；
- 未回答问题、后续承诺、下一集钩子；
- 已确认系列的集号、当前进度和连续性约束；
- 已存在商品、角色和场景对象的引用，不从成品文案反向复制商品参数。

### 必须经用户确认才能成为长期记忆的内容

- 系统增益出的婚姻、亲子、职业、创业等新核心人生设定；
- 是否把一次内容建立为长期系列；
- 是否把反复编辑形成的候选偏好提升为正式 `ExpressionProfile`；
- 是否把同一账号记忆扩大到其他账号、区域、门店或品牌；
- 是否把经验提交为品牌规则、商品事实、人物履历或领域知识候选。

### 第一版禁止自动写入连续记忆的内容

- 未采用、明确拒绝或只被系统生成的版本；
- 商品参数、价格、库存、销量、功能效果等硬事实；
- 用户未输入或未确认的重大身份和人生事实；
- 其他发布账号、其他工作区、其他品牌的连续记忆；
- 仅因机器评分高、播放量高或系统认为“好用”的写法。

### 作用域和召回顺序

账号连续记忆默认限定在 `tenant + workspace + brand + PublishingAccount`，必要时再加 `ContentRole`；同一员工操作多个发布账号不能串记忆，同店不同发布账号也默认不共享，总部不能借管理员身份读取加盟店账号的连续记忆。

下次生成只召回固定的小摘要槽位，顺序为：品牌/账号预设 → 当前硬事实 → 当前系列状态 → 最近有效连续记忆 → 已确认表达档案。禁止把全部历史正文塞回模型。界面应显示“本次沿用了哪些前情”，允许使用者排除、关闭、替代或删除。

## 第一版最小工程落点

1. 复用 ADR-010 已要求的 `GenerationEvent` 和内容版本链，不另建一套全量记忆库。
2. 内容模块新增 `AccountContinuityMemory` 和 `SeriesState` 两个轻量运行结构；连续内容成立后，`ContentSeries` 可成为内容模块的一等应用对象，但暂不因“同心圆”扩张共享核心本体。
3. `ExpressionProfile` 继续保存经确认的长期表达规则；单篇 AI 草稿被采用不能立即修改表达档案，避免模型学习自己的默认腔并循环放大。
4. 事实知识继续走 D-011 的 `KnowledgeStatement/PolicyRule` 升级路径。采用、高播放和连续复用都不能自动越过该路径。
5. 同心圆只增加内容任务的轻量运行语义，例如 `semantic_distance`、`brand_relevance_path`、正交的 `product_role/commercial_intent`、`organization_scope`、`account_authority` 和可选趋势引用；字段名和值域仍待 D-031/黄金任务验证，不建立庞大“同心圆本体”。

## 对现有决策的影响

- D-004：后续已确认 P1—P5 五类正式基线；生活、家庭、情绪和社会观察主要由 P3“品牌/账号观点与生活叙事”和 P4“一线问题回应与本地生活”承接，不因主题增多而新增大量内容产品。
- D-031：本纪要给出明确的增益、竞争力评测、采用语义和连续记忆建议，待用户裁决后可关闭。
- D-011：内容记忆复用与事实知识升级继续严格分开。
- D-016：暂不把情绪、家庭关系、同心圆层级或通用“内容记忆”加入 15—25 个共享核心类别；先把账号连续记忆和系列状态作为内容模块运行结构验证。
- ADR-010/ADR-011：全量生成继续留痕，发布反馈继续自愿；未采用输出只能用于机器评估和反重复，不能作为账号已表达过的前情。
