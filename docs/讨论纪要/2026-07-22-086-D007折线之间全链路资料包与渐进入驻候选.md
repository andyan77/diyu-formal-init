## 一、先把“生产就绪”说准确

用户建议由笛语方完整模拟“折线之间”的第一阶段演示资料和必要商品资料，并据此形成新租户入驻模板；新租户完成最低程度资料准备后即可转入生产。方向成立，但需要区分三个结果：

1. **合成演示资料就绪**：系统已经有一套前后一致的原创品牌、组织、账号、商品、素材和任务夹具，可以支撑第一阶段能力演示与回归；
2. **租户某项能力可生产**：针对当前要做的事情，系统已经知道谁在说、代表谁、对谁说、哪些事实可用、哪些不能编，可以立即生成相应成品；
3. **系统生产就绪或真实试点通过**：真实账号、租户隔离、权限、检索、模型调用、成品保存、版本留痕、删除恢复和实际人员使用均已在软件中跑通。

前两项可以由“折线之间”和入驻模板推进；第三项不能由资料文档、合成数据或黄金成品替代。资料准备好了，等于食材和菜谱齐了；生产系统就绪，还要求厨房真实通水通电并稳定出菜。

## 二、当前不是从零开始

现有资料已经覆盖：

- “折线之间”目标受众、品牌选择与拒绝、观察方式、关系姿态和表达幅度；
- 品牌母账号、总部零售/服务与设计/商品角色、南城店账号和终端经营角色；
- `ZX-A701`、`ZX-B463`、`ZX-C218` 三件内容级商品，以及 `ZX-S104`、`ZX-K126`、`ZX-P211`、`ZX-V113`、`ZX-Q117` 五件陈列级商品；
- P1 两包、P2 两包、P3 两包、P4 两包、P5 一包及其既有对照与变形检查；
- 抖音、小红书、微信视频号的五份文字型待制作媒体成品；
- 南城店双层挂杆条件、二十件当次清单、首次方案和反馈后的局部修订；
- 私人/组织素材边界、文字输出边界、未成年人真人原件禁入等首期规则。

真正缺少的不是再造一批黄金内容，而是把散落事实收成一个逻辑真源，并补齐演示入驻过程、最小区域与门店差异、实际合成素材记录和新租户可直接使用的资料准备样板。

## 三、“折线之间”只补一个最小充分的全链路资料包

### 3.1 一个共享品牌真源

只建设一套前后一致的资料，不为每个案例复制品牌事实。资料包至少包含：

1. **品牌总档案**：原创背景、经营定位、目标受众、稳定选择与拒绝、观察方式、关系姿态、表达幅度、审美边界和少量正反例；
2. **入驻过程样板**：展示“已有资料与行业原型预填—少量具体对照—自然对话补差—系统起草人话版草案—企业修改确认”的全过程；
3. **最小三级演示网络**：一个总部、一个区域代理、三家独立加盟店。南城店保留完整深度，另外两店只补组织身份、所在地、一个真实差异、一项服务特点和一个终端账号，不复制全部黄金任务；该拓扑只是演示品牌自身事实，不成为新租户入驻条件；
4. **最小账号与角色方案**：总部品牌母账号及已确认的两个专业角色、一个区域经营/本地转译账号、三家门店各一个终端关系账号。每个账号只说明代表谁、服务谁、从哪里观察、可以说什么、不能冒充谁；
5. **八张统一商品卡**：前三件保持内容生产级深度，后五件保持陈列执行级深度；每张明确已知、未知、可使用主张和不得编造项，不批量创造价格、销量、顾客评价和开发故事；
6. **四个组织素材边界样例**：一份总部下行素材、一份区域下行素材、一份南城店本店素材和一份合成账号私人素材，用最少记录证明调用范围；私人样例与品牌共享真源物理或逻辑分开，不能因为服务演示就变成企业公共资料；
7. **账号冷启动与连续内容样板**：一个总部账号和一个门店账号的首批方向及第一条可生产任务；另用一张轻量系列计划证明用户可以跳集、插集、重排和指定前情，不建设固定三十天排期或内容比例；
8. **能力覆盖与诚实说明**：逐项引用现有 P1—P5、D-032、D-005 和相关边界证据，明确哪些是合同已验证、合成资料已具备、文字成品可生产，哪些尚未真实拍摄、部署或由企业采用。

账号私人协作偏好和私人素材只使用独立的合成账号夹具演示，不进入品牌共享入驻资料。平台规则、服装领域通用知识和公开地域搜索能力属于系统能力，也不要求租户重复填写。

### 3.2 明确停止条件

达到以下条件即停止补全：

- 上述八项前后一致并有唯一当前版本；
- 现有九个 P1—P5 黄金任务、五份媒体包和一个墙面挂杆闭环都能从该资料包取得所需稳定事实；
- 新租户可以看懂“资料怎样进入系统、怎样被系统起草、怎样确认并开始第一项生产”；
- 已实现、待制作、待开发和待真实试点四种状态没有被混说。

不继续模拟完整季度商品企划、全国全部区域、五家以上门店、真实素材库规模、三十天日历、ERP 库存、流量结果或企业内部审批。

## 四、新租户模板不是一张巨型表

后台可以有一份完整覆盖清单，但新租户看到的是“上传已有资料或自然说明—系统起草—改几句确认—立即开始—使用中渐进补齐”。允许三种低负担方式并用：

- 上传或对照现有的品牌手册、商品册、账号资料、门店清单和素材；
- 对系统预填草案做“像/不像、用/不用、A/B、都不对”等具体判断；
- 直接用自然语言补充和纠正，不要求理解数据库、本体、字段和产品编号。

### 4.1 第一次激活只需要四件事

1. **确认自己是谁**：租户、实际控制组织和真实存在的组织关系；单店就只建单店，不补造总部或区域；
2. **内容生产才确认怎样与受众相处**：由系统起草 D-014 人话版品牌、企业或店铺表达基线，企业修改并确认；只启用对内陈列时跳过对外人格，复用当前品牌/陈列规则；
3. **激活一个真实工作入口**：内容生产至少有一个发布账号、适用内容角色和有效操作授权；对内陈列只需要实际执行组织，不套用对外 IP 人设；
4. **让至少一类任务满足最低成立条件**：满足哪类就先生产哪类，不等待全部商品、账号、门店和能力补齐。

### 4.2 不同能力只补自己的最低条件

| 要开始的事情 | 额外最低条件 |
|---|---|
| P1 穿衣选择 | 一项对目标受众有意义的处境或弱种子；涉及具体商品时才需要相应商品事实 |
| P2 商品真相 | 至少一件真实商品及本篇承重的商品事实；未知可以明确保留，不能猜补 |
| P3 人格叙事 | 当前账号角色与一个生活、工作或情绪种子；不要求商品 |
| P4 在地关系 | 账号合法在地位置；公开地域背景可由系统按需搜索，当次真实门店信号按需提供 |
| P5 视觉叙事 | 至少一件真实商品、足以避免失真的可见特征和当前制作条件 |
| 连续系列 | 一张轻量系列方向、用户确认的前情和本次想推进的位置 |
| 墙面挂杆陈列 | 首次记录本店这组挂杆的固定条件；日常只提供当前实际可用商品和数量 |

一个租户只满足 P3 条件时即可先做 P3；不能因为尚未具备 P2、P5 或陈列资料，就把整个租户判为“不能生产”。

### 4.3 每次任务才出现的信息

生活种子、评论、门店观察、当次活动、所选商品、人工库存、平台、时长、表现形式、制作人数、当集前情和临时场地限制，默认留在本次任务。只有用户主动保存或确认进入相应长期范围后才跨任务复用，不能为了入驻完整度提前制造会过期的数据。

## 五、系统构建不阻塞与 D-007 真实试点门

新租户不阻塞系统构建。具体只需区分四个层次：

1. **系统整体构建**：不等待新租户或真实品牌。可以直接使用“折线之间”推进资料真源、渐进入驻样板、合成演示、黄金回归、数据结构、检索编排、交互原型和第一版软件纵向闭环；
2. **某个新租户启用能力**：该租户缺少某项资料时，只影响自己的相应能力。例如没有真实商品时不能完成 P2/P5 商品成品，但仍可使用已经成立的 P3；不能阻塞其他租户或系统全局；
3. **某次具体任务**：缺少不可替代输入时，只收窄这一任务或追问一次必要信息，不能把任务缺口升级为租户停用或系统停工；
4. **真实试点和对外声明**：在宣称“首期真实试点通过”、关闭生产部署或完成企业业务验收前，仍必须按 D-019 使用授权真实企业、区域和 3—5 家独立门店复演至少一条内容生产路径及 D-005 墙面挂杆闭环；ERP/POS 仍非前提。合成结果只能称演示、合同验证或原型证据，不能称企业采用、市场效果或真实试点通过。

因此，真实租户是**验证门**，不是**开发启动门**；租户资料缺口是**局部能力门**，不是**系统总开关**。
