# 2026-07-17 首期试点范围确认与三级合作关系

## 一、已确认的首期试点范围

1. 一个单品牌企业/总部作为主方；
2. 一个区域代理；
3. 3—5 家加盟终端门店；
4. 其他品牌、区域、门店和经营形态暂不纳入首期正式试点。

该范围是试点配置，不得被写死成系统最大容量或永久业务模型。

## 二、兼容性表述纠偏

正式表述为：

> 首期范围收敛，架构保持可扩展，已有数据和已发布合同保持向后兼容。

不使用“所有未知未来需求保持前后向兼容”的无法验收承诺。

## 三、下一项只讨论三方在系统中是什么关系

### 具体场景

品牌企业掌握品牌定位、官方商品资料和全国规则；区域代理负责该区域的渠道运营与门店辅导；3—5 家加盟店独立经营门店，负责本地内容、素材和现场执行。

### 三种关系设计

| 设计 | 实际后果 | 判断 |
|---|---|---|
| 把代理商和加盟店当成总部内部部门 | 看似简单，但会误认为总部天然拥有所有门店资料和无限管理权 | 不符合加盟经营现实 |
| 总部、代理和门店各用一套完全隔绝的系统 | 各自边界清楚，但品牌规则、内容任务和门店反馈无法正常流动 | 违背三级协同目标 |
| 同一个品牌协作网络，但保留三方独立经营身份和各自工作区 | 品牌规则可下达，区域可辅导所辖门店，门店可保留本地工作空间与责任 | **建议采用** |

### 建议的业务说法

1. **品牌企业/总部**：是试点主方和品牌规则提供方，不因此自动变成所有门店本地资料的所有者。
2. **区域代理**：是经品牌授权、在指定区域和合作期内管理/辅导门店的独立经营参与方，不是默认的总部内部部门。
3. **加盟终端门店**：是受品牌和区域授权关系约束的独立经营参与方，不是总部或代理商的普通员工账号。
   - 如果某家门店实际由品牌企业或区域代理直接经营，则按真实关系单独标记为品牌直营店或代理直营店；不得因为都叫“终端门店”就把直营与独立加盟混为一类。
4. 三方进入同一套试点系统，但以“品牌 + 区域 + 合作期限 + 被授权门店”建立合作关系；授权到期或解除后，停止后续访问，但保留必要审计记录。
5. 具体谁能看什么、门店素材能否向上或横向复用，留给后续两项单独确认，不由“同一系统”自动推导。

## 四、用户已确认第 2 项

> 首期试点中，品牌企业/总部是试点主方和品牌规则提供方；区域代理和真实的加盟门店保留独立经营身份，不当作总部内部部门。三方在同一套系统中，按品牌、区域、合作期限和被授权门店建立关系；区域代理只管理/辅导本次授权的门店，门店保留自己的工作区和责任。

用户进一步确认：本批 3—5 家试点门店全部为独立加盟店，不包含品牌直营店或代理直营店。
