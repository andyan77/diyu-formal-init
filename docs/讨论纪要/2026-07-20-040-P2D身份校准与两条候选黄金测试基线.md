## 一、本轮先关闭的身份歧义

### 1.1 P1—P5 与账号人设不是同一层

P1—P5 是对外内容生产的上层成果合同，规定一篇成品最终要帮助选择、解释商品、建立人格、经营关系或创造欲望。它们不是账号类型、人设模板或组织权限。

一条实际对外成品必须同时解析：

> **代表组织 + 发布账号 + 组织性内容人设 + 一个主要内容产品合同 + 当前任务与事实。**

因此，同一个 P2 合同可以由总部设计/商品专家、区域经营角色或门店店长分别执行，但三者的观察视角、可以代表谁说话及不得冒充谁并不相同。

P1 的两条已确认黄金任务已经分别明确绑定：

- 包 1：折线之间品牌母账号 + 总部零售/服务专家；
- 包 2：独立加盟终端南城店账号 + 店长/搭配顾问。

此前 P1 使用的是最小充分角色卡，已经足以约束当时的任务，不需要重开 P1；真实品牌实施时再由企业正式角色卡替换合成夹具。

### 1.2 对内经营陈列不使用对外 IP 人设

用户进一步明确：对内经营陈列应首先对照组织身份。正式口径为：

> **对外内容使用“发布账号 + 内容人设”；对内经营陈列使用“组织身份 + 业务任务 + 必要业务条件”。**

总部、区域代理和加盟门店面对的陈列培训、指导或执行范围不同；岗位只在需要说明谁制定、执行或复核时出现，不为内部成果另造“陈列老师 IP”、粉丝关系或一套内部人格路由。

### 1.3 “折线之间”的性质

“折线之间”是本项目为黄金测试创建的**完全合成品牌名称**。其品牌原则、账号、角色、商品、数据和场景均为受控测试夹具，不对应现实企业、人物、门店或商品，也不能自动进入真实领域知识或任何租户品牌知识。

## 二、两条基线共同使用的具体角色卡

本轮撤回 V0.1 临时新增的“商品观察栏目”称谓，直接使用《服装品牌组织型 IP 内容战略与知识库建设基线》已经定义的总部角色。

| 项目 | 本轮固定值 |
|---|---|
| 代表组织 | 折线之间品牌总部 |
| 发布账号 | 折线之间品牌母账号 |
| 内容人设 | **总部设计/商品专家**，属于专业权威角色，不对应某位具体员工 |
| 稳定权威 | 使用当前商品事实，解释商品结构、开发取舍、商品组合和适用边界 |
| 服务对象 | 30—45 岁、需要在工作与个人生活状态之间切换的城市女性 |
| 稳定立场 | 让商品的价值与限制同时可见，不用焦虑、口号或逼单替代解释 |
| 稳定语气 | 成熟、平等、观察具体、有生活温度和少量幽默 |
| 身份禁区 | 不冒充创始人、具体设计师、顾客、门店店长或内部陈列培训岗位；不声称实际运营者亲自参与开发或拥有未提供的个人经历 |

任务 1 额外拥有品牌确认的组织级开发说明，因此可以使用“我们在本款开发中……”；任务 2 没有该说明，角色和创作自由不变，但不得解释品牌当初为什么这样做。

## 三、正式黄金基线 1：有品牌确认开发理由

### 3.1 本篇商品认知主线

> **这不是“一面成衣加一面内里”的双面外套；品牌明确要求两面都成为完整正面，也在已知当前样衣重量差异的情况下接受这一取舍，不以极致轻量为目标。**

注意：现有输入能证明品牌知道约 310 克的样衣差异并作出上述开发选择，不能证明物理上全部差异都由某个具体结构造成。

### 3.2 当前择优参考媒体成品

**标题：**这件外套，没有哪一面肯当里子  
**封面字幕：**两面都要当正面，取舍也不躲  
**参考载体：**约 45 秒竖屏；一位运营者、固定手机、一件 M 码样衣、普通室内空间

| 时间 | 画面与动作 | 完整台词/解说 | 关键字幕 |
|---|---|---|---|
| 00:00—00:05 | 固定机位。运营者手持炭灰面朝外的样衣，让衣服自然落下，随后看向镜头。 | **“这件外套最不配合的地方，是两面都不肯当里子。”** | `两面都不肯当里子` |
| 00:05—00:14 | 在镜头前完整翻到深绿色细格纹面，再分别把两面的口袋带近镜头，用手实际插入。 | **“炭灰纯色翻过去，是深绿色细格纹；两面都有完整外观，口袋也都能用。”** | `炭灰纯色 / 深绿细格`<br>`两面外观完整，口袋可用` |
| 00:14—00:28 | 画面仍只展示当前样衣；数字以字幕出现，不虚构第二件对照样衣在场。 | **“开发时，我们明知道这件 M 码样衣约九百六十克，比同季同长度的单层样衣重约三百一十克，还是没有把其中一面简化成内里。因为本款的要求就是：两面都能独立成为正面。”** | `当前M码样衣约960g`<br>`同季同长单层样衣约650g`<br>`已确认开发要求：两面完整` |

**声音建议：**

- 开场不铺音乐，先保留衣服自然落下和第一次翻面的布料声；
- 00:14 后进入极低音量、克制的节奏底声，报数字时再压低；
- 两次翻面保留真实衣料声，不添加新品发布会式重音；
- 最后一句只保留人声和轻微环境声，不加购买、到店或评论 CTA。

### 3.3 同制作条件平庸对照

**标题：**一件外套，两面都可以穿  
**动作：**使用同一运营者、固定机位、室内空间和样衣，依次展示两面与口袋。

**完整口播：**

> 今天带大家看折线之间这件双面短外套。A 面是炭灰纯色，B 面是深绿色细格纹，两面都可以作为完整正面，口袋也都能正常使用。M 码衣长 62 厘米，样衣约 960 克；和同季同长度单层短外套约 650 克相比，相差约 310 克。开发时，品牌要求两面都不是内里，因此保留了两面的完整外观和可使用口袋，也没有把极致轻量作为目标。一件外套可以呈现两种外观，同时当前样衣的重量也更高，大家可以根据自己的搭配需要和重量接受程度选择。

该稿没有明显事实错误，也使用了开发说明，但只是按顺序朗读外观、参数、原意和结论；新增资料没有被编译成值得观看、能够记住的商品理解，任何同类双面商品账号都可以套用。

## 四、正式黄金基线 2：无品牌确认开发理由

### 4.1 本篇商品认知主线

> **两种完整外观和当前 M 码样衣约 960 克的分量同时存在；系统不解释品牌为何作出这一结果，却仍能把“衣服可以翻面，取舍不能”编译成有记忆点的外部商品内容。**

本篇只能描述当前商品结果和同条件样衣比较，不得使用“为了、宁愿、坚持、初心、开发时我们决定”等隐含历史动机，也不得把约 310 克全部归因于双面结构。

**封面字幕：**两面都是真的，重量也是

**标题：**双面短外套怎么选？

**完整口播：**

该稿没有编造品牌动机，却把“不知道原意”误处理成了谨慎报数据；没有清晰观点、账号人格和观看回报，结尾问题也可以无差别套在任何双色或双面商品上。

## 五、两任务单变量检查

### 5.1 保持不变的输入与结果

- 品牌、代表组织、发布账号、总部设计/商品专家人设、目标受众和制作资源完全相同；
- 商品事实均为两面完整外观、两面可用口袋、M 码样衣约 960 克、同季同长度单层样衣约 650 克；
- 两篇均面向外部受众，不是商品培训、销售话术或对内陈列指导；
- 两篇均保持“成熟、平等、具体、诚实讲取舍”的账号声音和相近创意投入；
- 两篇均未新增面料、工艺、成本、售价、库存、功能、耐用性、普遍穿着感、顾客经历或设计师故事；
- 两篇均没有把 62 厘米衣长推导为显高、显瘦或适合某种身材，也没有把炭灰和格纹硬分配给职场、家庭或周末身份。

### 5.2 唯一应该发生的变化

- **基线 1** 使用了组织级确认信息：品牌知道当前样衣与同季同长单层样衣存在约 310 克差异，仍要求两面都成为完整正面，并明确不以极致轻量为开发目标；
- **基线 2** 完全不回答品牌为何这样开发，只把商品结果和对照数字编译为外部商品观点；
- 基线 1 删除开发说明后无法原样成立；基线 2 没有因为缺少原意而退化为说明书，也没有偷用“宁愿、为了、坚持”等隐性动机语言。

### 5.3 专项防错结论

两篇均没有声称“多出的约 310 克全部来自另一面的完整结构”。现有输入只允许表达：两种完整外观与当前样衣重量落在同一件外套上；任务 1 另有品牌在知道差异后仍作出相应开发选择的组织级说明。该区别必须保留。

## 六、内容竞争与路由判断

- 基线 1 的赢点是以“两面都不肯当里子”切入真实开发取舍，并用“取舍也该正面说”形成品牌可记忆表达；
- 基线 2 的赢点是在没有官方原意时仍用轻微状态演绎和“衣服可以翻面，取舍不能”形成观看回报，不以谨慎话术逃避创作；
- 两篇的笑点、动作和语言都服务具体商品理解，移除 ZX-C218 后无法保留核心，因而没有滑入 P3 泛生活观点；
- 两篇解释商品而不替某个具体人作购买选择，没有滑入 P1；
- 两篇都不承诺流量、互动或成交。“黄金”只表示在当前受控输入下通过 P2 价值、事实、身份、品牌和竞争性验证。

## 七、确认记录与关闭结论

用户于 2026-07-20 明确确认上述两条成稿正式成为 P2 黄金语义基线，并同意进入 P3-A。确认对象不是每个固定句式或镜头模板，而是：

1. 具体发布账号和总部设计/商品专家人设真实约束了表达；
2. 两篇均形成商品专属新理解和诚实取舍，而不是内部讲义；
3. 有原意时自然使用，无原意时仍充分创作且不编品牌动机；
4. 两篇均有明确观看赢点且不存在致命平庸、身份越级或事实越界。

据此，P2-D 与 P2 正式关闭，五类专属验收完成度由 1/5 更新为 2/5，唯一下一工作项转入 P3-A。两条黄金成稿只用于证明 P2 价值、事实、身份、品牌、创作与单变量边界同时成立；后续不得按其题材、句式、镜头、声音或结构要求其他 P2 内容照抄，也不得把成稿反向升级为商品事实。
