## 一、怎样算“等义且无损”

旧条款与新条款只有在以下要素全部相同或更清楚时才算等义：**适用主体、作用范围、触发条件、规范强度、允许/禁止动作、失败或降级后果、例外条件、明确后置边界**。仅有近似措辞或相同主题，不算完成映射。

出现以下任一情况即判定有损：

1. 旧稿仍有效的确认规则没有新位置；
2. “必须/不得”被改成“建议/可以”，或反向无授权加严；
3. 主体、范围、触发条件、例外或降级方式丢失；
4. 产品专属差异被误删为共用规则；
5. 黄金/性质测试被误写成日常运行流程；
6. D-016 的对象/物理存储或 D-032 的媒体规格被提前冻结。

本证明只使用一组映射边，下面先按旧稿排序，再按新稿排序。它不是第二套规范，也不建设规则数据库、语义哈希或自动迁移器。

## 五、两次独立语义审查

两名审查者分别读取母稿、ADR-011、X-01 证据和本映射，不共享中间结论。每次都必须独立回答：

1. 正向映射是否存在有效旧规则无去处；
2. 反向映射是否存在无权威来源的新稳定规则；
3. 是否发生强度、范围、触发、例外、降级或产品差异漂移；
4. 是否把评测变成运行时，或提前冻结 D-016/D-032；
5. ADR-011 是否完整进入基线且没有残留强制反馈门；
6. 稳定编号是否足以在章节移动后继续引用，且没有重复建立相反正文。

### 审查 A

最终状态：**PASS，四项结果 `0 / 0 / 0 / 0`，无阻断项。**

首轮审查曾正确判为 `FAIL (0 / 0 / 1 / 0)`：母稿把 ADR-011 的“可以非阻断提醒、不得反复形成事实强制”擅自收紧为“只能提醒一次”。这属于未经授权的允许范围缩窄，即使看起来更克制也不是等义改写。母稿随后删除“一次”，改为“低干扰、非阻断、可跳过/关闭/稍后处理、不得靠反复弹窗形成事实强制”；审查 A 对受影响条款重新复核后通过。

其余结论：旧有效规则均有稳定去向，新稳定 ID 均有正式来源，P1—P5 差异无误删，X-01/黄金测试没有变成日常流程，D-016/D-032 无实现泄漏。

### 审查 B

最终状态：**PASS，四项结果 `0 / 0 / 0 / 0`，无阻断项。**

审查 B 独立确认：原 §3.11/§9 已合并为 `SH-09` 唯一规范正文；原 §11 已去重为状态与下游边界；P1—P5 专属差异完整；ADR-011、稳定编号和后置边界均成立。其提出的三项非阻断精确化已吸收：顶部第 8 项改指 `SH-09`，G11 与未执行 G12 的处置分开表述，`SH-13` 的反向来源补充 ADR-010 与既有访问边界。上述修订及审查 A 的提醒措辞修复后，审查 B 再次确认 `PASS`。
