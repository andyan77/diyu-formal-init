# 2026-07-18 成长期公共素材与上下文降级检索

## 一、用户提出的核心观点

1. 每个区域可有区域公共素材包；门店账号使用者可以把本人认可的私有素材提供给本区域共同使用。
2. 总部可有公司级公共素材区；总部、区域代理和全国门店的账号使用者都可以把本人认可的私有素材提供给公司范围共同使用。
3. 当现有知识和素材不足以为内容创作组装高质量上下文时，系统可配置为从公共素材区补充相关素材。
4. 该能力不进入第一版开发，只在成长期实现；第一版只保留合理的扩展能力。

## 二、与已确认首期边界的关系

这不是对 [ADR-008](../架构决策/ADR-008-个人私有原始素材隔离与首期禁止共享.md) 的推翻。ADR-008 仍约束第一版：私人素材不共享、不上提、不转组织、不跨账号复用。公共素材贡献和检索属于成长期的新模块，只有后续正式立项、补齐权限与生命周期规则并通过验收后才注册启用。

公共素材模块也不等于 BBS：前者管理可复用素材、适用范围和生成时检索，后者管理讨论、评论和协作关系；二者可以独立立项，不能为了公共素材顺带预建讨论区。

成长期建议保留三个明确不同的素材空间：区域共享素材池只向该区域网络开放；品牌网络共享素材池向租户内符合适用品牌条件的授权成员开放；总部官方素材库仍由总部维护并代表更高权威等级。所谓“素材包”只是公共素材版本的清单或保存查询，不为每个区域复制文件或另建物理数据库。

## 三、五项必须纠正的表述

### 1. “转入”应改为“主动发布一个公共版本”

系统不能把私人原件直接搬走或让公共权限穿透私人素材箱。使用者必须主动选择素材和共享范围，系统建立一个权限、版本和生命周期独立的公共发布记录；私人原件仍保持私有。底层文件可以按内容哈希去重，但业务上私人原件和公共版本必须是两个不同资产身份。

公共版本至少要记录发布者、来源谱系、区域或公司范围、适用品牌、允许用途、有效期、是否允许改编、权利说明、当前状态和撤下记录。私人原件以后删除，不应悄悄破坏已发生的生成和审计；公共版本的撤回、保留与既有使用处置应在成长期立项时单独确认。

### 2. “公司公共”不等于“总部官方”或“正式知识”

任何账号主动贡献的公司公共素材，只代表它被提交到指定共享范围，不自动代表已允许系统检索，更不代表总部审核、品牌背书或事实正确。公共贡献、总部认证素材和正式知识必须保留不同的权威等级。热度、使用次数和内容表现也不能把公共素材自动升级为品牌规则或事实知识。

未来兼容多品牌时，公司级可见也不等于所有品牌都可用；公共版本必须声明适用品牌为“企业通用”或明确的品牌集合，检索时按任务品牌过滤，防止跨品牌串用。

### 3. “隐藏设置”应改为“总部管理员可见的高级开关”

开关可以不展示给普通创作者，但不能成为无人知晓、无法审计的暗门。它应是租户级或品牌级的后台配置，默认关闭、带版本和生效时间；每次触发都记录使用了哪些公共素材版本，并在生成任务中向使用者显示“本次使用了公共素材补充”及来源。

### 4. “完美上下文”不可验收，应改为“任务上下文覆盖度”

系统不能可靠判断何为“完美”。每种 `ContentProduct` 应声明上下文需求槽位，并至少分成：

- **硬事实/硬约束**：商品、价格、库存、活动、品牌规则、平台规则、人物或素材授权等。缺失时只能请用户补充、明确降级或停止相应主张，绝不能从公共素材猜补。
- **创意与生活语境**：场景、故事线索、消费者语言、视觉参考、地方生活细节和表达灵感等。只有这类软槽位不足时，公共素材降级检索才可启动。

### 5. “人人可贡献”不等于“投稿后立即被系统自动调用”

允许所有账号直接把素材变成全国自动检索源，会很快形成低质、错标、过期、重复或权利不清的噪声池，并被降级检索成倍放大。成长期应把两种权利拆开：任何当前有效的自然人账号都可提交自己的素材；区域素材管理员决定区域投稿何时成为可用公共版本，总部素材管理员决定公司投稿何时成为可用公共版本。管理员治理的是投稿快照和公共版本，仍不得借此打开私有源。

最小状态可以是 `contributed → active`，并支持 `rejected / suspended / withdrawn / expired`。素材进入公共区后也要继续区分“可供成员人工选用”“允许自动检索”“总部官方精选”三个等级；公开可见不自动等于可投喂模型，更不等于总部背书。

这里的激活不是区域或总部审批门店要发布的自媒体内容，不改变门店内容自主权；它只决定一份投稿是否进入供其他账号复用和供系统自动召回的公共资产池。激活也不代表系统已经替各方完成法律权利判断，贡献者仍对权利说明负责，素材管理员只承担租户内部的公共池质量和使用范围管理。

## 四、成长期的工程实现

建议采用下面的受控链路，而不是让大模型直接搜索整个公共素材库：

1. **主动贡献**：使用者从私人素材中选择某一版本，明确提交到本区域或公司范围；系统创建独立的投稿快照、公共版本与使用声明。区域或总部素材管理员只审核该公共版本，并决定是否激活及是否允许自动检索。
2. **加工与索引**：对象存储保存媒体文件；关系型数据库保存权限、版本、适用品牌、权利和状态；图片/视频按需生成说明、OCR、语音转写和多模态向量，全文与向量只是可重建索引。投稿中的文字、OCR、转写和 AI 说明一律按不可信数据处理，与系统指令隔离，防止素材内容反向操纵生成流程。
3. **上下文盘点**：内容任务先根据产品合同检查硬槽位和软槽位的覆盖度。硬槽位不足时不进入公共素材补齐。
4. **受控召回**：高级开关开启且软槽位不足时，区域/门店任务先查任务所在区域，再按策略扩大到公司范围；总部全国任务直接查公司范围，除非任务明确指定某个区域。先按租户、品牌、区域、`active`/允许自动检索状态、有效期和允许用途硬过滤，再做全文/向量召回与重排。
5. **反同质化重排**：相关性之外，还要考虑本地性、新鲜度、近期开采次数、与该账号近期内容的相似度和候选之间的多样性；高频、重复或过度相似的素材降权，避免公共池反而制造全国同款内容。
6. **分区投喂**：把可验证事实与“仅供灵感的公共素材”放在不同上下文区，禁止模型把灵感素材当成商品或品牌事实。
7. **生成与留痕**：记录是否触发降级、触发原因、使用的公共素材编号/版本、排序结果和最终引用关系，供使用者复核、问题追踪和撤下影响分析。

可用三个成长期对象承载这套能力，不需要扩大第一版核心本体：

- `AssetPublication`：私人版本向区域/公司公共范围的主动发布及生命周期；
- `ContextFallbackPolicy`：开关、允许补充的槽位、检索次序、数量和阈值；
- `ContextAssemblyTrace`：本次上下文缺口、召回来源、采用素材与版本留痕。

它们主要属于应用数据与运行审计，不应为了凑“本体完整性”全部提升为第一版核心语义类别。

## 五、第一版究竟预留什么

第一版不创建区域/公司公共素材表、发布按钮、公共权限、公共索引、降级开关、后台任务或空实现。所谓“预留接口”只保留两项真实且当前就有用途的稳定能力：

1. 当前上下文组装器通过版本化的“上下文来源提供器”读取品牌知识、组织官方素材和使用者本次主动选择的私有素材，而不是把这些来源写死在生成器里；成长期再注册公共素材提供器。
2. 现有 `MediaAsset`、任务上下文和生成审计保留稳定资产编号、版本、来源类型、适用品牌/工作区和来源谱系，使以后增加公共版本时不必破坏旧记录。

这符合 D-024：首期的扩展点服务于当前真实来源，不是为未知未来预建空壳；公共素材模块立项后才通过新合同版本、迁移和验收加入。

## 六、本轮已确认的裁决 D-029

> **第一版继续执行 ADR-008，不提供私人素材共享或公共素材功能；只把当前已经需要的上下文组装能力做成版本化来源扩展点。成长期可另行增加区域公共素材和公司公共素材模块：任何有效账号可主动把本人选中的版本贡献为独立公共版本，但贡献权不等于自动生效权；区域素材管理员激活区域版本，总部素材管理员激活公司版本，公司公共素材还必须标记适用品牌。公共贡献不等于总部官方素材或正式知识。公共素材降级检索由总部管理员可见、默认关闭且全程留痕的高级开关控制，只能调用已激活且允许自动检索的版本，只能补充创意和生活语境，不能补商品、价格、库存、品牌规则、平台规则或授权等硬事实；区域/门店任务默认先区域后公司，总部全国任务直接使用公司范围，并通过新鲜度、多样性和重复使用惩罚避免同质化。**

用户于 2026-07-18 明确接受上述裁决。D-029 已关闭并形成 [ADR-009：成长期公共素材发布与创意上下文降级](../架构决策/ADR-009-成长期公共素材发布与创意上下文降级.md)，进入成长期架构路线；第一版范围不增加。
