# P2-C 商品认知主线的成品化编译候选

状态：`owner_confirmed_product_contract`（用户于 2026-07-20 正式确认）  
日期：2026-07-20  
上游确认：P2-A“条件性商品真相与预期管理”、P2-B“对外最小商品真实底线 + 充分创意表达”均已关闭。  
当前唯一工作项：P2-C。

## 一、为什么本项可以极简收口

P2-A 已经回答受众最终获得什么，P2-B 已经回答哪些内容可以自由创造、哪些商品表达不能编造；ADR-014 也已确认系统只交付成品、共享能力按需组合、媒体规格后置、日常只作快速必要自检。

因此 P2-C 不再设计新流程，只需关闭一个剩余问题：

> **商品事实、专业取舍、品牌/账号表达和创意怎样自然熔成一篇外部内容，而不是被排列成商品资料。**

## 二、必要纠偏

以下三条路线均不采用：

1. **六要素显式合成**：事实、价值、代价、适合、不适合、替代逐项输出，会重新变成培训说明书；
2. **媒体形式先行**：先选故事、情景剧或口播再填商品，会让形式反客为主，容易滑向 P3/P5 或生硬植入；
3. **模块化动态拼装合同**：把事实、取舍、故事、视觉和 CTA 建成业务模块清单，会把内部实现猜想升级为产品流程。

## 三、推荐合同

**正式名称：单一商品认知主线的成品化编译。**

通俗说，就是：先确定这篇内容要让受众对这件商品真正多懂什么，再把必要真相、专业取舍、品牌/账号人格和创意表达自然熔成一篇内容，而不是逐项念出来。

> **每个独立 P2 成品应围绕一项最值得传播的商品专属认知主线，明确本篇希望受众新增、修正或重新建立什么商品预期。系统只调用支撑这条主线所必需的真实商品信息和专业解释，并把相伴的价值、限制或取舍自然融入符合品牌及账号人设的完整外部表达。商品事实不得写成资料朗读，真实取舍不得写成固定“劝退段”，品牌关联不得依靠片尾口号、生硬贴标或强制品牌宣言。比较、替代、生活情境、幽默、演绎、口播、视觉呈现、声音和 CTA 均为按需能力，只在增强这条商品认知主线时加入。内容改写为其他时长、媒体形式或制作方式时，可以重新组织全部表达，但不得丢失这项商品专属理解。**

品牌相关性可以通过选择哪项商品真相、以什么立场解释、由哪个账号怎样发声而自然成立，不要求每篇念品牌名、Logo、品牌历史或价值观。

## 四、成品不可丢失的结果

这些是后台判断结果，不是正文四段：

1. **商品专属新认识**：能用一句话说清，受众看完对这件商品新理解了什么；
2. **条件性取舍**：商品特点得到什么、限制什么或在什么条件下成立，不能只剩单向卖点；
3. **人与品牌的自然连接**：商品真相转译到身体、行动、审美、生活、身份或情绪之一，并以该品牌/账号成立的观察方式表达；
4. **观看回报**：至少提供认知反转、误解解除、可观察呈现、新鲜语言、真实共鸣或值得继续讨论的问题之一，不能只做到信息正确。

最终仍须交付与当前媒体任务匹配的完整成品；台词、解说、画面、动作、字幕、声音和互动等具体组合由 D-032 决定。

## 五、最小验收

1. 能用一句话复述本篇唯一商品认知主线；
2. 商品事实、真实取舍、品牌/账号表达和创意共同服务同一主线，而不是拼成几个段落；
3. 去掉真实商品锚点后，如果内容仍能原样套给相邻商品或任何品牌，则 P2 不成立；
4. 改成口播、对白、演示或图文后，主线仍成立，具体结构、镜头数和时长不冻结；
5. 即使事实全部正确，成品仍像内部培训讲义、商品说明书或课堂包装，也不合格。

“换相邻商品”属于 P2-D 黄金任务和实质版本回归的离线性质测试，不是每次日常生成都额外生成一篇对照稿。

## 六、明确不新增

P2-C 不新增：

- 事实层、专业层、创意层、品牌层四段式流程或四个 Agent；
- 价值、代价、不适合、替代、CTA 等固定栏目；
- 来源审计界面、逐句标签、证据数量或用户评分面板；
- 商品角色、设计理由类型等字段/枚举；
- SKU、ERP、在售和完整商品档案硬门；
- 强制品牌口号、品牌故事、显式价值观、替代商品或购买行动；
- 平台、时长、镜头、声音和拍法组合矩阵；
- P2 专属记忆状态机或日常多路反事实生成。

## 七、唯一下一动作

用户于 2026-07-20 正式确认 P2-C 采用“**单一商品认知主线的成品化编译**”：事实、取舍、品牌/账号人格和创意是同一成品中的语义贡献，不是固定段落或用户可见流程。P2-C 据此关闭。

唯一下一动作转为 P2-D：冻结“同一商品有品牌确认原意 / 无品牌确认原意”的两个最小黄金任务，再生成黄金成品和平庸对照，完成后关闭 P2。
