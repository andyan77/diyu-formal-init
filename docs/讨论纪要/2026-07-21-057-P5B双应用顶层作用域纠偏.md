# P5-B 双应用顶层作用域纠偏

- 日期：2026-07-21
- 版本：V0.1-confirmed
- 状态：**用户重申既有 D-022 并修正 P5-B 第 1 小项的作用域歧义**
- 后续状态：**P5-B 已全部关闭；当前进入 P5-C 专属成品合同收敛**

## 一、正式纠偏

> **第一版交互 UI 在逻辑顶层明确提供“内容生产（对外）”与“陈列搭配（对内）”两个应用入口。P1—P5 只属于内容生产；P5-B“用户明确提交本品牌真实商品即可启动、无需 SKU/ERP/库存/完整档案”的最低条件，也只适用于当前顶层选择为内容生产、且主要产品路由为 P5 的任务。顶层选择陈列搭配后，即使使用同一商品和相同造型知识，也不进入 P5，其最低业务输入由独立经营陈列合同判断。**

这不是新增第三层流程，而是把已经确认的 D-022 双应用边界落实到交互作用域。自然对话仍是两个应用内部的共同交互方式。

## 二、最短路由规则

1. **先确定顶层应用**：用户通过 UI 进入内容生产或陈列搭配；当前选择成为默认应用合同。
2. **再在应用内部形成成果**：内容生产在 P1—P5 中路由并生成 `ContentItem`；陈列搭配按培训、概念、执行、巡检或整改等经营陈列任务形成 `DisplayArtifact`。
3. **知识和能力可以共享，成果身份不能共享**：商品、颜色、组合和造型知识不复制；共享能力不把内容成品变成经营方案，也不把经营方案变成对外内容。
4. **明显冲突只提示一次切换**：用户明确要求的最终成果与当前入口冲突时，系统用一句话建议切换，不静默套错合同，也不建设独立意图分类服务、状态机或二次 P5/陈列选择器。

## 三、同一商品的边界例子

同一名门店用户上传同一件没有 SKU 的真实外套：

- 在“内容生产”中要求“做一条让人看见敞开穿和系起穿两种状态的短视频”：主要成果面向外部受众，创造新的穿着想象，进入 P5；可观察特征足以启动。
- 在“陈列搭配”中要求“这件外套放哪具橱窗模特，中岛和挂杆怎样呼应，并给店员执行说明”：主要成果服务内部经营和执行，生成 `DisplayArtifact`；不能因为 P5 允许无 SKU 启动，就认定商品数量、空间、载体或其他执行条件已经充分。

决定应用归属的不是输入里是否出现“搭配、模特、橱窗”等词，而是**最终成果给谁使用、主要解决什么问题**。

## 四、没有提前确认的事项

- 不在 P5 中定义经营陈列的最低商品、空间、库存、载体或数量条件；这些留给 D-005。
- 不冻结具体页面、导航样式、按钮名称或前端技术实现；只冻结逻辑顶层的双应用选择。
- 不新增两套商品数据库、两套搭配知识或重型跨应用意图识别引擎。
