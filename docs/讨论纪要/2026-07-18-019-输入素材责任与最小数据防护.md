# 2026-07-18 输入素材责任与最小数据防护

> **后续纠错（2026-07-18）**：用户指出本文把顾客敏感信息清理、CRM/顾客数据业务和 `RightsRecord` 核心本体错误并列，三者没有因果关系。该纠正成立：D-030 只保留“上传素材中的偶发敏感信息如何处理”；CRM 不属于当前内容生产范围；`RightsRecord` 是否进入核心本体退回 D-016 独立判断。用户同时正式接受 D-003，见 [ADR-012](../架构决策/ADR-012-授权期个人隔离与退出后所属主体接管.md)。当前准确说明见 [2026-07-18-020](2026-07-18-020-顾客敏感信息清理与RightsRecord解耦.md)。本文原联合建议仅保留为纠错过程，不作为现行完整口径。

## 一、用户本轮核心观点

1. 笛语不应把顾客信息、著作权、肖像或声音权属建设成系统内部的业务领域。
2. 所有进入系统的资料，先由品牌企业、区域代理或加盟店自行筛选；相应经营主体对其输入负责。
3. 系统对可能出现的具体顾客信息自动做匿名处理，并向使用者提供风险提示和责任声明。
4. 若系统继续为每份素材建设权属、授权和顾客信息治理链路，会显著增加数据工程与业务操作复杂度，不符合第一版目标。

## 二、方向成立，但“问题不存在”必须改成准确产品口径

本轮应接受的产品边界是：

> **笛语不管理顾客档案，不裁定素材法律权属，也不建设逐素材权利证明系统；系统只接收各经营主体已筛选并声明可用于业务的资料，提供最小来源留痕和敏感信息清理。输入与最终使用责任仍由对应经营主体承担。**

不能把它写成“客观上不存在相关问题”。原因不是要把笛语做成监管系统，而是三个前提都不是绝对保证：

- 经营主体筛选是一项上游责任和输入声明，不是系统已经证明所有权利完整；
- 自动处理只有达到无法识别且不能复原时才可称为匿名化；一般遮挡、替换和脱敏可能仍只是去标识化；
- 风险提示和责任声明可以明确分工，但不能替代系统提供方对自身接收、暂存、清理、权限和删除过程的必要责任。

全媒体自动匿名也不能被当作轻量替代方案。对任意图片、视频、音频和文档完成识别、跟踪、遮挡、转写与效果验证，本身就是一个独立的多模态工程，还会误伤有意保留的模特、员工和生活场景。正确做法是**取消重型权利工程，只对明确支持的类型提供很薄的输入卫生边界**，而不是在产品文档中宣称风险不存在或承诺机器兜底一切输入。

## 三、曾被错误并列的独立问题

1. 当前内容生产不需要顾客主数据、顾客画像或 CRM，但这只是当前业务范围，与素材敏感信息清理没有因果关系。
2. 是否建设逐素材使用范围对象、是否要求权利材料或是否把 `RightsRecord` 纳入核心本体，是 D-016 的独立建模问题。
3. D-030 只处理素材里偶然出现的明显敏感信息，并不承诺机器能够识别一切个人信息或对全媒体自动匿名。

租户隔离、品牌适用范围、来源追溯和公共素材激活仍是系统正常运行边界，但不能再用 D-030 顺带裁决 `RightsRecord`。

## 四、第一版只做一层最小防护

### 1. 经营主体级输入责任

品牌企业、区域代理和独立加盟店分别确认一份版本化的《素材输入与使用责任规则》。账号在相应主体授权下上传，素材自动继承该主体及规则版本，不再逐文件要求权属表单或审批。

### 2. 支持范围内的敏感信息检查与清理

原始上传继续保存在账号本人的隔离素材箱。系统自动移除定位等不需要的文件元数据；对首期明确支持的文字、截图和媒体类型，检查手机号、地址、订单号等明显标识，并在支持时生成单独的清理派生版。上传者应预览确认检查结果。原件与派生版必须分开，不能用清理版冒充原始素材。

若系统已经识别到明显敏感信息、但当前类型无法可靠清理，则该原件仍可由本人查看、导出或删除，但在修正前不得进入模型检索或生成。未被系统识别不等于已经证明不存在风险；最终输入筛选仍由对应经营主体和上传者负责。

若品牌内容有意保留模特、员工或其他可识别真人，系统无法同时把此人“匿名化”并保留其可识别表达。此类素材只能依赖上传经营主体的事前筛选与责任声明，笛语不核验其内部授权材料。

### 3. 极简数据字段

每份素材只需保留：来源经营主体、上传账号、上传时间、输入规则版本、检查/清理状态、原件与派生版关系、素材编号/哈希以及适用品牌和工作区。它们服务于来源追溯、隔离和质量控制，不构成权属登记。

### 4. 产品用语

界面使用“支持范围内的敏感信息检查与清理”，不宣传“全媒体自动匿名”“百分之百自动匿名”或“系统已经确认合法”。提示语只说明输入与最终使用责任，不能声称系统提供方因此没有任何自身义务。

## 五、与 D-003 没有因果关系

D-003 处理账号生命周期和平台访问控制，并不由顾客信息清理推出。用户随后独立接受以下状态：

`授权期个人隔离 → 固定退出窗口 → 所属经营主体接管区`

期满未清理的素材，自动转入账号实际所属经营主体在平台内控制的接管区：总部员工对应品牌企业，区域人员对应区域代理，加盟店员工对应独立加盟店。该主体的授权人员可以查看、下载、归档、删除或在具体任务中人工选用，不再建设额外的交接审批和权属迁移流程；品牌总部仍不能越级取得区域代理或独立加盟店的素材。

接管区不是组织官方素材区。系统不记录“法律所有权已经转移”这一结论，只记录平台内素材控制主体和状态变化；接管素材默认不参加自动检索，不自动进入组织官方素材、区域/公司公共素材、正式知识或训练数据。授权人员人工选用接管素材不需要再走逐文件权属审批，但不改变上述自动化边界。

## 六、原联合建议已撤回

> **[已撤回的联合写法]** 笛语第一版不管理顾客数据，不裁定著作权、肖像或声音权属，也不建设逐素材授权证明系统……

该句把三个独立问题捆绑，已由本轮用户纠正。D-003 已独立关闭；D-030 与 D-016 分别继续讨论。

## 七、依据说明

现行个人信息规则明确区分匿名化与去标识化；匿名化须达到无法识别且不能复原，接收、存储、加工和删除本身都属于处理活动。著作权规则也区分文件/原件控制与著作权归属。因此，本纪要采用“不建设权属裁判系统、只做最小输入卫生”的产品边界，而不采用“风险已经不存在”的绝对承诺。

- [中华人民共和国个人信息保护法（工业和信息化部）](https://www.miit.gov.cn/jgsj/zfs/fl/art/2022/art_515a4b20c12f430eab54bb4f56d89f56.html)
- [中华人民共和国著作权法（国家版权局）](https://www.ncac.gov.cn/xxfb/flfg/flfg_532/202103/t20210309_50530.html)
