## 一、里程碑唯一结果

在不重写现有后端业务闭环的前提下，把笛语从“可验证的服务端合成演示页”升级为“可供第一阶段共享 SaaS 部署的正式响应式产品工作壳”。

完成后，合成演示用户能够在同一套 React + TypeScript 界面中：

1. 从中立入口选择“内容生产（对外）”或“陈列搭配（对内）”，始终看见服务器可信解析的当前身份；
2. 在内容侧自然输入，生成既有 P1—P5 与抖音/小红书/视频号视频或图文文字成品，继续修改并读取旧版本；
3. 在陈列侧提交自然库存清单，得到 DM01 双层挂杆方案，反馈后只形成受影响区域的新版本；
4. 在桌面和移动端阅读完整成品/方案、继续修改、复制或导出；移动端使用“对话/成品（方案）”两段切换；
5. 使用“继续最近/我的内容/历史方案”，并在合法范围内查看版本；
6. 由授权管理者完成一条渐进入驻路径：确认品牌表达草案，规划至少一个发布账号及表达身份，使相应内容入口达到可用；
7. 创建和维护一份轻量连续系列，允许跳集、插集、改序，并按用户明确要求承接前情；
8. 将用户主动选择的文字、图片或视频保存到本人私人素材；授权维护人可以录入本组织素材。两类入口、权限和复用范围严格分开；
9. 图片和视频保持原件，只作为内容创作参考；系统不编辑、打码、匿名化或生成媒体文件。

达到以上结果即停止。M5-3 不部署 ECS、不实现真实生产认证、不接 ERP/POS、不连接平台发布、不建设公共素材、多人共创、实时协作、媒体处理、自动年龄识别、权利审批、日历排程、计费系统、向量库、DIFY、微服务或独立设计系统。

## 四、正式前端的最小技术落点

### 1. 一套工作壳

采用一套 React + TypeScript + Vite 前端，继续由现有 FastAPI 提供 `/api/v1` 和生产构建后的静态入口。允许开发期使用 Vite 开发服务器；不引入 Next.js、微前端、SSR、BFF、Electron 或移动原生框架。

最低依赖：

- React、TypeScript、Vite、React Router；
- 服务器数据可使用 TanStack Query；
- 普通布局和局部状态优先组件状态；没有已发生的跨模块状态痛点时不引入 Zustand；
- 直接使用 CSS variables/普通 CSS 消费正式 VI；不引入 Tailwind、shadcn 或独立 Design System 工程，除非能证明它比现有最小组件更少；
- OpenAPI 是服务合同真源。可生成类型或保持等义的薄类型客户端，但不得为未来页面预建 19 组空端点。

正式前端直接引用 `assets/brand/diyu-vi/` 的 SVG、PNG 和 favicon，不用 CSS、字符或近似图形重画 Logo。

### 2. 迁移而不是双轨

- 现有服务端页面只在开发期间作为回退和合同探针；
- M5-3 进入 `REVIEW` 前，默认产品入口必须使用新工作壳；
- 不长期维护两套业务工作台；
- FastAPI API、服务、仓储、RLS 和业务编译器保持真源，前端不得复制 P1—P5、DM01 或事实判断。

### 3. 自适应工作面

桌面：

- 当前应用、身份摘要、最近工作和自然输入保持清楚；
- 生成前以对话/输入为主；
- 生成后成品或陈列方案成为主视觉，对话退为协作面；
- 当前平台、媒体、系列、商品和制作条件以少量人话标签或展开面板呈现，不把内部上下文对象暴露给用户。

移动：

- 只保留“对话/成品（方案）”两段切换；
- 顶部始终可见当前账号或执行组织摘要，点开才看完整身份；
- “继续改”和“复制/导出”可达；
- 不自建触屏选字浮层，不把桌面三栏硬塞进窄屏。

V1.1 的 16 态是交互覆盖参考，不等于 16 个路由、16 张独立页面或 16 套测试。

## 五、复用现有两条业务闭环

### 1. 内容生产

新前端必须直接调用现有可信 API，保持：

- 普通交流不创建任务；
- 自然输入形成唯一 P1—P5；
- 抖音视频、小红书视频、小红书图文、微信视频号视频由服务器可信账号映射；
- 同目标修改形成 V2/V3，旧版本可读；
- 跨目标重编译形成独立目标任务并保留源版本；
- 用户正文只展示自然导读和完整成品；
- 页面不接收或允许切换 `tenant_id`、`brand_id`、`organization_id`、`account_id`、`ContentRole` 或 `user_id`。

用户动作按 V1.1：

- 常用：继续改、复制、导出；
- 查看非当前版本时，确有直接消费者才提供“设为当前版本”；
- “存为素材”和“纳入系列”保持独立；
- 不恢复全局“采用此版”按钮，不为了留痕要求用户额外确认。

### 2. 陈列搭配

保持现有 DM01 确定性路径：

- 自然库存 → 20 件识别 → 15 件上墙 V1；
- 反馈右侧 C 组过厚 → 仅该组变化 → 14 件 V2；
- V1 可读，错误或无关反馈不产生半版本；
- 前端不调用模型生成 layout，不把对外账号人设带入对内陈列。

## 六、渐进入驻、账号与系列

### 1. 渐进入驻

只建设能够使一个品牌开始当前业务的最小路径：

1. 选择最接近的行业访谈原型；
2. 通过少量具体对照和自然对话生成一份人话版品牌表达草案；
3. 授权管理者修改并确认一个 `BrandStrategyBaseline` 版本；
4. 按真实组织拓扑创建必要组织，不强制总部—区域—门店三级；
5. 创建至少一个 `PublishingAccount`，为其关联一个独立 `ContentRole`/表达身份和合法操作授权；
6. 清楚显示当前仍缺什么、只影响哪项能力；
7. 达到当前内容任务最低条件后允许进入生产，不要求一次填完完整知识目录。

首期不建设组织架构设计器、通用流程审批、逐级审核、完成度百分比、自动排期或完整企业 IAM 后台。

### 2. 账号与表达身份

前端人话化显示但后端语义继续分离：

- 实际登录用户；
- 代表组织；
- 发布账号；
- 表达身份/`ContentRole`；
- 对内陈列执行组织与岗位身份。

“同属一个租户”不能让账号或角色跨品牌、跨组织自动可用。经营关系不是权限包。

### 3. 连续系列

使用现有 `ContentSeries` 语义形成一张轻量系列卡：

- 系列定位、账号、稳定设定、主题/叙事大纲、分集计划、当前进度和未完线索；
- 用户可跳集、插集、改序、重设；
- 生成、浏览、导出、设为当前版本均不自动更新系列；
- 用户明确“纳入系列”“下一集接着这条”或自然指定历史内容时立即按原合法范围承接；
- 不建设自治编剧 Agent、日历排期引擎或重型工作流状态机。

## 七、私人和组织素材生命周期

### 1. 一套底座、两个入口

复用 `MaterialAsset` 语义，物理实现只建立当前直接消费的最小表和对象存储适配器：

- 私人素材：控制者为当前自然人，授权有效期内只有本人可看、改、删和选入任务；
- 组织素材：控制者为一个明确组织，只有授权维护人可录入/修改；默认仅本组织可用，明确配置后才允许相应区域或门店使用；
- 总部、区域、门店有效自然人均可以拥有本人私人素材；
- 管理员身份不自动取得私人素材读取权或组织素材维护权；
- “组织官方”不自动升级为品牌知识、商品事实或训练数据。

### 2. 对象存储

- 图片和视频原件不得以应用容器或 ECS 本地磁盘作为长期真源；
- 实现一个最小 S3 兼容对象存储端口。开发/CI 可以使用同合同的本地测试实现或单个 MinIO，不建设存储平台；
- M5-4 再绑定实际生产对象存储参数，不在 M5-3 猜测生产桶、区域和密钥；
- 对象键、读取和删除必须由服务器可信租户/品牌/组织/私人范围派生，客户端不得指定任意对象路径；
- 数据库只保存必要元数据、作用域、类型、大小、校验摘要和版本引用；失败不得产生只有数据库或只有对象的半成品。

### 3. 首期媒体边界

- 支持文字、图片、视频原件作为创作参考；
- 系统不裁剪、转码、识图、生成缩略图、打码、匿名化、评分或审批；
- 用户上传前必须声明素材不属于“可识别真人未成年人原件”；已知或声明为该类时，在持久化和外部模型传输前拒绝；
- 不建设年龄识别、监护人授权、儿童库、人物权利图谱或媒体审批；
- 生成文字继续按既有规则使用非识别性顾客称谓；
- 素材未被用户明确选择时，不自动进入模型上下文。

### 4. 退出与删除

- 授权有效期间，私人素材定义为本人独占入口；
- 账号停用后，按 ADR-012 转入所属企业/经营主体可处置状态，不再供离职账号继续生成；不建立额外个人退出门户；
- 用户主动删除不可恢复原件时使用一次明确确认，数据库与对象删除失败必须可见并可重试；
- 不为普通编辑建立审批或恢复工作流。

## 八、最小 API 与数据扩展

只按上述可见能力增加：

- 当前可信身份/应用上下文只读端点；
- 当前用户的任务/成品/版本最小列表和读取；

- 私人/组织素材元数据、上传授权、读取和删除；
- 仅当前对象范围的基础 SQL 搜索。

不得一次性补齐 V1.1 里所有概念 API，不建设 GraphQL、通用工作流、通用策略引擎、通用搜索平台或事件总线。

- 请求角色不可绕过；

### 3. 渐进入驻与连续性

- 一个新的合成品牌管理会话从品牌草案进入至少一个合法发布账号/表达身份，并解锁当前内容入口；
- 非三级组织同样可以完成，不虚构区域或门店；
- 一份成品被明确纳入系列后可承接；未纳入的普通成品不会在无关任务中自动复用；
- “下一集接着这条”、跳集和改序不要求补齐前一集。

### 4. 素材和隔离

- 总部、区域或门店自然人各自私人素材只有本人可读；
- 组织素材只有授权维护人可写，门店组织素材不向总部穿透；
- 兄弟品牌、另一组织、另一用户和另一租户的诱饵元数据与对象均不能读取、搜索、删除或送入模型；
- 已知/声明可识别真人未成年人原件在持久化前被拒绝；
- 素材保存失败不产生半记录，删除失败清楚可恢复。

### 5. 质量闸门

开发期只运行受影响定向检查。候选稳定后一次执行：

- 前端 lint/typecheck/build；
- 现有 `make lint`、`make typecheck`、`make golden`、`make test`；
- 上述两条浏览器闭环和最小入驻/素材隔离检查；
- 一次远程 CI。

真实 DeepSeek 只需通过新正式内容页面完成一条受影响内容生成/修改冒烟；M5-1/M5-2 已证明的六卡和五份媒体成品不整批重跑。DM01 不调用 DeepSeek。

不得增加多浏览器矩阵、逐像素视觉回归、16 态逐页 E2E、全设备截图、覆盖率门槛、性能平台或第三轮审查。

## 十、执行自主性与停止线

执行端在本包范围内拥有完整普通实现授权：

- 自主选择目录、组件拆分、CSS 实现、最小依赖、迁移、测试夹具和错误修复；
- 不因 npm 包、浏览器依赖、端口、路径、类型、测试或普通网络问题停工询问；
- 先交付一个真实 UI 纵向闭环，再扩入驻、系列和素材；
- 发现普通缺口时在同一轮修复、复测并继续；
- 只有产品语义改变、范围实质扩大、需要生产基础设施写入、真实客户数据、无法保留用户修改，或只能降低租户隔离/数据安全/既定验收时，才停下请求裁决。

达到验收后立即：

1. 更新 `README.md`、`docs/项目记忆.md`、路线图、OpenAPI 和 `MILESTONE.md`；
2. 提交并推送当前完整实现；
3. 等待同 SHA 远程 CI；
4. 将里程碑置为 `REVIEW`；
5. 报告用户现在能做什么、未证明什么和唯一下一动作；
6. 停止，不启动 M5-4，不追加未来能力。

## 十一、不得冒充的结论

M5-3 通过后只证明：

- 当前合成数据和合成身份下，正式响应式工作壳可完成约定闭环；
- 渐进入驻、系列和素材的首期软件合同成立；
- 本地/CI 对象存储适配和作用域隔离成立。

它不证明：

- ECS、`diyuai.cc` 或生产认证已经上线；
- 真实企业已经完成入驻或采用；
- 真实门店、平台流量、内容竞争力或销售结果；
- 多品牌真实运行；
- 媒体生成、识图、审批、公共素材或 ERP 已实现。

唯一下一动作：主控终审 M5-3；通过后才进入 M5-4 共享 SaaS 部署。
