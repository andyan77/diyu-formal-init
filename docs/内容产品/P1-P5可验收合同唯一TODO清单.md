## 一、使用规则

1. 严格按 **P1 → P2 → P3 → P4 → P5** 串行推进；任何时刻只激活一个待办。
2. 每类产品只收口四件事：**成品承诺、输入与路由边界、专属交付与创作边界、最小黄金验证**。
3. 用户明确确认后才勾选；专家建议、现有草稿和黄金任务候选均不算已经完成。
4. 每类先用一个最具代表性的纵向场景验证；只有确有另一种关键业务边界无法覆盖时才增加夹具，不预设题数、反例数、评分阈值或永久权重。
5. 共用运行语义直接继承 ADR-014，不在五类合同里重新讨论或复制。
6. 平台、自然时长和具体媒体成品组成已经由 D-032/ADR-015 冻结；对象、字段、表、JSON 和投影仍留给 D-016。本清单不安装 SDK、不写代码、不设计物理数据库。
7. 某类达到完成条件后立即停止扩展并进入下一类；五类未全部关闭前，不机械拆出五套重复规范或五套 Agent 流程。
