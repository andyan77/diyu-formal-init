# 新租户第一阶段资料准备与渐进就绪模板

- 版本：V1.0-first-phase-current
- 用途：新租户第一次进入笛语时，与系统一起完成最低资料准备并开始第一项实际生产
- 使用者：品牌负责人、内容负责人、集合店/买手店经营者、区域或门店负责人
- 使用原则：**这是一份可以边聊边完成的工作本，不是要求企业一次填完的调查表。**
- 完整参考：[“折线之间”第一阶段全链路演示资料包](折线之间演示品牌运行档案.md)

---

## 先看这一页：你不需要先把公司资料整理完

第一次进入笛语，只需要完成四件事：

1. 说清楚当前是谁在使用、代表哪个真实组织；
2. 如果先做对外内容，和系统一起确认一版“我们希望怎样和受众相处”的人话说明；如果只做对内陈列，这一步不要求先建立对外人格；
3. 激活一个真实的内容账号，或者一个真实的陈列执行门店；
4. 选择第一件想做的事，并补齐这件事不可缺少的少量信息。

完成这四件事，就可以先开始已经具备条件的内容或陈列任务。其他商品、账号、门店、系列和素材，以后用到时再补。

你可以任选一种方式开始，也可以混合使用：

- [ ] 上传已有的品牌手册、商品册、账号资料或门店资料，让系统先起草；
- [ ] 像聊天一样自然介绍，系统边听边整理；
- [ ] 直接在本模板中勾选和补充少量内容；
- [ ] 我暂时说不清楚，希望系统先给一版参考草案。

任何问题都可以回答“**不知道**”“**暂时没有**”或“**不适用**”。系统不能为了填满资料而替你编造事实。

## 一、这些能力由首期产品提供，企业无需重复整理

本节表达的是首期产品合同，不表示当前文档已经证明软件实现完成。企业无需为了入驻，自行建设下面这些系统能力或整理一份相同资料。

以下内容不应变成企业入驻作业：

- 服装行业通用的面料、版型、色彩、穿衣判断、造型和基础陈列知识；
- 内容选题、结构、口播、对白、解说、画面、动作、字幕、声音和制作提示等生产方法；
- P1—P5 五类内容产品的内部判断与编排能力；
- 抖音、小红书和微信视频号的首期媒体适配能力；
- 服装行业常见的账号与角色参考原型；
- 公开地域背景的按需搜索能力；
- 内容事实边界、避免编造、租户隔离和最小留痕等系统规则。

行业原型和“折线之间”演示品牌只帮助系统起草和举例，不会直接复制成你的品牌人格、账号人设、商品故事或内容计划。

## 二、第一次激活，只完成四件事

### 第 1 件：确认“我们是谁”

用一句话介绍当前经营主体：

> 我们是：____________________________________________________________

例如：“一家经营三个自有女装品牌的公司”“一家独立设计师买手店”“某品牌在南城的独立加盟店”。

当前最接近哪种实际情况？可以多选，不确定时直接写真实情况。

- [ ] 一个企业经营一个自有品牌
- [ ] 一个企业用同一套组织经营多个品牌
- [ ] 集团下不同品牌各有相对完整的组织
- [ ] 集合店、设计师买手店或多品牌零售店
- [ ] 区域代理、加盟店或品牌合作网络中的独立经营主体
- [ ] 单店独立经营，没有总部、区域和加盟层级
- [ ] 其他：__________________________________________________________

真实存在的组织关系是：

> ____________________________________________________________________

不知道怎么画组织图没有关系。请直接说“谁经营谁、谁和谁合作、哪些店是独立经营”。系统只建立真实存在的组织，不会为了套用三级渠道模板补造总部、区域或门店。

可以上传或提供：

- [ ] 企业/店铺简介
- [ ] 品牌或店铺手册
- [ ] 实际组织和门店清单
- [ ] 已有账号清单
- [ ] 暂时都没有，直接通过对话完成

### 第 2 件：做对外内容时，确认“我们怎样和受众相处”

如果当前只启用对内陈列，可跳过本节。对内陈列复用已有品牌规则、陈列标准和商品知识，不要求先建立对外 IP 人设；以后启用内容生产时再回来完成。

不需要填写“品牌哲学”“精神内核”这类抽象词。请先做几个轻判断，系统会据此起草一份人话版表达基线。

我们最想长期服务的人，大致是：

> ____________________________________________________________________

他们在穿衣、生活或关系中反复遇到的真实矛盾是：

> ____________________________________________________________________

我们更接近下面哪种关系姿态？

- [ ] 给出专业判断，但不居高临下
- [ ] 像熟悉生活的朋友一样陪伴和回应
- [ ] 用审美和视觉打开新的穿着想象
- [ ] 直接、务实，帮人减少选择和行动成本
- [ ] 以上可以组合
- [ ] 都不准确，请系统先给两个对照版本

再用四道具体对照帮助系统理解取舍；每题都可以选“两者都不对”。对照中可以出现用于识别边界的反例，选中反例不表示系统必须照办；若与已确认事实、平台红线或“不利用身材、年龄和身份焦虑”等产品边界冲突，系统应直接指出并给出可行替代。

1. 商品存在真实限制时：[ ] 主动讲清取舍　[ ] 除非顾客问，否则尽量不提　[ ] 其他：__________
2. 涉及年龄、身材或身份时：[ ] 用焦虑推动行动（反例，不进入正式基线）　[ ] 不利用焦虑，但仍给具体判断　[ ] 其他：__________
3. 一段生活还没有答案时：[ ] 尽量收成积极结论　[ ] 允许矛盾和未完成存在　[ ] 其他：__________
4. 一句话与商品没有自然关系时：[ ] 仍应带到商品　[ ] 先把内容做好，不强接商品　[ ] 其他：__________

我们希望内容经常观察什么？

- [ ] 真实穿衣问题与选择犹豫
- [ ] 商品的真实取舍，而不是卖点堆砌
- [ ] 普通人的工作、家庭、情绪和一地鸡毛
- [ ] 本地街区、门店关系和日常服务
- [ ] 商品组合、造型和视觉审美
- [ ] 其他：__________________________________________________________

我们明确不想成为：

> ____________________________________________________________________

可以写“像上课”“满嘴卖货”“永远正确”“假装励志”“高高在上”，也可以上传一条你认为“像我们”和一条“绝对不像我们”的既有内容。

- 像我们的参考：____________________________________________________
- 不像我们的参考：__________________________________________________

系统接下来应交付一份自然语言草案，企业只需：

- [ ] 基本准确，可以确认
- [ ] 大方向准确，我想改几句话
- [ ] 不准确，请根据我的纠正重写

未经确认的系统草案只能用于继续讨论和预览，不能自动成为正式品牌、企业或店铺人格。

### 第 3 件：激活一个真实工作入口

先选择本次准备启用什么：

- [ ] 对外内容生产
- [ ] 对内陈列搭配
- [ ] 两者都要，但先完成其中一个

如果先做对外内容，只需确认第一个真实账号：

- 账号规范名称或当前展示名：________________________________________
- 平台：[ ] 抖音　[ ] 小红书　[ ] 微信视频号　[ ] 暂时未定，仅先形成账号/角色草案
- 该账号已经由当前组织实际创建或确认控制权：[ ] 是　[ ] 否，仅作策划草案
- 这个账号代表谁：__________________________________________________
- 它从什么真实位置观察和发声：______________________________________
- 谁实际使用这个账号完成工作：______________________________________
- 企业确认该使用者可以代表上述组织操作该账号：[ ] 是　[ ] 尚未确认
- 它不能冒充什么身份：________________________________________________

平台暂时未定、账号尚未实际创建或当前组织尚未确认控制权时，可以继续做账号策划和内容讨论，但不能把该草案称为已激活的发布账号；准备交付平台成品前，至少明确一个由当前组织控制的实际平台账号及其操作授权。

如果先做对内陈列，只需确认：

- 实际执行组织/门店：________________________________________________
- 谁使用系统并执行方案：____________________________________________
- 首次准备处理的陈列区域：__________________________________________

对内陈列使用真实组织身份，不套用对外 IP 人设。账号运营者、发布账号和内容角色也不是同一个概念：实际员工可以运营账号，但不能因此随意更换账号代表的组织和人设。

### 第 4 件：选第一件想做的事

- [ ] 帮受众解决一个穿衣或购买选择问题
- [ ] 讲清一件商品真实解决什么、牺牲什么、适合谁
- [ ] 从一句生活、工作或情绪种子发展账号内容
- [ ] 做一篇与当地人、街区或门店关系有关的内容
- [ ] 用真实商品做一份以视觉和造型为主要价值的内容
- [ ] 策划一个可以连续推进的内容系列
- [ ] 给门店一组双层挂杆形成可照做的文字陈列方案
- [ ] 我不知道先做什么，请系统根据已有资料推荐一项

只要所选事项的最低条件已经满足，就先生产这一项；其他能力不同时开通，也不会阻塞当前任务。

## 三、按能力首次补齐：只补当前要用的资料

下面不是一张必须全部填写的清单。只看你刚才选择的那一行。

| 想开始的事情 | 第一次额外需要什么 | 不需要先准备什么 |
|---|---|---|
| 帮助选择 | 一句模糊输入、一个目标人群处境或一件商品，三者有其一即可；具体谈商品时再补相应事实 | 不要求用户先提出标准问题 |
| 解释商品 | 至少一件真实商品，以及本篇真正要用到的成分、结构、工艺、特点或已知取舍 | 不要求完整商品目录，也不能用系统猜测补开发理由 |
| 人格与生活叙事 | 当前账号角色，加一句生活、工作或情绪种子 | 不要求必须出现商品，也不要求所有内容都有故事情节 |
| 在地关系内容 | 账号与当地的真实位置；当次门店事实有就提供，公开地域背景可由系统按需搜索 | 不要求填写地域文化大表，也不要求先有真实评论 |
| 造型与视觉内容 | 至少一件真实商品、足以避免实物失真的特征，以及本次一人/两人/团队等制作条件 | 不要求昂贵场景、专业团队或固定时长 |
| 连续系列 | 系列大致方向、已经确认的前情，以及本次想推进到哪里 | 不要求固定三十天排期；可以跳集、插集和重排 |
| 墙面挂杆陈列 | 首次记录这组挂杆的上下结构、容量、主要视线带、来客方向和现场硬限制 | 不要求 ERP、识图巡检、评分或整改流程 |

墙面挂杆资料首次确认后，日常任务只需提交当前实际可用商品和数量。系统只根据本次清单给出上下/左右搭配区、主次正挂、侧挂节奏、数量间距、替代办法和执行步骤；信息不足时只收窄方案或追问一次，不猜库存，不替 ERP 判断真值。

## 四、每次任务才提供的信息，不要提前塞进入驻资料

以下内容容易变化，通常只在本次任务中自然说明：

- 今天想到的一句话、生活琐事、情绪或临时选题；
- 当前想用的商品、人工库存或活动信息；
- 一条评论、门店观察或当次本地信号；
- 本次平台、时长倾向、视频/图文形式；
- 一人手机拍、两人配合还是团队制作；
- 当前场地、预算、时间和不能完成的动作；
- 连续系列本次要跳到哪一集、插入什么或引用哪段前情。

可以直接这样说：

> “今天喝了常喝的蓝山，居然是甜的，帮南城店账号做一条。”

> “这件双面外套想拍得简单一点，一个人对着手机讲。”

> “这是今天能上墙的商品和数量，按我们店那组双层挂杆重新排。”

系统应从自然语言中理解任务并主动完成，不要求用户填写产品编号或后台字段。只有缺少一项不可替代信息、且不同答案会实质改变结果时，才追问一次。

这些当次信息默认不进入长期资料。只有使用者主动保存素材、确认系列前情或有权限者确认组织资料后，才按相应范围跨任务复用。

## 五、已有资料怎样交给系统

企业不需要重做已有资料，也不需要按笛语字段重新命名文件。可以直接提供：

- [ ] 品牌手册、企业介绍、招商册或培训资料
- [ ] 商品册、吊牌信息、款号表或设计/选品说明
- [ ] 已有账号主页、代表性内容或过去的内容计划
- [ ] 门店清单、合作关系或组织说明
- [ ] 门店空间和普通商品图片/视频，可作为素材保存或参考
- [ ] 现有陈列标准或门店执行说明
- [ ] 其他：__________________________________________________________

系统读取可理解的文字资料后，应先给出一份“我理解到的内容、仍然不知道的内容和建议草案”，让企业用自然语言纠正。图片/视频能否被当前实现直接理解，以实际启用的输入能力为准；不能直接理解时，用户可以补一句自然说明，媒体理解不得成为入驻就绪硬门。上传资料不是自动确认；系统推断也不能升级成企业事实。

### 素材放在哪里

- **我的素材**：本人主动保存，仅本人使用；总部、区域、门店管理员和同事均不能查看。
- **本组织素材**：由被授权的素材维护人从组织入口直接录入；默认仅本组织使用，存在真实下行网络时才能明确向下提供。
- 私人素材不能一键转成组织素材；组织素材也不因为“官方”二字自动成为品牌事实或正式知识。
- 门店组织素材默认不向总部、区域和其他门店开放。

### 图片和视频的第一阶段边界

- 笛语第一阶段交付的是文字型成品，包括内容概要、正文、台词/口播/对白/解说、画面、动作、字幕、声音和必要制作提示；不生成或编辑成片、图片和视频文件。
- 获准进入系统的普通图片/视频只能按原有私人或组织作用域参与创作：私人素材仍只供对应自然人本人选择，组织素材按其明确范围调用；系统不负责自动打码、换脸、变声或视频清理。
- **任何临时附件、私人素材或组织素材均不得包含能够识别或合理关联到真人未成年人的照片、视频、声音及可还原派生内容；总部已经在外部取得授权也不作为第一阶段例外。**
- 亲子题材仍可使用童装商品资料、成人叙述、非人物画面和不指向具体儿童的生活种子生产；不得编造真实儿童案例、真实试穿结果或真实顾客反馈。

## 六、不同经营形态，只走与自己有关的分支

### A. 一个企业经营一个自有品牌

优先确认品牌表达基线、首个账号角色和第一项内容能力。只有真实存在区域、代理和加盟网络时才补相应关系；没有就跳过。

### B. 一个企业用同一套组织经营多个品牌

组织和员工无需按品牌复制。企业可以先激活一个品牌；每个需要独立发声的品牌分别确认自己的表达基线。只有商品品牌码、没有独立表达基线的品牌，只用于商品识别和过滤，不能由系统凭空创造品牌人格。

### C. 集合店、设计师买手店或多品牌零售店

不要求拥有自有服装品牌，也不要求虚构总部、区域和加盟体系。先确认店铺/买手自身的表达基线，以本店身份谈选品和商品。上游品牌只作为明确引用，销售或选品关系不代表取得上游品牌人格、设计原意、内部知识或官方陈列规则。

### D. 区域代理或独立加盟店参与品牌合作网络

按真实经营身份建立工作入口，不把代理或加盟店当成总部员工。内容账号只能使用自己的合法角色位置；共享品牌知识不等于取得总部人设。门店私人素材仍仅本人可见，门店组织素材默认仅本店使用。

### E. 其他形态

请直接说明真实业务：

> ____________________________________________________________________

系统先按真实组织、品牌、账号和经营关系形成最小方案，不要求企业先选择一个固定业态标签。

## 七、怎样判断“资料已经可以开始生产”

不计算完整度百分比，只回答三个问题：

- [ ] 已确认当前真实经营主体及使用者代表谁；
- [ ] 已确认一条真实工作路径：内容生产已有人话版表达基线，并有一个由当前组织实际控制、平台明确、代表组织明确、内容角色明确且操作人获授权的发布账号；或陈列搭配已有实际执行组织以及当前可用的品牌/陈列规则，不要求对外人格；
- [ ] 当前选择的一类任务已经具备本模板第三节所列最低条件。

三项都成立，结论就是：

> **该租户的这一项能力资料已就绪，可以开始相应生产。**

这不表示其他 P1—P5、连续系列和陈列能力同时就绪。缺少商品只限制商品相关任务；缺少地域资料只限制相应在地内容；缺少挂杆资料只限制该陈列任务。任何一个租户或任务的资料缺口都不能阻塞系统整体、其他租户或已经成立的能力。

还要区分：本模板只能判断“资料是否足以支持某项任务”，不能证明真实账号、权限、检索、模型调用、保存、留痕和删除等生产软件已经完成，也不能证明企业采用、平台流量或经营效果。

## 八、第一阶段明确不要求企业准备

- 不要求一次建成全量品牌数据库或完整企业数字镜像；
- 不要求一次录入全部商品、门店、区域、账号和员工；
- 不要求所有租户具备总部—区域—门店三级渠道；
- 不要求固定账号数量、固定内容比例或固定三十天排期；
- 不要求把组织型 IP 参考母版或“折线之间”照抄成自己的方案；
- 不要求 ERP/POS 接入，也不要求系统判断实时库存；
- 不要求 CRM、顾客库、会员画像或批量顾客信息；
- 不要求权利图谱、逐素材合同表或复杂素材审批流；
- 不要求图片/视频生成、自动清理、识图巡检、陈列评分或整改闭环；
- 不要求为了以后可能使用的能力预建空组织、空账号、空字段、空流程或空接口。

企业只需要为当前真实要做的事情提供不可替代的事实，并确认系统起草得是否准确。

## 九、系统完成首次入驻后，应交付什么

企业最终看到的应当是自然语言成果，而不是后台字段：

1. 内容生产路径交付一页“我们是谁、为谁服务、怎样表达、明确拒绝什么”的确认稿；纯对内陈列路径只交付当前执行组织、可用规则与商品依据的简明确认；
2. 当前真实组织和首个工作入口的简明说明；
3. 内容路径交付首个账号的人设与冷启动建议；陈列路径交付首个陈列执行入口说明；
4. 当前已经可以生产的事项，以及暂时缺什么只会影响哪项能力；
5. 用户选择的第一份完整内容成品，或第一份可直接照做的文字陈列方案。

内容成品先用一小段概要说明“这次为什么这样做、核心表达是什么”，再交付可直接使用的完整正文及按需需要的台词、解说、画面、动作、字幕、声音和制作提示。用户可以继续用自然语言讨论和修改，系统不向用户展示路由、评分、提示词、重试轨迹或后台判断过程。

## 十、内部依据（企业无需填写）

本模板统一落实以下已确认边界，不另建平行规则：

- D-014：系统预填、人话起草、企业修改确认的品牌内容战略基线；
- D-002 / ADR-023：组织型 IP 母版只作参考，三级渠道不是租户入驻前提；
- D-007：合成演示品牌支持系统构建，新租户按任务渐进就绪，真实品牌仍是后续真实试点与企业验收证据；
- P1—P5：一个独立成品执行一个主要内容产品合同，共享能力由系统内部编排；
- D-032：抖音、小红书、微信视频号，以及视频/图文两类文字型完整成品；
- D-005：首期对内陈列只做卖场墙面双层挂杆的轻量文字执行闭环；
- D-018：私人/组织素材双入口和总部、区域、门店按真实组织形成的逻辑视图；
- D-020 / D-030：真人未成年人原件首期禁入；普通媒体只作参考，首期只生产文字型成品。

本模板是第一阶段唯一的新租户资料准备逻辑基线，不等于未来界面只能有一张固定页面。对话、资料上传和分步入口都可以消费同一基线；后续新增能力通过相应任务的版本化条件加法接入，不把本模板扩张成巨型问卷。
