## 迁移目的与边界

迁移用于获得 Linux 原生的权限、大小写、软链接、文件监听、依赖性能和 CI Runner 一致性。它不等于安全隔离，也不证明已经具备 CI/CD。

真正的工程风险隔离继续依赖 Git 分支或独立 worktree、最小文件系统权限、项目级依赖、测试与生产分离、密钥不入库、必要容器和外部操作单独授权。第一版不为迁移新建独立 WSL 发行版、重型本地容器平台或双目录同步系统。

## 无损切换验收

执行迁移时必须：

1. 以执行开始时的实际源目录生成相对路径、文件类型、大小和 SHA-256 清单；历史统计的“203 个文件”只是一时现场事实，不是冻结数量；
2. 先复制、后核验、再切换，禁止直接剪切或删除源目录；
3. 不复制 Codex 工作区注入的空 `.git`、`.agents`、`.codex` 挂载占位；正式根文件 `AGENTS.md` 属于项目内容，必须复制；
4. 在目标目录逐文件核对路径、大小和摘要，并抽查 Markdown、JSONL、YAML、HTML、DOCX 及中文路径可读性；
5. 核验通过后才在目标目录初始化正式 Git 仓库和基线版本；
6. 将 Codex/编辑器工作区统一切换到 WSL 路径后，才允许继续第一条软件闭环；
7. 至少完成一次完整工作循环并证明 Git 或外部备份可恢复后，才讨论归档 C 盘副本；任何删除仍需单独授权。

若核验失败，继续使用未改动的 C 盘原目录；不得人工双向拼接两个版本。
