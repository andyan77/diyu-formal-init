## 三、已冻结架构，不得重新讨论

1. 第一阶段是一套模块化单体，不拆微服务；
2. 第一阶段取消 DIFY，应用核心经极薄适配器直连一个 DeepSeek 模型；
3. 权威关系数据使用当前项目独立 PostgreSQL 数据库、共享 schema；
4. 租户根隔离使用应用显式作用域 + PostgreSQL 强制 RLS；不采用每租户 schema；
5. 一套部署服务所有租户，不为品牌、区域、门店或账号复制代码、数据库或 CI/CD；
6. 平台上线与品牌入驻解耦：M5-4 只需“折线之间”，M6 才创建授权真实租户；
7. 三个入口复用同一个认证核心，但页面、会话受众和服务端权限失败关闭；
8. 同一自然人一人一号，可按需同时拥有租户用户和租户管理员资格；进入哪个入口就只获得哪个入口的会话；
9. 企业发布账号不是自然人登录账号，不允许共享发布账号密码；每位实际操作者使用自己的登录身份并获授权；
10. 每个租户用户最多维护一个本人默认表达人设；企业发布账号的 `ContentRole` 独立管理；
11. 租户用户只能由租户管理员创建；无公开注册；首期无短信、邮件和社交登录；
12. 平台运维使用独立平台身份和 MFA，不属于任何租户，不绕过租户业务边界；
13. 系统只生产文字成品；图片/视频原件只在原作用域作为参考，不做媒体生成、识别、打码或审批；
14. 构建测试不设模型费用闸门；生产限流用于稳定性和防滥用，不建设账单系统。

## 五、正式认证与三个入口

### 1. 一个认证核心，三个会话受众

用现有用户、租户和授权模型做最小扩展，不复制三套用户系统。生产会话必须：

- 使用高熵不可预测令牌，服务端只保存令牌摘要；
- 记录自然人、入口受众、租户（平台运维为空）、创建/到期/撤销时间；
- 登录成功后轮换会话，Cookie 使用 `Secure`、`HttpOnly`、`SameSite=Lax`；
- 所有修改请求执行同源/CSRF 最小保护；
- 用户、租户或入口资格被停用后，现有会话立即失效；
- 不把 `tenant_id`、品牌、组织、账号或入口资格交给客户端声明；
- 不记录密码、激活令牌、重置令牌、TOTP 秘钥或完整 Cookie。

生产模式必须完全禁用当前 `/ui/select/*` 合成入口和固定 HMAC 会话。测试模式可以继续使用合成夹具，但不得通过生产配置意外开启。

生产启动不得再要求 `DIYU_DEMO_*` 固定 UUID、固定演示用户、固定发布账号或硬编码商品/SKU 配置。合成演示数据是生产数据库中的普通受作用域约束记录，测试夹具仍可使用固定 ID；生产身份、账号、角色、商品引用和陈列组织全部从正式会话与数据库授权解析。

### 2. 租户用户入口 `/login`

- 登录后只进入租户用户工作壳；
- 根据服务端授权展示内容生产、陈列搭配或二者；
- 不出现租户管理员和平台运维导航、链接或 API；
- 无当前业务资格时给出自然说明，不自动授予身份；
- 保持当前内容账号与 `ContentRole`、经营陈列组织身份的语义区别。

### 3. 租户管理员入口 `/tenant-admin/login`

- 登录后只进入租户管理工作壳；
- 可创建、停用当前租户自然人用户，授予/撤销用户入口资格、发布账号和必要组织资格；
- 创建用户后由系统生成全平台唯一但不承载权限语义的登录用户名，并生成一次性激活链接；管理员复制后安全交付，用户只需用户名和密码登录，不填写 `tenant_id` 或内部组织 ID；首期不发短信或邮件；
- 激活链接只保存摘要、短期有效、使用一次即失效；用户在链接页自行设置密码；
- 用户可自行修改密码；忘记密码时由租户管理员生成一次性重置链接，不建设客服审批流；
- 租户管理员继续管理企业发布账号及独立 `ContentRole`，但发布账号本身不能登录；
- 租户管理员不能读取其他用户私人素材、提示原文或生成正文，也不能进入平台运维入口。

### 4. 平台运维入口 `/ops/login`

- 平台运维身份与租户用户表分离，首位运维通过一次性服务器引导命令创建；
- 强制密码 + TOTP MFA；不建设短信 MFA、企业 SSO 或复杂 IAM；
- 最小运维台只提供：
  - 创建租户壳及首位租户管理员激活链接；
  - 查看/停用租户；
  - 查看已启用账号数、生成调用量、成功/失败、延迟、供应商用量和当前限流状态等聚合；
  - 查看应用、数据库、对象存储和模型适配器的健康状态；
- 平台运维不得浏览租户成品正文、提示原文、评论、私人素材或组织素材原件；
- 平台运维 API 不得复用迁移/超级用户凭据。跨租户开户和聚合只能通过显式白名单表、窄权限数据库角色或返回固定聚合字段的受控函数实现；不能给运维应用路径授予租户业务表的任意查询权或 `BYPASSRLS`；
- 首位运维的一次性激活材料和 TOTP 引导信息只能写入 ECS 上 root-only、权限 `0600` 的短期文件或等效安全终端，不得进入 Git、应用日志、CI 或普通执行反馈；完成激活后删除。

### 5. 密码与最小防护

- 使用成熟库的 Argon2id 或等强度自适应密码哈希，不自研密码学；
- 设置合理密码最低长度，允许密码管理器，不建立复杂字符谜题；
- 登录和激活/重置端点执行轻量 IP + 账号有界限速；首期单应用实例可使用最小进程内/数据库实现，不引入 Redis；
- 登录失败返回统一提示，避免枚举用户名；
- 密码修改、账号停用、入口资格变化和平台高风险操作形成最小审计事件，不记录敏感正文。

## 六、共享数据、对象存储和租户隔离

### 1. PostgreSQL 独立边界

- 允许复用现有 PostgreSQL 引擎，但必须新建当前项目独立数据库、迁移角色和请求应用角色；
- 请求角色不得为表所有者、不得拥有超级用户或 `BYPASSRLS`；
- 迁移角色只在部署迁移步骤使用，不进入长期应用进程；
- 所有租户拥有表继续 `tenant_id NOT NULL` 并 `ENABLE/FORCE ROW LEVEL SECURITY`；
- 平台运维表不伪装成租户表，且不得通过平台查询联接读取租户正文和素材；
- 生产应用只使用请求角色连接；缺少服务端租户上下文必须失败关闭；
- 不读取或迁移历史项目数据库、旧租户、旧账号、旧知识或旧事件。

部署后用实际请求角色定向验证：

- 未设置租户上下文不能读取或写入租户表；
- 伪造/错误租户上下文不能读到“折线之间”记录；
- 应用 API 请求不能提交 `tenant_id`、品牌、组织或账号来切换作用域；
- 账号停用和入口资格撤销立即影响后续请求；
- 不建立第二个权限引擎或每租户 schema。

### 2. 对象存储

把 `LocalObjectStore` 保留为开发/CI 实现，生产通过现有端口使用 S3 兼容实现：

1. 优先复用现场已经存在且可安全创建独立桶/独立凭据的 OSS、S3 或 MinIO 能力；
2. 没有可用共享服务时，可以在 ECS 上用一个当前项目独立的轻量 MinIO 容器和持久卷，但必须纳入备份并确认磁盘足够；
3. 不得复用历史项目桶、对象前缀、元数据表或访问密钥；
4. 对象键由服务端生成，并承接不可被客户端覆盖的租户/所有者作用域；普通列表和下载仍先经过数据库授权；
5. 私人和组织素材原件不写入应用容器文件系统，不生成公开永久 URL；
6. 已知或用户声明的可识别真人未成年人原件在持久化前继续拒绝；不新增年龄识别、媒体清理或审批；
7. 删除继续遵守现有生命周期，不借部署扩展退出接管、公共素材或数据飞轮。

对象存储具体产品属于现场普通实现选择，只要满足独立边界、备份、最小凭据和现有端口合同，不提交用户裁决。

### 3. 系统知识和演示种子

- 在独立生产数据库导入当前 243 条系统候选和 41 条激活治理真源；
- 每条候选必须有来源和生命周期，每条激活必须有消费者与适用条件；202 条候选继续保持候选，不形成待清偿积压；
- 只创建“折线之间”合成演示租户及满足当前在线验收的最少组织、用户、账号、角色、商品和 DM01 条件；
- 演示用户必须通过正式激活与认证使用，不能保留固定公开密码或生产合成会话；
- 不创建笛语服饰或其他真实品牌租户。

## 七、模型调用、限流和运行观测

### 1. DeepSeek

- 生产内容生成固定使用已经冻结的 DeepSeek 适配器和现场核验可用的模型参数；
- 从已授权受保护来源安全迁移连接参数到 ECS root-only 配置；不得输出、提交或写入镜像；
- ECS 必须直接访问供应商 HTTPS，不保留本地 SSH 隧道或 DIFY 代理；
- DM01 不调用模型；
- 真实模型失败、超时、限流或结构不合格时，`GenerationRun` 清楚失败且不产生半成品版本。

### 2. 相称限流

围绕模型调用而不是 ECS CPU 建立简单、可配置的：

- 全局并发上限；
- 每租户并发/速率上限；
- 单用户短时防重复提交；
- 供应商 `Retry-After` 或有界指数退避；
- 高但有限的应急安全上限。

默认值由执行端结合供应商现场限制和“最多 5 租户、200 已启用账号”设置；开发和验收调用不受人为成本预算限制。不得引入消息队列、分布式调度、Redis 集群或完整计费系统。

### 3. 最小观测

在现有 `GenerationRun`/事件基础上按租户、任务/会话记录：

- 成功/失败；
- 供应商模型标识；
- 用量回执；
- 延迟、重试和限流；
- 可归因的错误类别。

提供平台运维聚合视图和最小结构化应用日志。日志不得包含：

- 密码、令牌、Cookie、TOTP、密钥；
- 用户完整输入、提示词、成品正文；
- 私人或组织素材原件；
- 外部模型完整请求/响应。

不引入 Prometheus/Grafana/ELK 等独立观测平台，除非现场已经存在且只需零侵入接入；即使存在也不得成为 M5-4 关闭条件。

## 八、可重复部署、域名切换、备份与回退

### 1. 最小部署形态

- 使用一个生产 Docker 镜像承载构建后的 React 静态资源和 FastAPI 模块化单体；
- 使用一个最小 Compose 或等效 systemd 管理单元；不引入 Kubernetes；
- 镜像以 Git commit SHA 标记，配置和数据不打进镜像；
- 应用只监听 ECS 回环地址的独立端口，Nginx 作为唯一公网入口；
- 增加不泄露内部信息的 `/health/live` 和 `/health/ready`；
- `ready` 至少验证应用、请求数据库角色和对象存储；外部模型故障应在运维状态中可见，但不让整个站点无法登录或使用 DM01。

允许新增的长期生产文件只包括：

- 一个生产 Dockerfile；
- 一个最小生产 Compose/服务配置；
- 一个部署脚本和一个安全回退脚本；
- 一个不含密钥的环境变量示例；
- 必要的 Nginx 示例、健康检查和备份/恢复脚本；
- 本里程碑直接消费的迁移、代码和测试。

不得建设自动 CD 平台。现有 GitHub CI 保持确定性；首次上线由执行端对指定提交进行有界人工部署。

### 2. 部署顺序

1. 备份当前 `diyuai.cc` Nginx 配置、当前项目将使用的数据库/对象边界和必要现场清单；
2. 构建以当前 SHA 标识的候选镜像；
3. 创建独立数据库/角色、对象存储边界和受保护运行配置；
4. 用迁移角色执行 Alembic，随后只用请求角色启动应用；
5. 在回环候选端口完成健康、登录、P1/DM01 和隔离冒烟；
6. `nginx -t` 通过后，只替换 `diyuai.cc` 根域路由并 reload；
7. 保持 `dify.diyuai.cc`、其他子域和历史容器不变；
8. 公网 HTTPS 终验通过后记录发布 SHA、schema 版本和回退目标。

DNS 已确认指向当前 ECS，正常情况下不得修改。若现场漂移，只修改实现 `diyuai.cc` 当前项目入口所必需的最小记录，不触碰其他域名。

### 3. 密钥

- 生产密钥放在 ECS root-only、权限 `0600` 的环境文件或现场已有等强度秘密管理能力中；
- 至少分离数据库迁移凭据、请求应用凭据、对象存储凭据、会话密钥、DeepSeek 密钥和 TOTP 加密/签名材料；
- Git、Docker 镜像、CI、普通日志和执行报告中不得出现明文；
- 不为了“更先进”新增外部秘密平台。

### 4. 备份与恢复

- 配置数据库定时备份，首期采用简单的每日备份和有界保留；具体天数由现有磁盘与恢复时间决定；
- 对象存储使用版本/备份或当前项目独立备份目标；
- 部署前至少生成一次数据库备份；
- 在独立验证数据库中真实恢复一次，并验证租户表、RLS、41 条激活、任务版本及会话敏感数据边界；
- 用一个无敏感信息的测试对象真实验证素材备份恢复；
- 恢复验证完成后清理验证资源，不建立长期第二套环境。

### 5. 首版回退

由于这是当前项目第一次正式上线，首版安全回退目标是：

- 可立即把 `diyuai.cc` 切回维护/503，不暴露损坏应用；
- 停止当前应用容器但保留数据、备份和镜像；
- 恢复部署前 Nginx 配置；
- 在迁移不可安全逆转时从部署前备份恢复到独立数据库，再经核验切换；不得直接破坏性 downgrade 生产数据；
- 后续每次发布保留上一个已知可用 M5-4+ 镜像，才启用常规应用版本回退。

实际演练一次“候选端口失败不切公网”以及一次“公网路由切维护再恢复当前版本”，不需要为了首发伪造不存在的上一生产版本。

## 九、最小在线验收

### A. 三入口与账号生命周期

1. 三个登录页可分别直达，互不提供跳转导航；
2. 未登录、错误入口、伪造 Cookie、跨入口 API 均为 `401/403`，且不创建业务数据；
3. 平台运维完成密码 + TOTP 登录，创建“折线之间”租户壳或复用幂等种子，并生成首位租户管理员一次性激活材料；
4. 租户管理员完成激活，创建一个租户用户并生成一次性激活链接；
5. 用户用链接设置密码，重复使用链接失败；用户可修改密码；
6. 同一自然人同时有用户/管理员资格时，两个入口会话互不扩权；
7. 停用用户、撤销入口资格或停用租户后，现有会话失效；
8. 平台运维和租户管理员均不能读取用户私人素材或成品正文。

### B. 生产业务路径

使用正式认证、正式生产数据库和真实 DeepSeek：

1. 租户用户从自然弱种子生成一份 P1 成品，形成 V1；
2. 自然修改形成 V2，V1 可回读；
3. 主动保存并在原作用域自然承接；
4. 再生成一份 P2—P5 中任一内容，确认价值路由和完整文字媒体包；
5. DM01 使用黄金库存得到 20 件输入、15 件上墙、5 件未上墙；自然反馈后只修改既定一项，形成 14/6 且 V1 保留；
6. 上传一个无敏感信息的私人测试素材和一个组织测试素材，验证对象存储、列表、删除及跨用户/组织拒绝；
7. 运行记录包含实际资产版本、模型用量/延迟/失败归集，但用户页面和日志不泄露内部标识或正文。

只做以上最小真实路径，不重跑六张真实内容卡、不重新人工审查 P1—P5 成品质量，因为模型、Prompt、内容合同和知识激活没有在 M5-4 改写。若部署实现实质修改了生成策略，才补一次受影响内容的人工语义审阅。

### C. 生产隔离与失败

- ECS 实际请求角色不拥有 `BYPASSRLS`、不拥有租户表；
- 无租户上下文、错误租户、错误品牌/账号/用户和跨入口请求失败关闭；
- 对象存储不能通过猜测对象键越权读取；
- DeepSeek 超时/失败不产生内容版本；
- DM01 不受模型故障影响；
- 重复部署和种子导入幂等；
- 数据库迁移失败不切换公网；
- 健康检查失败不 reload Nginx。

### D. 相称非功能验证

- 验证启动、重启和异常退出后的自动恢复；
- 对认证读取和普通页面做小规模突发，不做 200 并发压测；
- 对模型调用只验证配置的全局/每租户并发、速率限制、退避和失败可见；
- 验证数据库备份恢复、对象测试文件恢复和公网维护回退；
- 记录实测结果，不虚构 SLA、RTO 或 RPO；M6-2 才结合真实试点冻结合同目标。
