## 三、冻结的产品与路由语义

### 1. 内容任务先于产品路由

- 普通问候、闲聊、情绪交流和没有内容生产意图的输入可以正确结束于“不路由”；
- 不路由时只给自然回应，不创建 `BusinessTask`、`GenerationRun`、`ContentItem` 或版本；
- 用户无需说“生成内容”、无需选择产品编号。系统应结合当前内容工作台、自然输入和前情判断内容任务是否成立；
- 真正缺少不可替代输入时最多问一个自然问题；能够诚实交付时直接生成一个最佳成品；
- 顶层已进入陈列搭配时继续走既有 `DisplayArtifact`，绝不进入 P1—P5。

允许保留一个极小的高置信普通交流快路径；其他含混但可能形成内容的输入由同一价值路由能力判断。不得把“店里、商品、搭配、生活、镜头”等孤立词语写成产品路由规则。

### 2. 一个独立成品只有一个主要产品

内部稳定值可以采用现有名称或等义强类型：

| 产品 | 主要受众价值 | 当前成品必须兑现的最小差异 |
|---|---|---|
| P1 `dressing_decision` | 帮助选择 | 优先怎样选、何时改变判断、下一步怎样验证 |
| P2 `product_truth` | 解释商品 | 一项商品专属新增理解、相伴限制及成立/不成立条件 |
| P3 `brand_life_narrative` | 建立人格 | 账号面对该生活会注意什么、怎样判断和待人，并给受众具体观看回报 |
| P4 `local_response` | 经营关系 | 近场共同处境或真实信号、合法账号回应、未到店者也能带走的关系回报 |
| P5 `visual_styling_story` | 创造欲望 | 由真实品牌商品与品牌审美共同成立、且必须依赖画面才能看见的新穿着可能 |

商品知识、造型、构图、镜头、动作、声音、生活观察和本地语境均为共享能力。调用共享能力不自动创建第二产品，也不把 P3/P4 的重视觉成品升级为 P5。

用户明确要求另一份“可以单独拍、单独用、前一篇不动”的完整成果时，创建新的独立内容任务/成品并重新路由；不要给一个成品加双主产品，不要倒改原任务。复用现有“承接上一条/指定版本”的能力即可，不建设多成果工作流。

### 3. 路由不是五套 Agent

继续复用一个 `ContentService`、一个内容上下文编译入口、一个 DeepSeek 薄适配器、一个版本/事件底座和一套页面交互。

为了在不把所有产品资产塞进同一个 Prompt 的前提下完成价值路由，允许在同一内容编译器内部使用两个有直接消费者的最小步骤：

1. **价值判断**：使用当前任务种子、品牌/账号/角色/受众、用户明确选中的商品或前情，返回 `none` 或唯一 P1—P5；
2. **成品编译**：只加载该主要产品当前适用的领域资产和品牌事实，生成一份完整成品。

这不是第二个 Agent、DIFY 工作流或多模型路由。两个步骤使用同一个已确认 DeepSeek 供应商和同一个内容编译服务；不建设路由管理后台、置信度评分、阈值系统、产品对组合矩阵或可配置工作流。明显普通交流允许不调用模型。

价值判断只保存最终路由结果和必要运行事件，不保存模型隐藏推理或自然语言思维链；用户页面不显示路由过程。

### 4. 修订与路由稳定

- 同一任务的 V1→V2 默认保持原主要产品合同；
- 修改可以改变形式、开头、节奏、台词、画面和表达，但不能静默把 P2 改成 P3 或把 P3 改成 P5；
- 用户明确要求改变主要产品价值或新增一份可独立采用的成果时，应新建任务并保留原成品；
- 生成、浏览、导出和修改不自动写入长期记忆；现有保存和明确承接边界不变。

## 六、最小工程落点

### 1. 共用内容编译合同

将当前 P1 专用类型收敛为一个共用内容成品外壳和五个小型产品专属合同，避免五套服务和五套表。

最小持久语义：

- 每个内容任务具有一个非空 `primary_content_product`，旧任务安全回填为 P1；
- `none` 不持久化为业务任务；
- 路由结果属于任务；同任务版本继承，不在每个版本复制一套可漂移路由；
- P2—P5 的专属最小结果可以作为强类型联合或现有 JSON 结果中的受控联合，不要求新建四张表；
- `ContentVersion` 继续保存用户可见完整正文；`GenerationRun` 继续保存模型、耗时、重试、用量、实际资产和最小运行回执；
- 不保存隐藏思维链、不向用户显示内部合同字段。

产品专属最小结果：

- P1：`choice / boundary / next_action`；
- P2：`product_insight / tradeoff_or_limit / validity_condition`；
- P3：`persona_observation / audience_return / brand_account_link`；
- P4：`local_reality_or_signal / legitimate_account_response / public_relationship_return`；
- P5：`real_product_anchor / visible_styling_proposition / visual_dependency`。

字段名可以采用等义、清楚的代码命名；不得把它们变成用户表单、用户可见分析报告或评分器。执行端可以采用比上述更小但能够无歧义验证五个产品差异的联合类型。

### 2. 完整用户成品

沿用当前抖音视频文字演示路径，每个 P1—P5 成品仍给普通创作者一份可直接使用的完整文字包，至少自然包含：

- 一段人话版内容概要：这条内容从什么真实种子出发、希望受众得到什么；
- 完整标题及台词/对白/口播/解说或承担正文功能的字幕文本；
- 当前形式需要的画面、动作和顺序；
- 当前形式需要的字幕；
- 当前形式需要的声音、背景音效/音乐方向和最低制作提示。

上述是 M5-1 当前抖音视频演示路径的可执行要求，不冻结成所有平台、图文和未来媒体的统一机械字段。M5-2 再按抖音、小红书、微信视频号及视频/图文语境重编译。

服务端应以受控结构组装最终正文，避免内部专属合同与用户可见正文相互矛盾；但不得把五类内容都编译成同一僵硬模板。模型负责创意表达，应用负责产品唯一性、必要结构、可信事实、作用域和版本原子性。

### 3. 模型与失败处理

- 正式内容路径继续使用当前受保护配置中的 `deepseek-v4-flash`，不接 DIFY，不引入第二模型；
- 价值判断与成品编译均使用同一薄适配器和有界超时/退避；
- 路由或结构结果格式错误时允许一次受控修复/重试；普通内容质量失败可以在同一次请求内后台改写一次；
- 不允许无限自评、自我辩论、多模型比较或每篇完整反事实生成；
- 最终仍不满足产品硬合同则本次运行失败，且不得产生半任务版本；已经创建的运行可按现有失败原子性诚实留痕；
- 权限、租户/品牌/账号隔离、无真实商品却强行创建 P5、承重商品事实冲突等硬失败不得靠模型改写绕过；
- P4 当前只使用已知南城店上下文和用户输入的真实信号。公开地域搜索是已确认的可选共享能力，但本里程碑没有直接需要它的黄金任务，暂不接搜索工具、不预建空接口。

### 4. 页面与自然交互

- 用户不选择 P1—P5，不看到内部产品代码；
- 南城店合成内容身份可由自然按钮进入，页面显示品牌、实际操作人、代表组织、发布账号和内容角色的人话摘要；
- 生成结果先显示自然概要，再显示完整成品；
- 用户可以继续用自然语言讨论、修改或要求另一种表现形式，现有 V1/V2、读取旧版本、保存和明确承接继续可用；
- 当前路由、资产回执、事实引用、校验结果和模型运行信息只在后台，不增加用户认知负担；

## 七、X-01 七张在线验收卡
