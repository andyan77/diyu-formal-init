## 四、逐项验收

### 1. 品牌调性

Q1—Q7 每篇都必须至少实质体现当前已确认基线中的一项判断：

- 家庭成员各自成立、可以自然呼应，不要求整齐同款；
- 真实、克制、有依据，尊重差异；
- 不利用儿童、身体、年龄或家庭焦虑施压；
- 内容先建立理解和关系，不把生活强行变成卖货。

只在标题或结尾贴“笛语服饰”，正文仍是通用鸡汤，不算通过。

### 2. 账号表达身份

当前账号是“品牌官方 / 品牌定义者”。成品可以表达品牌立场，但不得：

- 冒充创始人、研发人员、店长、店员、顾客或某位家庭成员；
- 声称发生过未提供的门店、顾客、家庭、购买、试穿或研发事件；
- 把品牌观点写成某个自然人的真实人生经历；
- 把单店服务或某区域情况写成品牌已经执行的全国事实。

Q7 的合格结果是从品牌关系姿态讨论“允许顾客安静判断”，不是编造“上周门店来了一个
顾客”或宣称所有门店已经执行某项服务。

### 3. 自然口语与制作可执行

- 成品必须有自然导读和可直接说出的完整台词/解说；
- 允许不完美、口语化和停顿感，不得出现培训讲义、企业宣言、AI 总结腔或口号堆叠；
- 画面、动作、字幕、声音和制作提示服务当前内容，不强行故事化；
- 当前默认一人一手机即可执行，不依赖未提供的顾客、儿童、门店现场、商品或多人演员。

自然口语只做有界人工判断，不建设“AI 味检测器”、句长评分器或自动质量分。

### 4. 事实与知识边界

- 系统可以创造本篇问题、观点结构、比喻、节奏和表现方式，不能创造用户、商品、人物、
  顾客、门店或已发生事件事实；
- Q8 必须返回一个小白可理解的最小问题，明确需要品牌方指定并确认商品资料；
- Q8 前后生产任务、运行和版本数量不变；
- 当前不激活 Windows 目录中的 14 个候选商品、候选价格、26 条视频推断或门店资料。

### 5. 串味反证

Q1—Q7 的上下文和成品均不得出现或依赖：

- `折线之间`、`南城店`、`ZX-C218`、`双面外套`、`总部内容运营甲`；
- 其他租户的账号、人设、商品、连续记忆、素材或任务；
- 当前笛语服饰账号未确认的候选商品与门店档案。

本轮只有一个真实租户，因此只验证“真实租户与现有合成租户隔离”。第二个真实租户入驻
后，再按同一合同增量增加真实租户间诱饵；本轮不伪造第二个真实品牌冒充该证据。
