## 0. 证据等级制度（即日生效，全部前端文档适用）

| 等级 | 含义 |
|---|---|
| `prototype_verified` | 高保真 HTML 原型中已验证（交互/渲染/语法），**不证明生产运行** |
| `document_verified` | 与文档、ADR、契约核对一致 |
| `production_verified` | 仅限真实软件与运行证据，当前**任何前端结论不得使用** |

**换标声明**：《评审意见与修订指引 v1.0》中所有 `runtime_verified` 标注一律读作 `prototype_verified`；结论本身不变，不需重查。"一线水准"类表述统一读作"有竞争力的高保真候选"，真实用户尚未验证。

## 1. 动作语义修订（最重要，依据 ADR-026）

### 1.1 "采用此版"退出全局主动作

ADR-026 L18/L56（`document_verified`）：采用是 `ActivityEvent` 行为事件，如实记录、不自动升级；系统不要求用户为留痕多点按钮。因此：

| 场景 | 主动作 |
|---|---|
| 独立单篇成品（单版本） | **继续改** / **复制** / **导出**（按场景取舍，无"采用此版"） |
| 存在多个版本 | 版本条按需出现 **设为当前版本**（仅当查看的不是当前版时显示） |
| 需要跨任务复用 | 明确执行 **存为素材** |
| 连续内容 | 明确执行 **纳入系列**，或自然语言"下一集接着这条" |
| 留痕 | 后台按 ADR-010/026 记录行为事件，**不要求用户多点一次** |

### 1.2 "哪一版作数"的四处角色移交（补丁 1，防复活"保存语义混乱"）

| 位置 | v1.0 | v1.1 |
|---|---|---|
| 版本条徽标 | "V3 已采用" | "V3 当前版" |
| 首页/列表徽标 | "已采用" | "当前版" |
| 搜索结果优先级 | "已采用成品"第一 | "当前版本成品"第一 |
| 系列分集标注 | "已采用 · 不自动写入下一集" | "当前版 · 不自动写入下一集" |

P0 验收第 6 条改写为：**"版本指定、导出、存素材、纳入系列分别表达，均不自动发生。"**
用户六问第 6 问改写为：**"哪一版是当前版，是否要继续到系列、计划或下一任务？"**

### 1.3 端点映射注（不新增开发要求）

- 不新增 adopt 端点；行为事件走既有留痕。
- 现有 `POST content-versions/{id}/save` 语义 ≈ "存为素材"（主动保存 → 跨任务复用，ADR-014 §2）。
- "设为当前版本"进入对应里程碑时再配最小端点；OpenAPI 是实现真源，未来页面不提前建空 API。

## 3. 移动端结构（创始人裁决口径）

- 底部**两段**切换：`对话` ｜ `成品`（陈列侧为`方案`）。**不设"上下文"第三选项卡**——上下文是内部概念。
- 顶部常驻一行身份摘要（"南城店账号"），点按展开抽屉：完整发布账号/表达身份/组织 + 平台/商品/系列/制作条件标签。
- 成品视图常驻可达 **继续改** 与 **复制/导出**；"继续改"切至对话并聚焦输入框。
- **不自建触屏选字浮层**：支持点选一个内容段落，或直接自然语言"把结尾改短一点，其他不动"。

## 4. 三类复用异常模式（补丁 2：各配 1 个示例态，13 → 16 态封顶）

| 模式 | 覆盖 | 示例态（进高保真稿） |
|---|---|---|
| E1 空状态 | 搜索无结果 / 列表零数据 / 冷启动引导 | 搜索无结果："没有找到匹配的成品或方案。换个说法，或去成品列表看看。"（冷启动参数示例："先确认品牌草案，今天的建议就能开始了。"） |
| E2 可恢复失败 | 生成失败 / 断网 / 库存解析失败 | 生成失败："这次成品没有完成。你的输入都还在。" + **重试** 按钮（断网参数："网络断了。已写的内容都在，联网后接着来。"） |
| E3 高风险确认 | 仅不可恢复动作（删除私人素材等） | 确认弹窗："删除后找不回来。" [删除] [先留着] |

不再逐个建设 S14—S21 八张页面；三模式参数化复用。

## 5. 视觉修订

### 5.1 正式 VI 资产是唯一视觉真源

- 只使用 **Medium 字重版**正式资产（`笛语VI-补充资产-Medium字重版/`）；
- Logo/Symbol/favicon 直接引用正式 SVG/PNG（原型内联官方 SVG；生产工程引用资产文件）；
- **禁止用 CSS、字符或近似图形重画 Logo**（v1.0 原型的 CSS 手绘标志按占位处理，v1.1 替换）；
- 颜色 token 与官方规范第 3 页对齐：墨黑 #171614 · 朱砂红 #B7462E · 石灰灰 #8A887F · 呼吸白 #F8F7F3 · 纯白 #FFFFFF；
- 中文思源宋体 Medium、英文思源黑体，界面正文仍用系统无衬线。

### 5.2 朱砂红收敛（按 §37 原文范围，不做"唯一红"）

- 保留：当前应用指示、输入焦点、"当前版"徽标、关键待处理事项、少量品牌细节；
- 撤销：栏目眉标（成品导读/方案导读等）、"镜头 01—04"标签等装饰性用红 → 改 muted 灰；
- 主按钮保持墨色；`.btn.accent` 样式保留定义，仅供关键待办类确认使用。

### 5.3 其他

- 宋体只保留：品牌标识、成品大标题、成品内小节标题；状态/管理/加载标题回系统无衬线；
- 最小字号 ≥ 12px（8–9px 微标注：能删则删，不能删则升号）；
- 陈列方案卡中卡削层：示意图一层容器，四个分节改无边框分隔线列表；
- Hero 去发光同心圆；`i-spark` 闪星换中性内容意象；
- 暗色 Hero 降噪：系统建议不占用全站最重视觉语言。
