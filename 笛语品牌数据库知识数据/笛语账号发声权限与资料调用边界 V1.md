# 笛语账号发声权限与资料调用边界 V1

**文档编号：** DIYU-ACCOUNT-AUTHORITY-001
**版本：** V1.0
**状态：** 待品牌方验收
**适用范围：** 内容检索、生成、审核、账号发布准备与审计
**更新时间：** 2026年7月22日

---

## 0. 核心原则

任何账号能否使用一条资料，不由“系统能不能检索到”决定，而由以下六项共同决定：

1. 资料属于哪个组织；
2. 账号属于哪个组织；
3. 账号绑定什么角色；
4. 资料涉及什么对象；
5. 资料在哪个地区和时间有效；
6. 资料是否允许用于该类内容和账号。

---

# 一、权限判断六轴

| 权限轴 | 判断问题                |
| --- | ------------------- |
| 组织轴 | 资料属于总部、区域还是具体门店     |
| 角色轴 | 当前角色是否有资格解释这类资料     |
| 对象轴 | 资料涉及品牌、商品、人物、门店还是顾客 |
| 地域轴 | 资料是全国、区域还是单店有效      |
| 时间轴 | 资料是否仍在有效期           |
| 授权轴 | 是否允许用于生成、审核和发布      |

六轴中任一项不满足，资料不得直接进入最终内容。

---

# 二、资料层级

## 2.1 品牌级资料

包括：

* 品牌定位；
* AudienceProfile；
* 品牌声纹；
* 品牌视觉基线；
* 正式品牌规则；
* 总部确认的系列主题。

允许使用：

* 总部账号；
* 获得品牌授权的区域账号；
* 获得品牌授权的门店账号。

限制：

区域和门店可以调用，但不得修改原意。

---

## 2.2 商品级资料

包括：

* SPU/SKU；
* 面料；
* 版型；
* 尺码；
* 工艺；
* 洗护；
* 设计和开发资料；
* 官方商品图片。

允许使用：

* 品牌官方；
* 商品研发；
* 零售运营；
* 获得该商品销售和内容权限的区域与门店账号。

限制：

* 门店不能擅自改写商品硬事实；
* 没有销售权限的组织不能默认使用；
* 商品失效后不能继续写成当前在售商品。

---

## 2.3 组织级资料

包括：

* 总部组织事实；
* 区域代理事实；
* 门店基本资料；
* 组织职责；
* 授权关系。

使用原则：

* 总部可使用总部和已汇总确认的组织资料；
* 区域只能使用本区域资料；
* 门店只能使用本门店资料；
* 任何组织不得冒充其他组织。

---

## 2.4 人物资料

包括：

* 真实姓名；
* 岗位；
* 经历；
* 观点；
* 照片；
* 视频；
* 声音；
* 家庭和生活事件。

使用前必须确认：

* 本人是否授权；
* 哪些账号可用；
* 哪些事实可公开；
* 授权期限；
* 是否允许AI改写；
* 是否允许跨平台使用。

无授权时，只能使用组织性ContentRole。

---

## 2.5 门店运行资料

包括：

* 当日库存；
* 价格；
* 活动；
* 营业变化；
* 到店情况；
* 天气；
* 陈列；
* 高频咨询；
* 本地事件。

使用原则：

* 只在明确时间和门店范围内有效；
* 不进入长期稳定品牌事实；
* 过期后停止作为当前信息使用；
* 不得由其他门店复制成自己的现场。

---

## 2.6 顾客与评论资料

可以使用：

* 公开评论；
* 匿名化问题；
* 顾客明确授权的评价；
* 门店整理的高频问题。

不得使用：

* 未授权私人聊天；
* 姓名和手机号；
* 可识别儿童信息；
* 订单明细；
* 私人家庭情况；
* 虚构顾客反馈。

---

# 三、不同层级账号的调用范围

## 3.1 总部账号

可以调用：

* 品牌级资料；
* 总部组织资料；
* 全国有效商品资料；
* 已经汇总确认的区域或门店事实；
* 获得授权的人物和素材。

不得直接调用：

* 未经门店确认的单店现场；
* 未经区域确认的区域判断；
* 私人顾客资料；
* 过期库存和活动。

---

## 3.2 区域账号

可以调用：

* 品牌级已授权资料；
* 本区域组织资料；
* 本区域在售商品资料；
* 本区域天气、商圈和活动；
* 本区域门店确认资料。

不得调用：

* 其他区域经营事实；
* 总部未公开内部资料；
* 其他区域顾客和门店素材；
* 未授权品牌决策资料。

---

## 3.3 加盟门店账号

可以调用：

* 品牌公开基线；
* 本店有权销售的商品资料；
* 本店库存、活动和营业信息；
* 本店采集且授权的素材；
* 本店顾客匿名问题；
* 本地天气和商圈信息。

不得调用：

* 其他门店库存；
* 其他门店顾客；
* 其他门店自采素材；
* 总部内部资料；
* 区域敏感经营信息；
* 未授权人物内容。

---

# 四、声明等级

每个进入内容的关键声明，应属于以下一种等级。

| 等级 | 名称    | 定义              | 是否可直接发布     |
| -- | ----- | --------------- | ----------- |
| A  | 已确认事实 | 有正式资料或责任人确认     | 可以          |
| B  | 授权解释  | 基于事实进行专业解释      | 可以，须不改变事实   |
| C  | 创作构造  | 用于标题、比喻、场景和叙事组织 | 可以，但不得伪装成事实 |
| D  | 未确认声明 | 缺来源、越权或已过期      | 不可以         |

## 4.1 示例

商品面料为棉质：

* “面料成分为95%棉”：A级事实；
* “触感更偏柔软”：需要试穿或资料支持，可能为B级；
* “像夏天的风一样轻”：C级创作表达；
* “绝对不会闷热”：D级未确认声明。

---

# 五、跨组织资料继承规则

## 5.1 总部向下授权

总部品牌资料可以授权区域和门店使用，但必须明确：

* 是否允许原文使用；
* 是否允许改写；
* 是否允许结合本地事实；
* 是否允许二次剪辑；
* 是否有时间期限。

## 5.2 区域向门店授权

区域可以向本区域门店提供：

* 区域活动；
* 区域内容主题；
* 区域视觉素材；
* 本地季节和商圈资料。

区域不得：

* 改写总部品牌原则；
* 将一个门店的事实授权给其他门店冒充使用；
* 将未核验顾客信息向下分发。

## 5.3 门店资料默认不向上自动继承

门店上传的：

* 顾客问题；
* 现场照片；
* 店员自采视频；
* 经营事件；
* 本地生活素材；

默认只供本店使用。

总部或区域需要复用时，必须获得门店和相关权利人的明确授权。

---

# 六、操作人身份与发布账号分离

## 6.1 基本原则

* 每个操作人使用独立登录身份；
* 不允许多人共享同一后台账号；
* 操作人可以管理多个发布账号；
* 发布账号不能反向代表操作人的个人身份；
* 所有生成、修改和审核行为应记录操作人。

## 6.2 最低审计记录

至少记录：

* 谁发起内容任务；
* 使用哪个发布账号；
* 调用了哪些资料；
* 谁修改；
* 谁审核；
* 谁批准进入发布准备；
* 使用的资料版本；
* 内容最终版本。

---

# 七、真实人物绑定规则

账号绑定真人时，必须补齐：

| 项目   | 要求   |
| ---- | ---- |
| 姓名   | 本人确认 |
| 岗位   | 企业确认 |
| 公开经历 | 本人确认 |
| 肖像   | 明确授权 |
| 声音   | 明确授权 |
| 可讲主题 | 明确范围 |
| 禁止主题 | 明确范围 |
| 使用账号 | 明确账号 |
| 使用平台 | 明确平台 |
| 授权期限 | 明确期限 |
| 撤回方式 | 可执行  |

真人撤回授权后：

* 停止生成新的相关内容；
* 停止新发布；
* 已发布内容按授权协议处理；
* 角色可以退回组织性岗位人设；
* 不继续模仿其声音和私人表达。

---

# 八、账号间内容差异规则

同一商品被多个账号使用时，应形成不同内容来源。

例如同一件外套：

| 账号    | 合法内容角度       |
| ----- | ------------ |
| 品牌官方  | 为什么进入本季品牌系列  |
| 创始人   | 为什么品牌愿意做这种选择 |
| 商品研发  | 版型、面料和设计取舍   |
| 零售陈列  | 怎样与其他商品组合    |
| 区域官方  | 当地季节下如何使用    |
| 区域主理人 | 本地消费者为什么关注   |
| 门店账号  | 本店近期真实试穿问题   |

禁止仅替换：

* 账号名称；
* 开头语；
* 地名；
* 少量形容词。

---

# 九、内容生成停止条件

出现以下情况时，系统应停止生成关键事实内容，并列出补充资料：

1. 商品硬事实缺失；
2. 角色没有资格发表该结论；
3. 资料不属于当前组织；
4. 资料已过期；
5. 素材没有明确使用权限；
6. 真人事实没有本人授权；
7. 顾客内容未匿名或未授权；
8. 门店试图使用其他门店现场；
9. 区域试图修改总部品牌原则；
10. 创作构造可能被理解为真实事实。

停止生成不代表整条内容作废。

系统可以：

* 删除无法确认的声明；
* 改为条件性表达；
* 改为通用方法；
* 列出待补资料；
* 转为不涉及该事实的内容方案。

---

# 十、发布前权限检查

每条内容至少检查：

| 检查项         | 合格条件         |
| ----------- | ------------ |
| 账号组织        | 已绑定明确组织      |
| ContentRole | 有资格发表内容      |
| 商品权限        | 组织有权使用相关商品   |
| 地域范围        | 没有越过有效区域     |
| 时间状态        | 价格、库存、活动仍有效  |
| 人物授权        | 人物事实和素材可用    |
| 素材授权        | 图片、视频允许该账号使用 |
| 顾客隐私        | 已授权或匿名化      |
| 声明等级        | 无D级声明        |
| 账号差异        | 内容来自本账号真实位置  |
| 品牌一致        | 不改变笛语品牌基线    |
| 审核责任        | 有明确批准人       |

## 本文件结论

笛语账号权限的本质不是“谁能看到什么”，而是：

> 谁能够基于什么真实资料，以什么身份，在什么范围内，说出什么结论。
