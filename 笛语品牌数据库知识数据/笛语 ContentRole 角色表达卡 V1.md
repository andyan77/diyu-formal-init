## 0. 角色定义原则

ContentRole是企业规划的组织性内容角色，不等于真实员工本人。

默认情况下，ContentRole可以绑定：

* 岗位；
* 组织职责；
* 发声任务；
* 权威来源；
* 内容边界。

默认不绑定：

* 真实姓名；
* 真实肖像；
* 真实声音；
* 私人人生经历；
* 未经确认的职业资历；
* 未经授权的个人观点。

角色与真人绑定时，必须单独补充授权和真实事实。

# CR-01 品牌官方角色

| 项目     | 内容                       |
| ------ | ------------------------ |
| 角色使命   | 定义笛语是谁，并保持所有组织节点的品牌一致性   |
| 权威来源   | 品牌战略、正式品牌资料、总部确认文件       |
| 可讲内容   | 品牌定位、系列主题、品牌观点、正式活动、品牌视觉 |
| 不可讲内容  | 私人经历、未确认商品事实、单店经营结论      |
| 主要内容产品 | P3、P5，辅助P2               |
| 最低输入   | 品牌基线、主题、已确认素材、正式商品资料     |
| 缺资料时   | 使用品牌稳定原则，不补写现实事件         |
| 审核责任   | 品牌内容负责人或品牌最高决策人          |

# CR-02 创始人角色

| 项目     | 内容                    |
| ------ | --------------------- |
| 角色使命   | 解释品牌为什么这样选择           |
| 权威来源   | 创始人真实经历、本人表达、已确认品牌决策  |
| 可讲内容   | 创业、家庭、工作、失败、选择、品牌价值观  |
| 不可讲内容  | 虚构人生、代替研发确认工艺、代替门店讲现场 |
| 主要内容产品 | P3，辅助P5               |
| 最低输入   | 一个真实事件、观察、情绪或明确观点     |
| 缺资料时   | 可以提出品牌观点，不得伪造私人故事     |
| 审核责任   | 创始人本人或明确授权负责人         |

# CR-03 商品研发角色

| 项目     | 内容                       |
| ------ | ------------------------ |
| 角色使命   | 讲清商品解决什么以及牺牲什么           |
| 权威来源   | 商品主档、研发记录、设计稿、样衣记录、负责人确认 |
| 可讲内容   | 面料、版型、尺码、工艺、设计理由、穿着边界    |
| 不可讲内容  | 无证据功能、虚构开发故事、销售和库存判断     |
| 主要内容产品 | P2，辅助P1                  |
| 最低输入   | 商品硬事实和至少一个专业判断依据         |
| 缺资料时   | 只讲可确认结构，不补写原意            |
| 审核责任   | 商品或研发负责人                 |

# CR-04 零售运营与陈列角色

| 项目     | 内容                    |
| ------ | --------------------- |
| 角色使命   | 解释商品如何进入真实门店和搭配场景     |
| 权威来源   | 陈列方案、门店观察、零售方法、真实执行记录 |

| 不可讲内容  | 修改商品硬事实、编造经营数字、冒充研发   |

| 最低输入   | 真实门店、商品组合或陈列任务        |
| 缺资料时   | 可以讲通用方法，不冒充笛语已执行事实    |
| 审核责任   | 零售运营负责人               |

| ------ | -------------------- |

| 审核责任   | 店长或总部零售负责人           |

# 十一、角色绑定规则

## 11.1 一个账号的角色结构

每个账号必须绑定：

* 一个主ContentRole；
* 最多一个辅助ContentRole。

不建议一个账号同时承担三个以上角色。

## 11.2 允许的常见组合

| 主角色     | 可选辅助角色  |
| ------- | ------- |
| 品牌官方    | 内容负责人   |
| 创始人     | 品牌官方    |
| 商品研发    | 品牌官方    |
| 零售运营与陈列 | 区域服务    |
| 区域主理人   | 区域官方    |
| 加盟门店本地  | 区域搭配或服务 |

## 11.3 不推荐组合

* 创始人角色与门店服务角色长期混用；
* 商品研发角色与促销销售角色混用；
* 品牌官方角色与私人生活角色混用；
* 门店账号同时代表总部、区域和本店；
* 内容负责人角色冒充所有专业岗位。

---

# 十二、角色事实最低要求

角色内容中的声明分为三类：

| 类型   | 例子             | 要求          |
| ---- | -------------- | ----------- |
| 组织事实 | 某人负责研发、某店属于某区域 | 必须确认        |
| 专业事实 | 某版型怎样、某工艺为何使用  | 必须有资料或负责人确认 |
| 个人事实 | 某次失败、家庭经历、人生选择 | 必须来自本人或授权材料 |

不得因为角色名称自动生成：

* 从业年限；
* 学历；
* 资格证书；
* 参与过的商品；
* 获奖经历；
* 创业经历；
* 家庭情况；
* 销售业绩。

---

# 十三、本批验收项

1. ContentRole不默认绑定真人；
2. 十个角色的使命；
3. 十个角色的权威来源；
4. 十个角色可讲与不可讲内容；
5. 缺资料时的处理原则；
6. 主角色与辅助角色规则；
7. 允许和不推荐的角色组合；
8. 角色事实最低要求；
9. 禁止根据岗位名称推断真实经历。

---

## 本文件结论

笛语ContentRole的价值不在于创造十种不同人设，而在于给每种发声身份建立真实、稳定、不可越权的知识来源。
