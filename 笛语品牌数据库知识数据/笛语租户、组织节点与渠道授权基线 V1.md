## 0. 本批裁决

笛语采用：

> **一个品牌租户、总部统一定义、区域叠加翻译、门店真实证明、数据逐级回流的联邦式组织模型。**

组织型IP不等于大量开设账号。应先确定角色拥有什么真实权威、服务谁、可以讲什么，再决定由独立账号、共享专业账号还是固定栏目承载。参考文件也明确提出“先角色矩阵、再账号矩阵”，总部第一阶段的最小有效形态可以是品牌母账号、一个价值角色、一至两个专业角色及若干稳定栏目。

本文件建立的是：

1. 租户；
2. 组织节点；
3. 组织之间的品牌授权关系；
4. 组织与账号、角色、人员的绑定方式；
5. 总部知识、区域知识和门店事实的继承规则；
6. 账号、内容、素材和顾客信息的资产边界。

# 一、四类对象必须分离

笛语系统中，以下四类对象不得混为一体。

| 对象                | 含义                  | 示例                 |
| ----------------- | ------------------- | ------------------ |
| 租户 Tenant         | 与笛语系统签约并独立治理数据的企业主体 | 笛语品牌经营企业           |
| 组织节点 Organization | 企业内部或渠道体系中的经营单位     | 笛语总部、区域代理、某加盟门店    |
| 发布账号 Account      | 在具体平台上对外发布内容的身份     | 笛语品牌官方抖音号          |
| 内容角色 ContentRole  | 账号采用的专业或组织发声身份      | 创始人、QA、QC、样衣工、门店店长 |
| 操作人员 Operator     | 登录后台、发起、修改或审核任务的人   | 内容编辑、店长、区域运营人员     |

基本关系：

> 一个租户可以有多个组织节点；
> 一个组织节点可以管理多个发布账号；
> 一个账号绑定一个主角色和最多一个辅助角色；
> 一个操作人员可以被授权操作多个账号；
> 角色不等于真人，账号也不等于登录人员。

## 2.2 一个签约企业对应一个租户

第一阶段默认：

> 一个与笛语系统签约并承担最终治理责任的企业主体，对应一个独立租户。

区域代理和加盟门店原则上作为该租户下的授权组织节点接入，不另行复制一套笛语品牌核心。

未来存在以下情况时，才评估独立租户：

* 独立经营其他品牌；
* 法律与数据责任完全独立；
* 不能共享总部知识；
* 需要完全独立计费和管理；
* 合同明确要求数据物理或逻辑隔离。

# 四、各组织节点职责

## 4.1 笛语总部

总部的核心动作是：

> 定义。

总部负责统一：

* 品牌定位；
* 品牌声纹；
* 视觉边界；
* 商品核心事实；
* 专业标准；
* 总部角色；
* 全国内容母题；
* 合规和资产规则；
* 区域与门店接入规则。

参考基线将三级渠道定义为“总部定义、区域翻译、终端证明”，并要求全国核心知识不可被区域复制修改。

总部不得：

* 编造区域经营事实；
* 编造门店现场；
* 将单店顾客问题写成全国结论；
* 未经责任部门确认修改商品硬事实。

---

## 4.2 区域代理

区域代理的核心动作是：

> 翻译和组织。

区域可以叠加：

* 地区气候；
* 城市结构；
* 节庆；
* 商圈；
* 消费节奏；
* 区域库存；
* 区域经营案例；
* 当地高频穿衣问题；
* 区域门店协同资料。

区域不得：

* 修改笛语品牌定位；
* 覆盖总部商品核心事实；
* 新建一套互相冲突的品牌声纹；
* 将一个门店的事实复制给其他门店冒充使用。

---

## 4.3 总部直营门店

总部直营门店同时承担两类责任：

1. 真实零售经营节点；
2. 总部品牌理念的一线验证节点。

可以贡献：

* 试穿问题；
* 门店服务；
* 陈列执行；
* 商品搭配；
* 当地天气；
* 顾客匿名问题；
* 商品反馈；
* 总部策略落地结果。

不得把直营门店的局部事实自动升级为全国事实。

---

## 4.4 加盟门店

加盟门店的核心动作是：

> 证明和服务。

可以贡献：

* 本店位置和商圈；
* 本店团队；
* 本店商品；
* 本店库存和活动；
* 本店真实服务；
* 本店顾客匿名问题；
* 本店陈列和试穿观察；
* 本地生活素材。

参考基线要求门店通过真实人物、真实商品、真实服务和真实顾客问题证明品牌价值。

门店不得：

* 代表总部修改品牌原则；
* 使用其他门店的顾客或现场；
* 冒充研发、QA或QC确认商品技术结论；
* 编造库存、客流、成交和顾客评价。

# 五、知识继承模型

笛语采用四层继承。

| 层级 | 名称    | 维护方          | 允许的内容                   |
| -- | ----- | ------------ | ----------------------- |
| L1 | 全国核心层 | 总部           | 品牌、商品核心事实、声纹、视觉、角色和强制规则 |
| L2 | 区域叠加层 | 区域代理         | 气候、城市、商圈、节庆、区域库存和经营案例   |
| L3 | 门店事实层 | 具体门店         | 人员、试穿、服务、陈列、活动和顾客匿名问题   |
| L4 | 数据反馈层 | 总部、区域、门店共同贡献 | 评论、咨询、到店、退换、投诉和内容表现     |

对应原则：

> L2只能叠加L1，不能覆盖L1；
> L3只能贡献本店可验证事实；
> L4先进入候选区，经过审核后才升级为正式知识。

参考文件明确要求核心知识采用“全国核心、区域叠加、门店事实、数据反馈”四层模型，并要求同一对象使用唯一主ID。

# 六、品牌与组织授权关系

## 6.1 总部授权区域

总部可以授权区域使用：

* 品牌公开基线；
* 总部批准的商品资料；
* 总部视觉素材；
* 内容母题；
* 内容任务书；
* 平台适配模板；
* 总部专业角色输出；
* 合规规则。

授权需明确：

* 是否允许原文使用；
* 是否允许改写；
* 是否允许增加区域事实；
* 可使用平台；
* 可使用账号；
* 使用期限；
* 是否允许下发门店。

---

## 6.2 区域授权门店

区域可以向所属门店授权：

* 总部已批准的品牌资料；
* 本区域活动；
* 区域天气和节庆资料；
* 区域视觉素材；
* 区域培训内容；
* 区域商品组合建议。

区域不得将以下内容授权给其他门店冒充使用：

* 某店顾客案例；
* 某店自采人物素材；
* 某店库存；
* 某店活动；
* 某店经营数据。

---

## 6.3 门店资料向上使用

门店资料默认只属于：

> 提交门店及其授权范围。

总部或区域希望复用时，需要确认：

* 门店是否同意；
* 相关人物是否同意；
* 顾客信息是否匿名；
* 是否可以跨门店、跨区域使用；
* 是否需要修改地点和身份说明；
* 使用期限。

门店贡献不是自动放弃权利，也不是自动成为全国通用事实。

# 七、组织与账号绑定

每个发布账号必须绑定：

1. 租户ID；
2. 组织节点ID；
3. 主ContentRole；
4. 可选辅助ContentRole；
5. 平台；
6. 账号名称；
7. 账号持有主体；
8. 管理员；
9. 操作人员；
10. 审核责任人；
11. 可调用资料范围；
12. 素材权限；
13. 生命周期状态。

## 7.1 禁止绑定方式

禁止：

* 一个账号同时代表总部、区域和门店；
* 一个门店账号绑定多个无关门店；
* 个人离职后继续默认代表组织；
* 账号持有权和密码控制不清；
* 发布账号与后台操作人员混为同一身份；
* 门店人员直接获得总部内部资料的完全读取权。

# 九、账号及IP资产权属

正式接入前必须确定：

* 账号注册主体；
* 手机号或邮箱控制权；
* 管理员；
* 密码和双重验证；
* 账号名称和头像权属；
* 粉丝和历史内容归属；
* 脚本和成片版权；
* 人物姓名、肖像和声音授权；
* 顾客线索和私域数据归属；
* 商业合作边界；
* 离职、调岗、解约后的处置；
* 封禁或危机时的接管方式。

这些也是参考文件要求前置确定的账号与个人IP最低资产条款。

# 十、组织节点生命周期

| 状态         | 含义        |
| ---------- | --------- |
| candidate  | 候选组织，尚未接入 |
| onboarding | 正在入驻      |
| active     | 正常参与内容生产  |
| restricted | 部分权限受限    |
| suspended  | 暂停使用      |
| terminated | 合作或经营关系终止 |
| archived   | 历史归档      |

组织进入 `terminated` 后：

* 停止新的品牌资料调用；
* 停止新内容生成；
* 冻结其账号操作权限；
* 处理未发布内容；
* 按合同处理账号、素材和顾客线索；
* 保留审计记录。

# 十一、第一阶段建议接入范围

第一阶段建议采用：

* 1个总部；
* 1个区域代理；
* 3—5家加盟门店；
* 1家总部直营门店。

不建议第一阶段直接接入全国所有门店。

参考文件同样主张总部先行，再选择少量成熟省区和优质终端验证，不应以全国同时铺开为目标。

# 十四、仍需品牌方补充的真实资料

## 企业主体

* 企业正式全称；
* 统一社会信用代码；
* 总部所在地；
* 品牌权利主体；
* 租户最高管理员。

## 区域代理

* 企业或经营主体名称；
* 省区；
* 负责人；
* 合作状态；
* 可管理门店范围；
* 可使用账号范围。

## 门店

* 门店名称；
* 门店ID；
* 所属区域；
* 地址；
* 商圈；
* 店长；
* 营业信息；
