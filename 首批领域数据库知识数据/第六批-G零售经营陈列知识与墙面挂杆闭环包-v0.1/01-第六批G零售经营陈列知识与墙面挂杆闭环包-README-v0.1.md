# 第六批 G 零售经营陈列知识与墙面挂杆闭环包 README v0.1

## 1. 交付结果

本批完成首期第二应用 `display-merchandising` 的最小纵向闭环：

> **首次挂杆条件建档 → 日常库存清单 → 完整文字执行方案 → 门店自然反馈 → 受影响部分局部修订**

首期唯一业务合同为：

```text
视觉陈列 × 卖场墙面 × 一组挂杆
contract_id = DM01_wall_hanging_rail_execution
```

本批不把墙面挂杆扩张为完整陈列百科，也不建设橱窗、模特、中岛、巡检、评分、整改或 ERP。

## 4. 首期用户体验

### 首次

门店自然说一次：

- 挂杆大致容量或分层容量；
- 上下杆与固定正挂结构；
- 主要来客方向；
- 现场硬限制。

### 日常

只提交本次实际可用商品及数量。没有特殊目标时，系统按品牌当前标准执行。

### 成品

系统只交付完整文字执行方案：

- 商品分组；
- 左右次序；
- 正挂重点；
- 侧挂节奏；
- 数量与间距；
- 替代办法；
- 执行步骤。

### 反馈

门店自然说明实际差异，系统只修订受影响部分并形成新版。

## 5. 导入纪律

1. 先注册来源；
2. 注册 `display-merchandising` 与 `DM01_wall_hanging_rail_execution`；
3. 导入资产到 `review_candidate`；
4. 注册夹具并校验交叉引用；
5. 由零售陈列专业人员复核 G 类；
6. 用黄金与真实门店文字执行样本验证；
7. 按资产逐批激活，禁止全量自动 active。

## 6. 不做事项

- 不识图巡检；
- 不评分；
- 不下整改任务；
- 不接 ERP；
- 不核验库存真值；
- 不建立审批状态机；
- 不把建议写成已执行；
- 不让 P5 与 DisplayArtifact 混用；
- 不预建其他目的、区域、载体的空合同；
- 不冻结 D-016 的表、JSON、对象或状态机。

## 7. 非阻断提醒

- `PRJ-ADR-018` 目前依据用户在当前会话提供的确认摘要建立；项目仓合并时需绑定实际 ADR-018 文件路径。
- G 类资产需要零售视觉陈列专业复核和真实门店样本。
- 旧版项目记忆和待决清单可能仍显示 D-005 未关闭，需由维护者同步。
