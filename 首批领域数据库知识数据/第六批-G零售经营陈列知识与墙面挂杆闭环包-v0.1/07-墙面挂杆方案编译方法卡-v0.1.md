# 墙面挂杆方案编译方法卡 v0.1

> 目标：把品牌标准、门店挂杆条件和本次库存编译成可直接照做的文字方案；保存方法而不是南城店固定答案。

- 状态：`review_candidate`
- 方法数量：14
- 方法族：`display_merchandising`
- 适用合同：`DM01_wall_hanging_rail_execution`

## 共同边界

- 默认只交付一个当前条件下的完整方案；不附高配/低配或多套同义方案。
- 信息足够时直接执行；只有缺少不可替代首次条件时追问一个问题。
- 人工库存按原意使用，不核验、不改数、不写回库存真值。
- 自然反馈只形成修订输入，不生成巡检、评分或整改流程。
- 具体字段、Agent 节点、表和状态机均未冻结。

## GM01｜首次建档与日常任务形成

### GM-INTAKE-001｜首次使用从自然说明建立最小挂杆条件档案

| 项目 | 内容 |
|---|---|
| 需求目录 | `GM01` |
| 资产类型 | 陈列编译方法 |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**适合使用**
- 门店首次为某一组墙面挂杆生成数据依赖执行方案
- 系统尚无该挂杆的容量结构、主要来客方向或固定限制

**避免使用**
- 已有有效挂杆档案且用户未说明条件变化
- 用户只要通用培训或概念建议

**最低输入**
- 目标门店与墙面挂杆组
- 大致容量或分层容量规则
- 上下/正侧挂结构
- 主要来客方向或可见方向
- 固定硬限制

**可选输入**
- 现场照片
- 主要视线带
- 固定正挂点位置
- 品牌当前默认目标

**编译步骤**
1. 从自然语言中提取可复用的固定条件
2. 只对会改变数量、焦点或左右结构的缺口提出一个问题
3. 生成一份用户可纠正的简短条件摘要
4. 经用户继续使用或明确认可后作为该挂杆后续任务的默认条件

**成功信号**
- 门店不填写复杂表单
- 固定条件在日常任务中不重复询问
- 缺少不可替代条件时只追问一次

**失败模式**
- 把内部字段做成长问卷
- 首次仅凭库存清单猜容量和动线
- 把一次临时限制写成永久档案

**依据引用**  
`PRJ-ADR-014`; `PRJ-ADR-018`; `PRJ-D005-GOLD-V03`

**验证夹具**  
`DM-V01`; `DM-V21`

**复核策略**  
出现黄金闭环失败、真实门店不可执行或越界时立即复核；至少每12个月结合执行样本抽样复核

- 系统不核验 ERP 真值但不改数
- 未说明目标也能按品牌标准生成

**失败模式**
- 每次要求重新填写门店条件
- 把清单称为实时库存或系统核验

**适合使用**
- 用户只给日常库存清单
- 品牌已有当前有效的墙面挂杆视觉陈列标准

**避免使用**
- 用户明确提出促销、主题、主推或其他特殊目标

**最低输入**
- 品牌当前标准

- 缺少特殊目标不造成追问

**失败模式**
- 无品牌标准仍声称品牌正式方案
- 把默认目标写成不可变模板

**适合使用**
- 存在两个或更多可成为正挂/重点的商品
- 需要建立入口阅读顺序

**避免使用**
- 品牌明确要求等强双焦点并有真实依据
- 只有单件商品且单件即可成立

**最低输入**
- 主要来客方向
- 固定正挂点
- 商品可见差异

**可选输入**

- 墙面有清楚第一入口
- 次级点能回应但不争夺
- 低饱和品牌也能靠真实关系形成重点

**失败模式**

## GM05｜空间映射

### GM-LAYOUT-001｜把搭配区映射为上下、左右和正侧挂结构

| 项目 | 内容 |
|---|---|
| 需求目录 | `GM05` |
| 资产类型 | 陈列编译方法 |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**适合使用**
- 墙面挂杆组具有上下杆、固定正挂或明确左右空间

**避免使用**
- 载体结构与当前合同不一致，如橱窗、模特或中岛
- 首次条件档案不完整且会改变布局

**最低输入**
- 搭配区
- 墙面结构
- 动线
- 固定正挂点
- 商品数量

**可选输入**
- 主要视线带
- 不可移动设施

**编译步骤**
1. 从顾客面对墙的位置明确左/右
2. 先放主正挂和回应正挂
3. 将搭配区垂直对应到上下杆
4. 再分配侧挂序列和停顿
5. 核对所有位置与实际设施一致

**成功信号**
- 门店无需猜左右和上下
- 每组上下关系清楚
- 没有把本场景规则偷渡到其他载体

**失败模式**
- 只给抽象色彩顺序
- 把上杆商品移到下杆凑数
- 忽略固定正挂点

## GM06｜数量、间距与抽取性

### GM-DENSITY-001｜按数量、厚度和抽取性决定间距与正侧挂比例

| 项目 | 内容 |
|---|---|
| 需求目录 | `GM06` |
| 资产类型 | 陈列编译方法 |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**适合使用**
- 方案需要明确上墙件数、间距和拿取便利
- 商品厚度或袖量可能影响容量

**避免使用**
- 仅生成非执行概念案
- 缺少任何载体容量或商品数量

**最低输入**
- 各组商品数量
- 挂杆容量规则
- 正挂占位
- 商品大致体积或可见反馈

**可选输入**
- 一衣架宽等门店经验单位
- 顾客/店员抽取频率

**编译步骤**
1. 先确保总数不超过输入和容量
2. 为主正挂和搭配区保留必要停顿
3. 按商品体积减少相邻侧挂
4. 用相对单位表达可执行间距
5. 最后检查是否便于抽取

**成功信号**
- 件数、间距和正侧挂比例可照做
- 容量不是填满目标
- 厚商品会自动降低密度

**失败模式**
- 只给“适当留白”
- 把固定厘米当全行业标准
- 视觉饱满压过拿取便利

## GM07｜替代策略

### GM-SUB-001｜为每个关键组生成最短替代梯度

| 项目 | 内容 |
|---|---|
| 需求目录 | `GM07` |
| 资产类型 | 陈列编译方法 |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**适合使用**
- 本次清单存在未上墙后备或执行中可能出现商品不可用
- 方案需要门店可自行处理小变化

**避免使用**
- 关键主焦点全部失效，需要重新生成
- 没有任何真实可用替代

**最低输入**
- 已上墙商品
- 未上墙商品
- 每件商品在方案中的职责

**可选输入**

- 品牌允许的相近商品关系

- 影响范围清楚

- 新条件
- 受影响区域

**失败模式**
