# G01—G08 零售经营陈列知识卡 v0.1

> 目标：只建设首期“视觉陈列 × 卖场墙面 × 一组挂杆”文字执行闭环真正消费的无品牌知识；不建设完整零售陈列百科，不把南城店黄金方案固化为行业模板。

- 状态：`review_candidate`
- 卡片数量：32
- 适用合同：`DM01_wall_hanging_rail_execution`
- 激活条件：陈列专业复核 + 黄金/变形测试 + 至少一轮真实门店文字执行样本

## 使用纪律

- G 类知识只服务 `display-merchandising` 的经营陈列成果；共享 A/B 知识可以被调用，但不能因此启动 P5。
- 确定性知识只在明确口径内断言；条件性规律必须保留条件、代价和不适用情形。
- 品牌陈列标准、具体商品事实、门店固定条件和本次库存不属于 G 通用知识。
- 所有具体件数、三搭配区、双面外套和“一衣架宽”仅属于黄金夹具，不能成为默认模板。

### G-FIXTURE-002｜主要来客方向会改变第一视觉入口与左右阅读顺序

| 项目 | 内容 |
|---|---|
| 需求目录 | `G02` |
| 主题 | `customer_flow` |
| 硬度 | 🟨 条件性专业规律 |
| 语义距离 | `commercial_core` |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**知识断言**  
当顾客主要从某一侧进入或接近墙面时，靠近该方向且位于主要视线位置的商品通常更容易成为第一视觉入口；动线变化时，主焦点位置与阅读顺序需要重新判断。

**何时使用**  
门店能说明主要来客方向，且墙面位于顾客可顺向观看的位置

**可能效果**  
帮助确定主焦点、次级回应和从左向右或从右向左的阅读次序

**为什么**  
顾客最先接近的位置、观察角度和停留路径会影响信息进入顺序

**代价/取舍**  
过度依赖单一动线可能忽略双向客流或局部遮挡，需要结合现场固定条件

**不适用/需谨慎**  
来客方向不稳定、墙面可从多方向同等接近，或固定设施使焦点无法随动线调整

**禁止误用**  
不得把“主焦点永远在左边”写成通用规则；动线改变时不能只机械镜像文字而不看固定正挂点。

## G03｜商品分组与搭配区

### G-GROUP-001｜视觉陈列分组应优先形成可读的搭配区，而不是纯品类库存区

| 项目 | 内容 |
|---|---|
| 需求目录 | `G03` |
| 主题 | `outfit_zone` |

| 语义距离 | `commercial_core` |
| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**知识断言**  
当任务目标是墙面视觉陈列时，可将上装、外套和下装按可理解的穿着关系组织为若干搭配区；仅按衬衫、裤子、裙子集中排列容易退化为库存分类墙。

**何时使用**  
商品之间存在可被顾客理解的穿着或视觉关系，且墙面具备上下或左右分区条件

**可能效果**  
顾客更容易把单件商品读成可穿组合，并理解主商品的搭配方向

**为什么**  
搭配区同时提供商品关系、阅读顺序和穿着想象

**代价/取舍**

**禁止误用**  
不得为制造“搭配感”虚构不存在的商品或把所有品类混在一组。

### G-GROUP-002｜上下层垂直对应可以强化一组穿着关系

| 项目 | 内容 |

| 支撑合同 | `DM01_wall_hanging_rail_execution` |
| 状态 | `review_candidate` |

**知识断言**  
当墙面具有上杆和下杆时，把上方外套/上装与下方裤裙在垂直方向对应，可帮助顾客把它们读成一组；下装全部集中到一侧会削弱这种关系。

**何时使用**  
上下杆位置可对应，且商品长度与拿取空间允许

**可能效果**  
形成清楚的 A/B/C 搭配区和由上至下的穿着阅读

**不适用/需谨慎**  
墙面只有单杆；上下杆严重错位；商品长度遮挡下杆

**禁止误用**  
不得把垂直对应解释成必须成套销售；不得为了对应而把不相关商品硬配。

**禁止误用**
