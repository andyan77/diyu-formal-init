# E 稳定媒介与制作语法方法卡 v0.1

> 目标：保存跨平台、跨媒体可复用的制作与表现方法；回答在当前真实资源下怎样把一个主要 P1—P5 合同编译为完整成品，不保存平台规格、固定镜头清单或可直接照抄的拍法模板。

- 状态：`review_candidate`
- 方法数量：28
- 激活条件：P1—P5 黄金/回归验证 + 真实门店/团队制作测试 + D-032 后续媒体校准


## E01｜制作资源 × 表现语法解耦

### E-RESOURCE-001｜把制作资源与表现语法分开判断

| 项目 | 内容 |
|---|---|
| 需求目录 | `E01` |
| 主题 | `resource_grammar_decoupling` |
| 资产类型 | 制作方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 系统需要把同一内容任务适配到单人、双人或团队条件
- 用户先说设备/人数，容易被误解成固定内容形式

**避免使用**
- 用户已经明确要求某种不可替代的成果且当前资源无法完成
- 权限、真实商品或素材使用边界尚未成立

**最低输入**
- 主要产品价值
- 当前可用人数、设备、空间与时间约束
- 可以选择的表现语法

**可选输入**
- 平台与发布位置
- 已有素材
- 可接受的制作复杂度

**编译步骤**
- 1. 先写清受众必须获得的主要价值
- 2. 选择能承担该价值的口播、演示、旁白、对话、纯视觉或序列语法
- 3. 再把语法映射到当前真实资源
- 4. 删除需要不存在人员、设备或空间的动作

**变化轴**
- 单人/双人/团队
- 固定/移动机位
- 真人/静物
- 实时/分段拍摄

**制作边界**
- 不为产品×平台×时长×制作方式建立固定矩阵

**成功信号**
- 单人不被自动限制为纯口播
- 团队资源不被自动要求故事化
- 主要产品价值在资源变化后保持

**失败模式**
- 把一人一手机等同于只能自拍口播
- 给团队任务硬加剧情和多外景
- 先选形式再寻找内容价值

**依据引用**  
`PRJ-ADR-014`；`PRJ-D032-BOUNDARY`；`PRJ-P5-CLOSE-20260721`；`EXP-BE-MEDIA-001`

**复核策略**  
出现不可制作、主要产品价值丢失、媒体实际性误导或明显模板化时立即复核；至少每12个月结合黄金任务与真实制作样本抽样复核

**最低输入**

- 不产生多余选择负担

**最低输入**

- 空间层次
- 光线控制
- 镜头运动
- 声音层次

**制作边界**

- 复杂度换来可见价值而非空洞大片感

**失败模式**
- 团队版自动变成品牌宣传片
- 复杂布景替代商品关系
- 把总部/门店差异写死成两套方法

## E02｜单人轻制作

### E-SOLO-001｜用固定机位和明确动作区建立单人可执行画面

| 项目 | 内容 |
|---|---|
| 需求目录 | `E02` |
| 主题 | `fixed_camera_action_zone` |
| 资产类型 | 制作方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 一人一手机或没有摄影师
- 需要完整身体、商品轮廓或动作关系被看见

**避免使用**
- 必须追踪高速移动或复杂空间关系
- 固定机位会遮挡关键商品细节且无法补拍

**最低输入**
- 可稳定放置手机的位置
- 人物/商品活动范围
- 承重动作或画面

**可选输入**
- 地面标记
- 遥控或定时启动
- 自然光方向

**编译步骤**
- 1. 先确定画面必须覆盖的身体/商品范围
- 2. 在空间中划定可重复动作区
- 3. 安排进入、停留、转身和退出中的最少动作
- 4. 用补拍细节解决固定机位看不清的部分

**变化轴**
- 全身/半身
- 正面/侧面
- 静止/走入
- 人物/静物

**制作边界**
- 不假设有人移动跟拍

**成功信号**
- 一个人可以独立完成
- 关键动作不会出画
- 商品关系清楚且背景干扰可控

**失败模式**
- 脚本出现无人可执行的跟拍
- 人物不断出画
- 机位只方便拍摄却看不清承重关系

**依据引用**  
`PRJ-P5-CLOSE-20260721`；`PRJ-D032-BOUNDARY`；`EXP-BE-SOLO-001`

### E-SOLO-002｜单人多景别优先靠主体距离、方向与补拍变化完成

| 状态 | `review_candidate` |

**适合使用**
- 需要全身、局部和商品细节，但只有固定手机
- 视觉命题需要从整体过渡到细节或反向展开

**避免使用**
- 距离变化会造成严重失焦或商品颜色失真
- 用户当前空间无法安全移动

**最低输入**
- 固定机位
- 可移动距离
- 整体与细节各自要证明什么

**可选输入**
- 后续独立补拍的手部/面料/开合细节

**编译步骤**
- 1. 用人物接近或离开改变景别
- 2. 整体动作保持在主机位完成
- 3. 停止主动作后单独补拍关键细节
- 4. 剪辑只保留支持同一主线的景别变化

**变化轴**
- 远→近
- 近→远
- 全身→手部
- 商品静物→上身

**制作边界**
- 不把数字变焦当作唯一细节方案

**成功信号**
- 无需摄影师即可获得必要景别
- 景别变化有信息作用
- 整体与细节互相证明

**失败模式**
- 频繁靠近只制造动感
- 细节与整体不属于同一商品状态
- 数字变焦严重损失真实外观

### E-SOLO-003｜把切点放在可重复动作上，用动作连续保持关系清楚

| 项目 | 内容 |
|---|---|
| 需求目录 | `E02` |
| 主题 | `action_continuity_cut_point` |
| 资产类型 | 制作方法 |

| 状态 | `review_candidate` |

**适合使用**
- 需要在同一动作中切换商品面、穿着状态、景别或场景
- 单人制作希望降低复杂转场难度

**避免使用**
- 动作不能稳定复现
- 切换会让商品状态或事件真实性被误解

**最低输入**
- 可重复动作
- 切换前后的主要变量
- 需要保持不变的条件

**可选输入**
- 地面/身体位置标记
- 动作起止点

**编译步骤**
- 1. 选择自然且能重复的动作节点
- 2. 记录切换前动作的方向与速度
- 3. 切换后从相同动作位置继续
- 4. 检查观众是否能看清变量而不是只注意转场

**变化轴**
- 手入袋
- 系扣
- 转身
- 走入/停步
- 抬手/放下

**制作边界**
- 动作节点换面只是可选实现，不得固化为 P5 模板

**成功信号**
- 动作连续帮助看清变化
- 转场不抢走商品关系
- 一人可重复完成

**失败模式**
- 动作对位不准像普通换装
- 每条内容都复制同一转场
- 用转场掩盖商品差异不足

**依据引用**  
`PRJ-P5-CLOSE-20260721`；`PRJ-CDF-D`；`EXP-BE-SOLO-001`

- 靠墙/走道/门口
- 浅景/深景
- 人物/静物

**成功信号**
- 空间服务主线而不喧宾夺主
- 商品外观可辨
- 没有冒充已执行经营陈列或现场结果

**失败模式**
- 用门店背景就声称当前陈列方案已执行
- 为了氛围让商品颜色失真
- 背景顾客和信息未经处理

### E-ACTUALITY-001｜区分概念方案、摆拍捕捉与实际捕捉，不把故事真实性和媒体真实性混成一维

| 项目 | 内容 |
|---|---|
| 需求目录 | `E05` |
| 主题 | `media_actuality` |
| 资产类型 | 制作方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 成品包含待拍概念、摆拍画面、真实现场或混合素材
- 用户可能据此判断是否已经实拍或现场发生

**避免使用**
- 为每个普通镜头制造用户操作负担
- 把真实事件等同于所有画面都必须是实际抓拍

**最低输入**
- 成品默认媒体实际性
- 少数例外素材/镜头
- 用户需要理解或执行的差异

**可选输入**
- 自然语言说明
- 媒体实际性例外

**编译步骤**
- 1. 为整体设定默认实际性
- 2. 只记录少数不同的例外
- 3. 在会影响理解时自然说明概念/摆拍/实际
- 4. 不得把概念方案冒充已拍、已执行或真实商品外观

**变化轴**
- concept_plan
- staged_capture
- actual_capture
- 混合由默认+例外派生

**制作边界**
- 精确字段和值域由 D-030/D-032，持久化由 D-016 决定

**成功信号**
- 真实故事可以使用摆拍画面
- 概念成品可完整交付但不冒充实拍
- 不要求逐镜头手工标注

**失败模式**
- 真实事件=全部actual
- 概念图直接写成门店现场
- 选择mixed却不说明哪些部分不同

**依据引用**  
`PRJ-ADR-002`；`PRJ-ADR-014`；`PRJ-P5-CLOSE-20260721`；`EXP-BE-MEDIA-001`

**复核策略**  
出现不可制作、主要产品价值丢失、媒体实际性误导或明显模板化时立即复核；至少每12个月结合黄金任务与真实制作样本抽样复核

**验证夹具**  
`BE-V07`；`BE-V30`

### E-MEDIA-001｜媒体进入系统的来源路径与画面实际性保持正交

| 项目 | 内容 |
|---|---|
| 需求目录 | `E05` |
| 主题 | `media_provenance_vs_actuality` |
| 资产类型 | 制作方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 图片/视频可能来自用户上传、组织素材、系统生成、合成或外部连接器
- 需要避免把“用户上传”误解为真实抓拍

**避免使用**
- 试图在本批冻结完整媒体来源枚举、权利模型或支持矩阵
- 用来源路径推定素材已获授权

**最低输入**
- 媒体引用
- 进入系统的路径
- 已知实际性/创作方式
- 当前可用权限

**可选输入**
- 权利说明
- 清理版引用
- 组织/私人素材空间

**编译步骤**
- 1. 分别记录或理解来源路径与实际性
- 2. 不从上传方式推断真实/摆拍/合成
- 3. 不从实际拍摄推断具备跨账号授权
- 4. 只在当前权限内调用

**变化轴**
- 个人上传/组织官方/系统生成/未来连接器
- 实拍/摆拍/概念/合成

**制作边界**
- 支持词表、清理矩阵和持久化留给 D-016/D-030/D-032

**成功信号**
- 来源与实际性不互相推定
- 私人/组织边界不因生成而放宽
- 不会把合成或公共素材猜补成商品事实

**失败模式**
- 用户上传=真实现场
- 组织素材=已获所有用途授权
- 系统生成=概念已执行

**依据引用**  
`PRJ-ADR-008`；`PRJ-ADR-009`；`PRJ-ADR-014`；`EXP-BE-MEDIA-001`

**复核策略**  
出现不可制作、主要产品价值丢失、媒体实际性误导或明显模板化时立即复核；至少每12个月结合黄金任务与真实制作样本抽样复核

**验证夹具**  
`BE-V30`
