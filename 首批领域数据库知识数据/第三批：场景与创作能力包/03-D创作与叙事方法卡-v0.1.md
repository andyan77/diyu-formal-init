# D 创作与叙事方法卡 v0.1

> 目标：保存内容编译方法，回答何时使用、怎样转换和如何失败；不保存可直接复制的整篇模板，不把故事、问答、幽默或镜头当成顶层内容产品。

- 状态：`review_candidate`
- 方法数量：16
- 激活条件：P1—P5 黄金任务验证 + 反模板/反越权回归通过

## D01｜直接判断

### D-DIRECT-001｜先给可执行结论，再补决定条件

| 项目 | 内容 |
|---|---|
| 需求目录 | `D01` |
| 主题 | `direct_answer` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P4 经营关系 |
| 状态 | `review_candidate` |

**适合使用**
- 问题清楚，且主要答案可以在当前条件下直接形成
- 受众最需要先知道“优先怎样做/怎样看”

**避免使用**
- 缺失信息会改变主要答案
- 任务需要先澄清应用或主要产品
- 存在权限或不可替代硬事实缺口

**最低输入**
- 当前问题或认知任务
- 决定答案的关键条件
- 至少一个不适用边界

**编译步骤**
1. 用一句话给出当前最优结论
2. 紧接着说明决定结论的1—3个关键条件
3. 补充改变结论的情形或反选边界
4. 给出最短下一步验证或行动

**成功信号**
- 受众在首个主要信息段即可复述结论
- “取决于什么”与“何时改变”都清楚
- 没有为了像内容而强加故事

**失败模式**
- 长篇铺垫后才回答
- 只说“因人而异”而不给判断
- 固定三段式套在所有任务上
- 结论与下一步脱节

**依据引用**  
`PRJ-ADR-014`; `PRJ-MOTHER-V03`; `EXP-D-METHODS-001`

**复核策略**  
出现黄金任务失败、角色越权、刻板化或明显模板化时立即复核；至少每12个月结合黄金任务与真实样本抽样复核

### D-DIRECT-002｜把专业结论转成用户能执行的一个动作

| 项目 | 内容 |
|---|---|
| 需求目录 | `D01` |
| 主题 | `actionable_next_step` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P4 经营关系 |
| 状态 | `review_candidate` |

**适合使用**
- 受众知道原因仍不知道下一步怎样选、试、看或回应
- 任务可以通过一个低成本动作验证

**避免使用**
- 动作需要系统未具备的交易、实时库存或外部执行能力
- 行动会造成不可逆结果却缺少必要信息

**最低输入**
- 已形成的结论
- 当前可用资源与能力
- 能够验证或推进结论的动作

**编译步骤**
1. 选择一个最小动作：试穿、对比、观察、询问、到店确认或删去一项主张
2. 说明动作要观察什么
3. 说明出现什么结果时应改变原判断

**成功信号**
- 行动与结论直接相连
- 不需要用户补填内部字段
- 不把咨询/到店写成隐性强促销

**失败模式**
- 泛泛写“可以试试看”
- CTA与正文无关
- 暗示系统已有库存/预约/交易能力
- 一次给出过多操作清单

**依据引用**  
`PRJ-ADR-014`; `PRJ-ADR-006`; `EXP-D-METHODS-001`

**复核策略**

## D02｜条件性专业解释

### D-EXPLAIN-001｜条件—效果—原因—代价—不适用的专业解释

| 项目 | 内容 |
|---|---|
| 需求目录 | `D02` |
| 主题 | `conditional_explanation` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 需要把条件性专业规律转成普通受众可理解的内容
- 只给结论会造成绝对化或误解

**避免使用**
- 没有任何适用条件或真实锚点
- 任务只是情绪/视觉表达，专业解释会抢走主要价值

**最低输入**
- 当前专业规律
- 适用条件
- 可能效果
- 原因
- 代价或限制
- 不适用情形

**编译步骤**
1. 先用人话指出在什么条件下可能发生
2. 解释受众能感知的效果
3. 只保留支撑理解的原因
4. 主动承认代价、反例或改变条件
5. 把解释嵌入当前成品主线而非单独上课

**成功信号**
- 条件没有丢失
- 受众能理解为什么不是人人都适用
- 专业信息推进主要产品价值而非术语堆砌

**失败模式**
- 把可能写成一定
- 只讲收益不讲代价
- 用专业词替代解释
- 每篇都形成课堂五段式

**依据引用**  
`PRJ-ADR-014`; `PRJ-MOTHER-V03`; `EXP-D-METHODS-001`

**复核策略**  
出现黄金任务失败、角色越权、刻板化或明显模板化时立即复核；至少每12个月结合黄金任务与真实样本抽样复核

### D-EXPLAIN-002｜品牌确认理由与系统专业解释分层

| 项目 | 内容 |
|---|---|
| 需求目录 | `D02` |
| 主题 | `official_vs_inference` |
| 资产类型 | 生成方法 |
| 支撑产品 | P2 解释商品；P1 帮助选择；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 商品资料可能包含品牌确认的设计/开发理由
- 资料没有官方理由，但商品特征允许条件性专业解释

**避免使用**
- 系统只凭通用知识就想替品牌说“我们为什么这样设计”
- 解释需要未知测试、成本、研发或销量事实

**最低输入**
- 当前商品事实
- 是否存在品牌确认理由
- 可用的条件性专业知识

**编译步骤**
1. 有确认理由时，以明确归属的品牌口径使用并保持原意
2. 没有确认理由时改用“这种结构通常可能……”的专业解释
3. 把解释限制在已知商品特征和适用条件内
4. 删除任何会被理解为品牌内部动机的无依据句子

**成功信号**
- 受众能区分品牌事实与系统解释
- 没有理由时内容仍有价值而不缩成资料朗读
- 换商品后解释会实质变化

**失败模式**
- 编造设计师原意
- 把条件性规律写成品牌承诺
- 为了避免风险只复述参数
- 在成品中展示审计标签

**依据引用**  
`PRJ-ADR-014`; `PRJ-MOTHER-V03`; `EXP-D-METHODS-001`

**复核策略**  
出现黄金任务失败、角色越权、刻板化或明显模板化时立即复核；至少每12个月结合黄金任务与真实样本抽样复核

## D03｜认知反转

### D-REVERSAL-001｜用真实变化修正一个旧判断

| 项目 | 内容 |
|---|---|
| 需求目录 | `D03` |
| 主题 | `cognitive_reversal` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P3 建立人格 |
| 状态 | `review_candidate` |

**适合使用**
- 受众或人物存在明确旧判断
- 当前商品、事件或观察提供了真实且可解释的变化

**避免使用**
- 只有标题需要刺激但没有实际变化
- 反转必须靠新增虚构结果或夸张承诺成立

**最低输入**
- 旧判断
- 具体变化或反例
- 新的可成立理解
- 不能被改变的承重事实

**编译步骤**
1. 简洁呈现旧判断及其合理来源
2. 给出真正发生或可观察的变化
3. 用专业解释或人格反应说明为什么需要修正
4. 保留仍然成立的边界，而不是把旧判断完全妖魔化

**成功信号**
- 反转来自内容本身而非标题
- 受众能复述旧判断为何需要修正
- 没有新增交易、身份或商品事实

**失败模式**
- 假反转
- 把普通结果夸成奇迹
- 用“原来不是X而是Y”机械套句
- 末尾变成万能人生金句

**依据引用**  
`PRJ-MOTHER-V03`; `PRJ-ADR-014`; `EXP-D-METHODS-001`

**复核策略**

### D-REVERSAL-002｜让动作、对比或画面承担认知变化

| 项目 | 内容 |
|---|---|
| 需求目录 | `D03` |
| 主题 | `observable_reversal` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P5 创造欲望 |
| 状态 | `review_candidate` |

**适合使用**
- 核心新理解可以通过穿着动作、前后对比、商品细节或视觉关系被看见
- 仅口头解释会削弱说服力

**避免使用**
- 没有真实商品或可观察差异
- 对比需要伪造穿着结果、身体变化或交易反馈

**最低输入**
- 要修正的旧判断
- 可观察变量
- 保持不变的对照条件
- 当前制作条件

**编译步骤**
1. 确定一个承重可见变量
2. 尽量保持其他条件一致
3. 让画面/动作先呈现差异
4. 文字只解释已经被看见的变化
5. 若无法拍摄则改为概念方案并诚实标记

**成功信号**
- 移除承重视觉后主要价值明显下降
- 对比变量清楚
- 不会把拍摄设计冒充实际结果

**失败模式**
- 只靠字幕说发生了变化
- 前后场景条件完全不同
- 用滤镜/角度伪造效果
- 把当前黄金拍法固化为模板

**依据引用**  
`PRJ-P5-CLOSE-20260721`; `PRJ-MOTHER-V03`; `EXP-D-METHODS-001`

## D04｜观察转观点与人格反应

### D-PERSONA-001｜从弱生活种子提取一条人格反应主线

| 项目 | 内容 |
|---|---|
| 需求目录 | `D04` |
| 主题 | `persona_reaction` |
| 资产类型 | 生成方法 |
| 支撑产品 | P3 建立人格；P4 经营关系 |
| 状态 | `review_candidate` |

**适合使用**
- 用户只给一句生活、工作、家庭或情绪种子
- 种子中存在可被账号注意、判断或对待的一项具体反应

**避免使用**
- 找不到自然品牌/账号关系
- 需要编造账号履历、家庭成员或重大经历才能成立

**最低输入**
- 任务种子
- 品牌战略基线
- PublishingAccount + ContentRole
- 目标受众

**编译步骤**
1. 从种子中列出可能的观察、矛盾和反应
2. 只选一个最值得受众观看的主线
3. 用品牌基线决定为何值得说
4. 用账号位置决定谁能怎样说
5. 给受众一个具体理解、共鸣、幽默或继续认识账号的理由

**成功信号**
- 受众能多认识账号一点
- 删除品牌名后内容仍值得看，但换品牌/账号后主线会变化
- 不强制商品、故事或商业动作

**失败模式**
- 把所有情绪堆进一篇
- 强行升华
- 品牌只在片尾出现
- 用实际操作人的私人生活替账号编履历

**依据引用**  
`PRJ-MOTHER-V03`; `PRJ-ADR-014`; `EXP-D-METHODS-001`

**复核策略**  
出现黄金任务失败、角色越权、刻板化或明显模板化时立即复核；至少每12个月结合黄金任务与真实样本抽样复核

### D-PERSONA-002｜保留未解决的生活毛边，不强迫给答案

| 项目 | 内容 |
|---|---|
| 需求目录 | `D04` |
| 主题 | `open_ended_persona` |
| 资产类型 | 生成方法 |
| 支撑产品 | P3 建立人格 |
| 状态 | `review_candidate` |

**适合使用**
- 种子的价值来自矛盾、迟疑、尴尬或尚未解决
- 账号人格可以通过怎样对待未完成而被识别

**避免使用**
- 用户明确要求决策建议或问题解决，应路由 P1/P4
- 内容只有模糊情绪，没有具体观察

**最低输入**
- 未解决事件或矛盾
- 账号对人的基本姿态
- 受众可能获得的关系回报

**编译步骤**
1. 保持事件的未完成状态
2. 选择一个具体细节或反应作为承重
3. 让账号表达理解、边界或态度
4. 在有价值的位置停下，不用金句替用户解决生活

**成功信号**
- 结尾不解决但仍有观看回报
- 受众理解账号怎样对待人和不确定
- 没有假共情和空泛祝福

**失败模式**
- 强行励志
- 给出未经请求的人生建议
- 用留白掩盖没有内容
- 反复说同一个情绪

**依据引用**

## D05｜问答、评论与公开回应

### D-RESPONSE-001｜把私人提问编译为有公共观看价值的回应

| 项目 | 内容 |
|---|---|
| 需求目录 | `D05` |
| 主题 | `public_response` |
| 资产类型 | 生成方法 |
| 支撑产品 | P1 帮助选择；P2 解释商品；P4 经营关系 |
| 状态 | `review_candidate` |

**适合使用**
- 评论、搜索问句或门店问题代表更广泛的受众困惑
- 回应可以在保护个人信息的同时帮助更多人

**避免使用**
- 问题只涉及个别人的敏感事实
- 回应会变成客服工单、售后政策或私人争论

**最低输入**
- 原问题或评论
- 可公开的共性问题
- 账号合法回应位置
- 可提供的帮助或边界

**编译步骤**
1. 去除可识别个人细节
2. 提炼更多人也会遇到的共同张力
3. 从品牌账号合法位置给出具体回应
4. 让未参与原评论的人也能带走判断、理解或关系许可

**成功信号**
- 不依赖原评论上下文也能看懂
- 没有暴露评论者或建立 CRM
- 不是只回答一句“欢迎私聊”

**失败模式**
- 复述评论后自我表扬
- 用一个人代表全部顾客
- 泄露个人信息
- 只做客服话术或搜索摘要

`PRJ-MOTHER-V03`; `PRJ-ADR-011`; `EXP-D-METHODS-001`

- 主要成果是让品牌/账号进入当地日常并推进关系

**避免使用**
