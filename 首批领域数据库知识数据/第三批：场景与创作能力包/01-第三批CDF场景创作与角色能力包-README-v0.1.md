## 使用纪律

- C 卡只保存非个人化场景因素和创意方向，不建立人群刻板画像、人物实体、顾客故事库或 CRM。
- D 卡是生成方法，不是成品模板；任何方法都必须服从当前唯一主要 P1—P5 合同。
- F 卡只是冷启动原型；企业确认版本优先，模板升级不得静默覆盖。
- 总部、区域、终端的角色差异是发声资格和权威范围，不是知识读取权限。
- 真实种子允许充分增益，但身份、关系、事件结果、时间地点数量和硬事实必须忠实。
- P3 的真诚、自然口语和不完美不等于故意粗糙、错字、结巴、低信息量或万能金句。
- P4 的在地性必须实质改变处境、判断、行动或关系姿态，不能只换城市名、方言和地标。
- 本批不冻结平台时长、镜头数、图片数、媒体清单、本体类、数据库表、DIFY 节点或模型提示词。
- `review_candidate → active` 必须经过反刻板化复核、角色越权检查和相关黄金/变形测试。
