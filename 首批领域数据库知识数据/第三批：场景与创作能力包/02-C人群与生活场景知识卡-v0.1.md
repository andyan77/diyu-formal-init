## C01｜通勤场景张力

### C-COMMUTE-001｜通勤通常是多阶段任务，不是单一“上班场合”

| 项目 | 内容 |
|---|---|
| 需求目录 | `C01` |
| 主题 | `multi_stage_commute` |
| 硬度 | 🟨 条件性专业规律 |
| 语义距离 | `audience_life` |
| 支撑产品 | P1 帮助选择；P3 建立人格；P4 经营关系 |
| 状态 | `review_candidate` |

**知识断言**  
一次通勤可能同时包含移动、到岗、久坐、会议、临时外出和下班后的第二任务；正式度、活动量、温差、收纳与切换成本需要共同判断。

**何时使用**  
内容或选择任务涉及从出门到返程的连续一天，而非只看某个静态场合

**可能效果**  
更容易发现哪一段任务代价最高，并形成有边界的选择、观察或关系回应

**为什么**  
不同阶段对活动、体面、温度和携带的要求可能互相冲突

**代价/取舍**  
为兼顾多阶段可能牺牲某一场景的极致正式、极致轻便或极致保暖

**不适用/需谨慎**  
用户只明确一个单一场景；或任务与穿着、生活切换无关

**禁止误用**  
不得把“职业女性都忙、都需要万能通勤装”写成群体事实；不得替用户虚构会议、接娃或加班。

**适用条件摘要**
- scene_conditions: 至少存在两个连续生活/工作阶段

**依据引用**  
`PRJ-MOTHER-V03`; `PRJ-ADR-014`; `EXP-C-SCENES-001`

**复核策略**  
出现黄金任务失败、角色越权、刻板化或明显模板化时立即复核；至少每12个月结合黄金任务与真实样本抽样复核

### C-CARE-002｜代际穿衣分歧可以被理解为优先级差异，而不是谁更正确

## C03｜旅行与移动场景

### C-TRAVEL-001｜旅行中的复用价值来自组合兼容和使用条件，不是“百搭”口号

|---|---|
| 需求目录 | `C03` |
| 主题 | `packing_reuse` |
| 硬度 | 🟨 条件性专业规律 |
| 语义距离 | `audience_life` |
| 支撑产品 | P1 帮助选择；P5 创造欲望 |
| 状态 | `review_candidate` |

**知识断言**  
行李受限时，一件单品的价值取决于它能否与已带单品形成足够组合，并适应温差、活动、护理和场合跨度；可复用不等于在所有场景都最佳。

**何时使用**  
用户明确行李、件数、行程或重复穿着限制

**可能效果**  
可形成围绕真实约束的选择判断或一条清楚的造型可能

**为什么**  
有限行李会放大组合兼容、护理恢复、体积和多场景转换的重要性

**代价/取舍**  
更高复用常以牺牲某一场景的专门性、视觉变化或极端天气性能为代价

**不适用/需谨慎**  
目的地、季节或活动完全未知且会改变结论；或系统只能用“旅行必备”模板

**禁止误用**  
不得把“百搭”当作无条件优点；不得虚构目的地天气、行程、预算或洗衣条件。
