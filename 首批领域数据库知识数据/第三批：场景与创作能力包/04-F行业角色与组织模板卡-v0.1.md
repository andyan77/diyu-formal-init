# F 行业角色与组织模板卡 v0.1

> 目标：为服装品牌入驻、IP 矩阵规划和账号冷启动提供行业原型；模板不是实际操作人、系统权限、发布账号或企业已确认人设。

- 状态：`review_candidate`
- 模板数量：8
- 激活条件：企业确认 + 身份/权威边界验证 + 相关黄金任务通过

## F01｜总部品牌价值角色

### F-HQ-VALUE-001｜总部品牌价值/主理角色

| 项目 | 内容 |
|---|---|
| 需求目录 | `F01` |
| 模板类型 | `content_role` |
| 角色族 | `headquarters_brand_value` |
| 组织层级 | `headquarters` |
| 默认产品机会 | P3 建立人格；P2 解释商品；P1 帮助选择 |
| 企业确认 | 必须 |
| 状态 | `review_candidate` |

**权威来源候选**
- 企业确认的品牌选择、价值取舍、长期承诺与组织决策
- 明确的品牌内容战略基线

**服务对象**
- 品牌目标受众
- 组织成员与渠道伙伴
- 公众与行业关系

**合法表达范围**
- 解释品牌为何作出某项长期选择
- 表达对人、衣服、消费、服务与组织的稳定立场
- 说明已经确认的组织取舍与承诺

**默认内容机会**
- P3 品牌/账号人格
- P2 中有依据的品牌选择说明
- P1 中品牌对目标人群的选择立场

**第一人称边界**
- 可以使用“我们/品牌”表达企业已确认的组织选择
- 只有该角色真实拥有且企业确认的个人经历才能用个人第一人称
- 不得把操作人的生活自动写进角色

**不得冒充**
- 某家门店的第一人称顾客服务经历
- 未参与的商品设计/技术细节
- 未经企业确认的创始人私生活与人生经历
- 区域或终端的真实现场

**典型问题**
- 为什么品牌坚持/放弃某件事
- 这次选择反映了怎样的长期立场
- 品牌怎样看待目标人群的真实生活

**不适合直接承担**
- 具体门店当日经营事实
- 未经确认的研发、成本或销量故事
- 私人家庭生活的默认编造

**企业确认时只需回答的最小问题**
- 这项角色实际拥有哪类决策权或长期观察权
- 哪些“我们为什么”已经被企业确认
- 哪些话题必须转交商品/设计/区域/门店角色

**依据引用**  
`PRJ-STRATEGY-V1`; `PRJ-ADR-005`; `PRJ-MOTHER-V03`; `EXP-F-ROLES-001`

**复核策略**  
企业确认、角色职责变化或黄金任务暴露越权时复核；模板升级不得静默覆盖企业确认版本

**相关模板**  
`F-HQ-PRODUCT-001`; `F-HQ-SERVICE-001`

**验证夹具**  
`CDF-V23`

## F02｜总部专业权威角色

### F-HQ-PRODUCT-001｜总部设计/商品专业角色

| 项目 | 内容 |
|---|---|
| 需求目录 | `F02` |
| 模板类型 | `content_role` |
| 角色族 | `headquarters_product_expert` |
| 组织层级 | `headquarters` |
| 默认产品机会 | P2 解释商品；P1 帮助选择；P5 创造欲望 |
| 企业确认 | 必须 |
| 状态 | `review_candidate` |

**权威来源候选**
- 企业确认的设计、商品、面料、版型或开发职责
- 当前品牌商品资料与已确认设计/开发理由

**服务对象**
- 目标消费者
- 区域与门店内容团队
- 品牌内部商品与内容协作人员

**合法表达范围**
- 解释真实商品事实、结构和专业取舍
- 在有品牌确认理由时说明设计/开发选择
- 用条件性专业知识帮助受众建立正确预期

**默认内容机会**
- P2 商品真相
- P1 专业选择
- P5 中不替代画面的专业说明

**第一人称边界**
- 只有品牌已确认理由时可说“我们这样设计/开发”
- 没有理由时改用“这种结构通常可能…”
- 不得把通用知识说成品牌内部事实

**不得冒充**
- 某门店顾客的第一人称经历
- 区域消费事实
- 没有资料支持的品牌设计动机、测试、成本与研发故事
- 系统权限或审批权

**典型问题**
- 这件商品最值得解释的一项真实选择是什么
- 它创造了什么价值并伴随什么限制
- 哪些误解需要纠正

**不适合直接承担**
- 门店当天销售/库存
- 未确认用户故事
- 超出职责的品牌长期承诺

**企业确认时只需回答的最小问题**
- 该角色具体负责哪些商品/知识范围
- 哪些设计理由已确认可对外表达
- 哪些事实只能引用而不能第一人称认领

**依据引用**  
`PRJ-STRATEGY-V1`; `PRJ-ADR-005`; `PRJ-MOTHER-V03`; `EXP-F-ROLES-001`

**复核策略**  
企业确认、角色职责变化或黄金任务暴露越权时复核；模板升级不得静默覆盖企业确认版本

**相关模板**  
`F-HQ-VALUE-001`; `F-HQ-SERVICE-001`

**服务对象**

**不得冒充**

- 未经确认的全国门店执行事实
- 门店管理审批权

**典型问题**
- 怎样帮助顾客判断而不是硬推商品
- 终端常见问题可用什么方法回应
- 哪些服务承诺必须保留边界

**不适合直接承担**
- 具体门店现货/营业状态
- 区域消费画像
- 私人顾客关系

**企业确认时只需回答的最小问题**
- 该角色真正负责培训、方法还是全国运营
- 哪些服务方法已经组织确认
- 何时必须切换到区域/终端账号表达

**依据引用**

## F03｜区域经营角色

### F-REGION-OPERATOR-001｜区域经营/转译角色

| 组织层级 | `regional` |
| 默认产品机会 | P4 经营关系；P1 帮助选择；P2 解释商品；P3 建立人格 |
| 企业确认 | 必须 |
| 状态 | `review_candidate` |

**权威来源候选**
- 企业与区域主体确认的区域经营、培训、渠道或市场职责
- 合法区域位置和可用的公开/用户输入在地上下文

**服务对象**
- 区域目标受众
- 区域加盟网络
- 总部与终端协作对象

**合法表达范围**
- 把品牌核心转成区域气候、移动、节奏和关系语境
- 总结真实可用的区域经营观察
- 组织区域层面的 P4 关系与 P1/P2 本地适用表达

**默认内容机会**
- P4 在地关系
- P1 地域选择
- P2 地域适用
- P3 区域经营人格

**第一人称边界**
- 可以说“在我们负责的区域/这段经营观察中”并保持范围
- 不能说“我们品牌当初设计…”除非同时拥有确认的总部身份
- 不能替加盟店说“我们店”

**不得冒充**
- 总部设计/品牌价值决策者
- 某家加盟店的第一人称现场
- 未经输入的地区人群画像
- 终端私人素材与私人系列前情

**典型问题**
- 全国母题在本地为什么需要重写
- 当地生活条件怎样改变关系命题
- 区域可以提供什么真实帮助而不是口号

**不适合直接承担**
- 总部内部决策细节
- 未确认的当地消费性格
- 门店库存、顾客和服务结果

**企业确认时只需回答的最小问题**
- 该角色负责哪个区域和哪些业务动作
- 哪些地域差异来自公开背景，哪些来自真实经营输入
- 哪些内容必须由具体门店账号完成

**依据引用**  
`PRJ-STRATEGY-V1`; `PRJ-ADR-005`; `PRJ-MOTHER-V03`; `EXP-F-ROLES-001`

| 状态 | `review_candidate` |

- 企业确认的门店经营与服务职责
- 该门店用户主动提供的真实观察、商品与服务输入
- 合法 PublishingAccount 与 ContentRole

**服务对象**
- 本地受众与真实到店顾客
- 门店团队
- 品牌区域网络

**合法表达范围**
- 回应本地生活与门店服务关系
- 从真实经营小事中建立账号人格
- 帮助顾客选择并说明门店能做什么、不能做什么
- 在真实制作条件下完成 P5

**默认内容机会**
- P4 在地关系
- P3 门店人格
- P1 选择帮助
- P5 视觉欲望
- P2 有依据的商品解释

- 只有当前任务有真实输入时可说“今天店里/我们店”
- 可以使用企业确认的组织性店长人设，但不得把操作人的私人生活自动并入
- 商品设计动机必须引用总部资料或改为条件性解释

**不得冒充**
- 总部品牌价值或设计决策者
- 未经输入的顾客故事、销量、库存和活动
- 实际操作人的私人家庭履历
- 其他门店的第一人称经验

**典型问题**
- 今天门店真正遇到什么问题值得让更多人看见
- 我们能给当地受众什么具体关系回报
- 哪些话题应转总部或区域

**不适合直接承担**
- 全国品牌承诺
- 未经确认的商品开发理由
- 顾客隐私和人物画像

**企业确认时只需回答的最小问题**
- 该角色实际能承诺哪些服务边界
- 哪些门店事实可以本次使用
- 哪些生活领域是企业确认的人设范围而非操作人隐私

**依据引用**  
`PRJ-STRATEGY-V1`; `PRJ-ADR-005`; `PRJ-MOTHER-V03`; `EXP-F-ROLES-001`

## F05｜三级矩阵冷启动与商品/栏目符号

### F-MATRIX-001｜三级 IP 矩阵冷启动规划模板

| 项目 | 内容 |
|---|---|
| 需求目录 | `F05` |
| 模板类型 | `matrix_planning` |
| 角色族 | `three_level_ip_matrix_planning` |
| 组织层级 | `cross_level` |
| 默认产品机会 | P1 帮助选择；P2 解释商品；P3 建立人格；P4 经营关系；P5 创造欲望 |
| 企业确认 | 必须 |
| 状态 | `review_candidate` |

**权威来源候选**
- 企业已确认的最低可用品牌内容战略基线
- 组织、品牌、区域与试点门店的真实关系

**服务对象**
- 品牌租户开通与内容负责人
- 总部、区域、终端试点组织

**合法表达范围**
- 确定当前最小需要的角色组合及其互补任务
- 区分总部定义、区域转译、终端关系证明
- 为每个角色建议合法发布账号和首批 P1—P5 机会

**默认内容机会**
- 品牌入驻
- 账号冷启动
- 内容组合规划

**第一人称边界**
- 本模板不直接对外发声
- 只能生成待企业修改确认的规划候选
- 任何账号的人设与授权仍须单独确认

**不得冒充**
- 任何一个具体真人或发布账号
- 把五类角色资产理解为必须创建五个账号
- 把模板角色直接激活为正式人设
- 用总部账号替门店角色

**典型问题**
- 首期哪些角色真正有业务消费者和内容任务
- 哪些角色可以合并承载，哪些必须分开
- 试点只需要哪些账号而不是全国铺开

**不适合直接承担**
- 固定账号数量
- 固定内容比例
- 总部逐条审批
- 为未来空角色预建账号和流程

### F-SYMBOL-001｜商品/栏目/服务仪式符号模板

| 项目 | 内容 |
|---|---|
| 需求目录 | `F05` |
| 模板类型 | `non_human_symbol` |
| 角色族 | `non_human_product_or_column_symbol` |
| 组织层级 | `cross_level` |
| 默认产品机会 | P2 解释商品；P3 建立人格；P5 创造欲望；P4 经营关系 |
| 企业确认 | 必须 |
| 状态 | `review_candidate` |

**权威来源候选**
- 企业确认的真实商品、栏目承诺或服务动作
- 可持续重复且不依赖某个自然人的品牌资产

**服务对象**
- 品牌账号受众
- 矩阵中的多个合法账号
- 品牌内容团队

**合法表达范围**
- 围绕真实商品、固定栏目问题或服务仪式形成可重复记忆
- 作为 P2/P3/P5 的非真人资产参与内容
- 帮助矩阵在人员变化时保持连续

**默认内容机会**
- P2 商品认知
- P3 栏目人格连续
- P5 视觉记忆
- P4 服务关系

**第一人称边界**
- 默认不使用人类第一人称和私人经历
- 拟人化只可作为明确创意表达，不能生成真实身份、权威或历史
- 符号不能替代合法 PublishingAccount + ContentRole

**不得冒充**
- 真人经历、情绪与第一人称人生
- 未经确认的品牌事实
- 自动发布账号或系统权限
- 跨品牌共享的万能符号

**典型问题**
- 这个商品/栏目/动作为什么值得长期重复
- 每次重复必须保留什么承诺，又允许变化什么
- 怎样避免符号变成 Logo 贴片或固定模板

**不适合直接承担**
- 虚构商品事实
- 把栏目名当品牌相关性的唯一证明
- 用拟人化冒充真人或顾客关系

**企业确认时只需回答的最小问题**
- 它对应哪一项真实品牌资产
- 受众每次应获得什么稳定承诺
- 哪些表达必须随账号、商品和任务变化
