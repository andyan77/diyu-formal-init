# 离线黄金回归与变形测试协议 v0.1

- 已注册夹具：98
- 既有 AB：17
- 既有 CDF：25
- 既有 BE：30
- 本批新增 H 系统/运行夹具：18
- 外部项目黄金引用桩：8

## 1. 三种执行层

### `offline_gold`

用于冻结产品合同、知识硬度和关键边界。必须保存：

- 冻结输入与账号/品牌/商品上下文；
- 必须保持的产品性质；
- 一个确认成品或确认结果；
- 最接近平庸但可发布的近邻；
- 必要的单变量检查。

黄金成品只冻结合同性质，不冻结句式、镜头、拍法、题材和平台格式。

### `offline_regression`

在模型、提示模板、检索、知识、规则或媒体编译发生实质变化时运行。只选择受影响夹具，不要求每次全量重跑。

### `offline_diagnostic_sample`

用于真实生产事件的必要抽样和长期成熟度观察。不得扩大企业对具体正文、私人素材或后台轨迹的查看权。

## 2. 单变量纪律

每次测试只改变一个承重变量，其他条件保持：

- 品牌；
- PublishingAccount / ContentRole；
- 商品；
- 地域；
- 创作权限；
- 目标应用；
- 媒体实际性；
- 分发策略；
- 事件/反馈状态。

若多个变量同时改变，结果只能用于探索，不能证明哪一条规则生效。

## 3. P5 六项关闭检查

1. 移除真实商品：不得继续 P5；
2. 替换商品：造型命题与承重画面重写；
3. 替换品牌：视觉重音、空间、动作、时间结构实质变化；
4. 移除视觉承重：退出 P5，必要时转 P3；
5. 切换对内陈列：退出 P5，进入经营陈列；
6. 切换媒体实际性：概念不得冒充实拍。

对应主要夹具：`BE-V02`、`BE-V03`、`BE-V04`、`BE-V05`、`BE-V06`、`BE-V07`，并由 `AB-V13`—`AB-V16` 交叉验证。

## 4. 发布版本回归的最小证据

| 结论 | 要求 |
|---|---|
| 通过 | 受影响黄金与边界夹具没有合同退化；无新增租户泄漏或硬事实改写 |
| 定向修复后通过 | 失败集中在明确规则/知识/提示版本，修复后只重跑受影响组 |
| 暂停发布 | 权限隔离、主产品路由、关键前提、商品事实、P5 商品边界或应用边界失败 |
| 记录观察 | 成熟度指标或概率性质量波动，没有硬边界失败 |

## 5. 阈值纪律

本包不冻结：

- 相似度阈值；
- 抽样比例；
- 模型评委组合；
- 加权总分；
- 通过率百分比；
- 永久榜单。

这些数值必须由真实黄金、成本、延迟和发布反馈校准。硬边界按合同判定，不被总分抵消。

## 6. 夹具注册方式

机器版见 `06-验证夹具注册表-v0.1.jsonl`。其中：

- AB/CDF/BE 夹具保留原始输入、预期和禁止项；
- H-V01—H-V18 补齐权限、任务、事件、反馈和知识升级等系统级边界；
- 8 个 `G-*` 仅作为外部项目黄金引用桩，不在本包猜补详细内容。
