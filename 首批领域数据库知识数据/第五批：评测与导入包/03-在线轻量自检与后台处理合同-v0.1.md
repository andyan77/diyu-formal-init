# 在线轻量自检与后台处理合同 v0.1

- 覆盖规则：35 条逻辑规则
- 注意：**不是 35 次模型调用，也不是 35 个工作流节点**；实现可合并为少量确定性检查与一次质量评审。
- 用户可见：只见成品；只有不可绕过输入/权限问题时看到一句最短行动提示。

## 1. 执行顺序

### 第 1 层：最便宜的不可绕过边界

1. 账号与发布账号操作授权；
2. 租户、品牌、工作区隔离；
3. content-production / display-merchandising 应用边界；
4. P5 是否存在真实品牌商品。

失败时停止当前成果生成，只提示一个可执行动作。对应规则：

- `H-GLOBAL-AUTH-001` 账号与发布账号操作授权
- `H-GLOBAL-TENANT-001` 租户、品牌与工作区隔离
- `H-GLOBAL-APP-001` 对外内容与对内陈列应用边界
- `H-P5-PRODUCT-001` P5 必须绑定真实品牌商品

### 第 2 层：主合同与忠实性

- 唯一主要产品和主要价值是否成立；
- 用户承重前提是否保持；
- 品牌/商品资料和人工经营输入是否被擅自改写；
- 发布账号与内容角色位置是否正确；
- 品牌相关路径是否实质成立。

普通失败先在后台重写，不把失败稿和规则报告交给用户。

### 第 3 层：知识、制作与实际性

- 知识硬度是否用对；
- 当前资源下是否可制作；
- 概念、摆拍、实拍是否诚实；
- CTA 是否承诺了不存在的能力；
- 人群、身体、家庭和地域是否被刻板化。

非承重信息缺失时直接删除、条件化或降级，不启动补数工作流。

### 第 4 层：竞争性质量

- 是否真正增加专业、结构、关系或视觉价值；
- 是否明显模板化、AI 口水化；
- 是否按 P1—P5 专属规则完成；
- 是否交付完整成品而不是提纲或后台轨迹。

## 2. 失败处理

| 处理 | 适用 | 用户看到什么 |
|---|---|---|
| `block` | 权限、隔离、应用冲突、P5 无真实商品 | 一句最短行动提示 |
| `rewrite` | 主任务、品牌相关、事实忠实、角色、模板、产品专属价值等普通质量问题 | 只看到重写后的成品 |
| `degrade` | 非必要经营事实、动态断言或不可完成 CTA | 删除相应主张后的成品 |
| `record_only` | 不用于在线规则 | 无 |

本包的在线规则中：

- `block`：4 条；
- `rewrite`：28 条；
- `degrade`：3 条。

## 3. 明确不在线执行

- 不为每次请求生成换品牌、换账号、换商品、换地区的对照稿；
- 不计算永久品牌浓度分、真诚分、AI 文风分；
- 不向用户或企业管理员显示评分、规则名、重试、淘汰候选或模型隐藏推理；
- 不因概率性模型评委单独阻断权限、合规或发布；
- 不因未上传反馈、未采用或用户离开判定负向。

## 4. 实现停止条件

当一个候选已经：

1. 通过不可绕过边界；
2. 完成主要产品价值；
3. 保持关键前提与事实；
4. 形成品牌相关路径；
5. 在当前条件下可用或可制作；

系统应立即交付，不为“更完整”继续增加审计字段、候选数量或评审回合。具体重试次数、延迟预算和模型组合由实现评测决定，本包不冻结数值。
