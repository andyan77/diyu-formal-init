## 本批采用的核心口径

- 从 P1—P5 可验收任务反推，不建设服装行业百科。
- 系统全局层分为：领域知识、创作/制作方法、行业角色模板、产品与评测规则；动态趋势单独治理，经营陈列知识延后。
- “三种硬度”只用于领域知识断言，不用于方法、模板和规范规则。
- 领域通用知识默认供所有租户任务调用；品牌、商品、账号、门店、个人与动态经营信息不进入通用层。
- 逻辑字段不等于本体类、数据库表或前端表单；对象身份继续由 D-016 决定。

## 请优先对齐的 5 项

1. 是否采用“确定性知识 / 条件性专业规律 / 审美与创意候选”作为正式三档名称。
2. 是否确认 A—H 的最新定位：A/B/C 为领域知识，D/E 为方法，F 为模板，H 为规范，G 延后。
3. 是否确认 P1—P5 追踪矩阵中的首轮 49 个知识主题作为建设需求目录，而不是知识条目数量承诺。
4. 是否确认第二批优先生产 A/B 专业核心；P5 视觉核心中受 P5-C/P5-D 与 D-032 影响的部分保持候选状态。
5. 是否确认四类卡片仅是逻辑合同，不立即映射为四张表或四个一等对象。
