# 领域通用库建设边界清单 v0.1

## 1. 一句话边界

> 只建设“无品牌属性、跨租户可复用、被 P1—P5 或其共享能力实际消费”的知识与方法；没有明确任务消费者的内容不进入第一版。

## 2. 单项准入检查

任何候选内容必须依次通过以下检查：

| 检查 | 通过条件 | 不通过时去向 |
|---|---|---|
| 1. 是否无品牌属性 | 不依赖具体品牌、SKU、门店、真人或经营主体才成立 | 品牌专属库、商品资料、组织/账号配置或任务输入 |
| 2. 是否有明确消费者 | 至少映射一个 P1—P5 主任务、共享生成能力或已确认质量门 | 不建设；进入候选池等待真实任务 |
| 3. 是否跨租户稳定 | 不因品牌、平台热点或短期经营状态变化而立即失效 | 动态信号层、品牌层或当前任务 |
| 4. 资产类型是否明确 | 领域知识 / 方法 / 行业模板 / 评测规则四选一 | 先拆分，不允许混卡 |
| 5. 硬度是否明确 | 仅领域知识断言标记为确定性、条件性或创意候选 | 未判定不得进入生产库 |
| 6. 使用范围是否可写清 | 说明支持哪个产品、在什么条件下可用、何时不能用 | 补齐边界；否则不建 |
| 7. 是否真的需要静态保存 | 保存后会改善选择、解释、创作或验收 | 可由模型即时生成且无复用价值的内容不囤积 |
| 8. 是否属于当前阶段 | 不依赖未关闭的 P5、D-032、经营陈列合同或未来模块 | 标记候选或延后，不预建空表/空模块 |

## 3. 第一版纳入范围

| 范畴 | 纳入内容 | 资产定位 | 主要产品 |
|---|---|---|---|
| A 商品与穿着知识 | 材质、结构、版型、尺码、合身、品类、洗护和表达边界 | 领域知识核心 | P1、P2，部分 P5 |
| B 穿搭与造型方法 | 比例、轮廓、色彩、材质层次、TPO、搭配取舍、通用视觉造型关系 | 领域知识核心 | P1、P5，部分 P2 |
| C 人群与生活场景 | 通用场景张力、决策因素、生活转换与服务关系 | 轻量领域知识 | P3、P4，部分 P1 |
| D 创作与叙事方法 | 判断、解释、观察、问答、演绎、幽默、结构、反 AI 口水化 | 生成方法库 | P1—P5 |
| E 媒介与制作语法 | 跨媒介稳定语法、制作资源与表达方式解耦 | 制作方法库 | P1—P5 |
| F 行业角色与组织模板 | 总部、区域、终端及商品/栏目符号的冷启动模板 | 行业模板库 | P1—P5 入驻与账号编排 |
| H 评测与质量标准 | 主路由、品牌相关、前提忠实、产品专属门、离线变形测试 | 规范与评测库 | P1—P5 |

## 4. 明确排除项

以下内容不得进入领域通用稳定核心：

- 具体品牌定位、品牌声纹、品牌审美规则、账号人设和系列前情；
- SKU 成分、价格、库存、活动、销量、设计原意和门店经营事实；
- 创始人、店长、顾客或员工的真实履历、私人关系和个人素材；
- 用户本次生活种子、门店观察、评论、对话和临时经营输入；
- 平台热点、推荐趋势、网络梗、当前尺寸/时长/格式规则；
- 发布反馈、播放、评论、成交和系统高分结果；
- 经营陈列执行知识、巡检标准和整改知识正文；
- 未立项未来模块的空对象、空接口、空字段和万能 JSON；
- 仅“行业里存在”但没有 P1—P5 消费者的百科内容。

## 5. 分流规则

| 内容性质 | 正确去向 |
|---|---|
| 品牌、商品、账号、组织的已确认专属信息 | 品牌专属知识或配置层 |
| 用户本次生活/工作/经营输入 | `task_seed` 当前任务范围 |
| 用户主动保存的私人内容素材 | 本人个人素材区 |
| 组织授权图片、视频、设计稿 | 组织官方素材区 |
| 平台趋势与热点 | 带观察时间和失效时间的动态信号层 |
| 内容成品、版本和行为 | 内容资产与 GenerationEvent |
| 账号/系列已讲过什么 | 内容历史轻量投影或系列卡片 |
| 经营陈列知识 | `display-merchandising` 模块，待独立合同激活 |

## 6. 建设纪律

1. **任务反推**：每项内容必须标明支持哪个产品、哪类任务和哪种失败修复。
2. **够用即停**：黄金任务已无关键知识缺口时停止扩张。
3. **单项单硬度**：实际领域知识卡只标一个主要硬度；混合内容必须拆卡。
4. **条件优先**：涉及身体、比例、面料表现、场合和审美的结论默认先检查是否属于条件性规律。
5. **创意轻库化**：创意候选保存变化轴和反套路边界，不堆积大量成品句子。
6. **不提前物理化**：本批只定义逻辑合同，不决定表、JSON、投影或本体身份。
7. **平台与趋势分离**：跨媒体稳定方法可建；平台专属规则等待 D-032。
8. **陈列只留边界**：G 只保留模块命名与共享核心边界，不创建首版知识正文。

## 7. 单条知识进入首版的验收条件

- 有明确的 P1—P5 消费任务；
- 无品牌、SKU、门店、真人或短期经营属性；
- 资产类型和硬度已确定；
- 条件性规律写明条件、可能效果、代价和不适用情形；
- 创意候选不会被生成器当作行业真理；
- 能改变选择、解释、创作或验收结果；
- 与已有条目不重复；
- 不要求普通用户理解底层结构或填写技术字段。

## 8. 首轮停止条件

满足以下条件即停止扩建并转入真实任务验证：

- P1—P5 黄金任务所需专业知识无关键空缺；
- 新增条目已很少改变成品判断或质量；
- 高频问题主要来自品牌资料、商品资料、媒体实现或提示编排，而非通用知识缺失；
- 没有未处理的高风险硬度误分类；
- 后续候选没有明确产品消费者。
