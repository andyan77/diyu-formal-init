六层技术架构怎样贯穿路线图？

六层是长期稳定的责任边界，不是六个串行建设阶段或六套服务；每个软件里程碑只实现当前用户结果真正消费的纵向切片。

L1 · 交互 UI

  • M0 定义自然交互与成品语义
  • M3 跑通 P1 页面
  • M4 打开双应用入口
  • M5 完成平台、入驻与素材入口

L2 · 应用核心

  • 服务端可信身份和作用域
  • P1—P5 与陈列价值路由
  • 成品版本、系列、保存和事件
  • 不拆成三套 Agent 或 DIFY 工作流

L3 · 上下文编译 / RAG

  • M2 冻结先过滤再检索
  • M3 从 SQL 精确查询起步
  • 未来向量必须先限合法范围再 Top-K
  • 不把检索投影当权威真源
