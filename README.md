# 笛语 · 双应用合成演示

打开 `http://127.0.0.1:8000` 后，先选择“租户用户入口”或“租户管理入口”。两者使用独立 React 工作壳、互不展示页面或导航，并由服务器按当前合成会话资格失败关闭。租户用户入口再选择内容生产（对外）或陈列搭配（对内）；租户管理入口承载当前入驻、发布账号及已登记操作人，并可创建一个企业发布账号、它独立的表达身份和一位已登记操作者的最小授权。

这仍是合成会话，不是生产登录或真实授权。普通用户不能读取租户管理页面或 API；仅有管理资格的自然人不能读取业务工作台；同一自然人可持有两个入口资格，但每次只看当前工作空间。企业发布账号不是登录账号：多个内部或外部代运营者必须分别登记为自然人并获授权，不能共享发布账号密码。每个自然人最多维护一份本人默认表达人设，企业发布账号的 `ContentRole` 独立存在。正式开户、一次性激活、密码、三个生产入口及平台运维台均留在 M5-4。

内容工作台位于 `/content`，陈列工作台位于 `/display`；两者均须从首页进入后使用。内容页可在“总部内容账号”和“南城店内容账号”两个服务器固定的**合成演示**身份间切换；这不是生产账号管理或真实认证。连续系列由服务器绑定当前可信发布账号，不能混入另一账号的内容任务。

## M5-1 五类内容与 M5-2 媒体重编译

内容工作台可选择抖音视频、小红书视频、小红书图文或微信视频号视频；默认抖音视频。总部演示身份的三个发布账号是服务器固定的独立记录，南城店演示身份只可使用自己的抖音账号；页面和 API 都不接收或信任账号 ID。

直接自然输入即可获得友好交流或一份完整文字制作成品；无需选择 P1—P5。服务端只为形成内容任务的输入确定一个主要受众价值：帮助选择、解释商品、建立账号人格、经营在地关系或创造可见的穿着可能。普通交流不创建任务、运行或版本；同目标内自然修改保留 V1 并生成 V2。用户从既有成品说“改成小红书图文”或“换成视频号”时，系统保留源版本、产品价值、账号角色、商品事实和结论，只新建当前目标的一份成品，并用自然语言标明来源关系。

视频交付从封面/首帧到发布配文的完整可执行观看链；小红书图文交付首图、图序与每张职责、完整正文和必要排版提示的完整阅读链。它们都是文字型待制作成品：不会生成 MP4、图片或音频，不连接平台发布、账号管理、素材库、趋势或自动发布。

当前合成 ZX-C218 由内容与 DM01 共用 `brand_products` 商品真源。已知事实仅包括炭灰纯色、深绿细格纹、两面完整外观和口袋可用，以及当前 M 码样衣约 960g 与受控同季同长度单层短外套约 650g 的差异；系统不把差异全部归因于双面结构，也不编造价格、库存、面料性能、普遍上身结果或设计动机。

## M4-1 陈列搭配（对内）

同一启动入口也提供 `http://127.0.0.1:8000/display`。南城店陈列执行人员只需自然输入本次可用商品和数量，例如 `ZX-C218 3 件、ZX-S104 3 件`，即可得到 DM01 双层挂杆文字执行建议；自然反馈会生成不可覆盖的下一版本，旧版本仍可读取。此入口使用服务端可信租户、品牌、南城店组织和操作人，不接收这些作用域字段，不使用发布账号或内容人设。

人工清单只在当前陈列任务有效：不会写回或核验 ERP，也不会声称总部批准、系统巡检或门店已经完成。DM01 使用正式的确定性陈列编译器 `dm01-rule-compiler-v1`：库存、品牌陈列标准、南城店挂杆档案和当前适用资产编译为唯一 layout，结构合同校验后再生成用户可见文字。它不调用 DeepSeek，也不把模型正文作为陈列事实。

本仓库提供本地合成演示：自然弱种子生成可执行视频或图文文字成品、自然语言生成 V2、保留 V1、主动保存，以及只在同一用户和品牌的获授权源版本上进行明确改编。用户说“接着上一条”且当前目标唯一时，无需先保存即可续改；输入普通交流不会创建任务。默认是可重复的离线替身模式，页面会明确标记，不冒充真实模型。

领域候选由本项目迁移后的系统公共目录登记：243 条均可追溯，当前有 41 条激活（P1 4 条、DM01 13 条、P2—P5 13 条、M5-2 媒体方法 11 条），其余 202 条仍是候选。当前候选快照的路径与 SHA-256、41 条激活的消费者/适用条件及四条既有 H/夹具测试绑定统一位于 `首批领域数据库知识数据/第五批：评测与导入包/12-系统资产运行治理-v1.json`；数量是当前快照事实，不是导入合同。资产生命周期只用 `review_candidate`、`active`、`deprecated`；只有未过期且未被取代的 `active` 资产可进入运行上下文。每个生成运行只记录本次实际使用的资产 ID、版本、主要产品、目标平台/媒体、平台方向版本、制作条件和最小商品引用回执；全量登记、激活、实际调用和生产质量证明彼此不同，用户页面不显示资产 ID、提示词或运行轨迹。

`ContentBrief` 是内容任务在运行时的语义概念，现由 `business_tasks` 的目标/上下文及其 `content_items`、`content_versions` 共同承接；第一阶段不新增同义表或对象。

本地启动：

```bash
export DIYU_SESSION_SECRET='仅限本机的随机值'
scripts/run_app.sh
```

随后打开 `http://127.0.0.1:8000`。本地完整回归入口为 `make golden`；它创建/使用工作区内 PostgreSQL、合成演示数据和模型替身，并固定项目内 Linux 临时目录，不受 Windows `TEMP`/`TMP` 继承影响。

每次推送到 `main`、面向 `main` 的 PR 和手动触发都会在单个 Linux CI 作业中运行前端 lint/typecheck/build 以及 `make lint`、`make typecheck`、`make golden`；CI 不读取模型、ECS、生产数据库或密钥。

真实 DeepSeek 模式服务同一内容编译器的价值判断与 P1—P5 成品生成，必须由服务器环境提供 `DIYU_GENERATOR_MODE=deepseek`、`DEEPSEEK_API_BASE_URL`、`DEEPSEEK_API_KEY` 和已核验的 `DEEPSEEK_MODEL`。受保护配置只能在服务器环境安全引用，绝不提交；本仓不会内置密钥。DM01 不因模型可用性阻塞，也不通过 SSH 隧道调用模型。
