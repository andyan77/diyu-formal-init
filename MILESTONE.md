## 原失败、根因与最终修复

- 2026-07-24 早期正式 P2 的连续失败和不合格候选保留为历史诊断证据；它们不再是尚未裁决的外部模型阻断。
- 原根因是正式 API 的完整上下文、初稿、语义判定和局部修复组合与直接供应商诊断不同：机械词面检查误伤自然表达，局部修复又会复述或改写被拒绝片段，并遗漏间接/评价性重量归因及声画合同矛盾。
- 最终修复改为“显式商品事实投影 + 同供应商语义判定 + 关系级确定性反证 + 一次字段级修复 + 只删除最终拒绝句”的有界路径；修复请求不再回显违规片段或暗示候选原因，视频口播、字幕、声音和自然时长在提交前一致化。
- 边界没有降低：错误 SKU/数字/重量/颜色，未提供性能、材质、工艺、部位或设计动机，具体人物/门店/交易事件，个人标识/内部信息，未来拍摄冒充实测，以及第二件对照实物仍失败关闭；普通服装词、条件性解释和创意表达不因关键词本身被拒绝。
