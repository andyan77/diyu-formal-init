尚有两个限定阻断：

1. `ContentSeries` 没有绑定 `PublishingAccount`。主控已用现有 API 复现：同一系列先加入抖音任务、再加入小红书图文任务，两次都返回 `200`，导致账号连续性可能串线。系列必须由服务端可信 `scope.account_id` 绑定发布账号，列表和增删改均按该账号收窄，其他账号任务必须失败关闭；客户端不得提交账号 ID。
2. 渐进入驻当前只能确认品牌表达、查看演示种子中的既有发布账号并为操作者授权，没有从租户管理入口创建一个新的 `PublishingAccount`、关联一个独立 `ContentRole` 并授权给已登记操作者的可用路径。必须补一个最小自然表单和服务端合同，不建设账号矩阵、审批流、动态认证或完整 IAM。

修复只需增加上述两条定向反证，再运行一次前端构建、现有全量回归和远程 CI。两项都不改变模型、Prompt、生成正文或正式页面调用链，因此不得重跑 DeepSeek 冒烟、P1—P5 六卡或媒体五份成品。
